# ✅ dq-6d-check

## When to Use

当出现以下场景时调用本技能：

- **数据资产准入**评审时，需要评估数据集质量是否达标
- **系统上线前**数据质量评审，识别上线后可能导致问题的数据缺陷
- **周期性巡检**（月度/季度），监控数据质量的趋势变化
- **数据交易挂牌前**质量评分（配合 `trading/listing-quality-scorer`）
- **数据质量事件**发生后定位问题根因

本技能基于 GB/T 36344-2018《信息技术 数据质量评价指标》定义的六大维度，是数据资产管理的**质量基线工具**。

## 工作原则

1. **六维全覆盖**：每次检查必须覆盖全部六个维度，不可跳过——即使某维度无问题也需标注"通过"
2. **量化优先**：每个维度尽量输出量化指标（百分比、比率），而非定性描述
3. **问题分级**：发现的问题按严重程度分级（阻断/严重/一般/建议），阻断级问题必须修复后方可准入
4. **可追溯**：每条问题标注对应的字段、规则、标准条款

## 六大质量维度（GB/T 36344-2018 §4）

### 1. 完整性（Completeness）

数据记录和字段是否齐全，无缺失。

| 检查项 | 计算方式 | 达标标准 |
|--------|---------|---------|
| 记录完整性 | 实际记录数 / 预期记录数 | ≥99.5% |
| 字段完整性 | 非空记录数 / 总记录数 | ≥95%（核心字段100%） |
| 时间完整性 | 缺失时间段数 / 总时间段数 | ≤0.5% |

### 2. 准确性（Accuracy）

数据值与真实值的吻合程度。

| 检查项 | 计算方式 | 达标标准 |
|--------|---------|---------|
| 值域准确率 | 合法值记录数 / 总记录数 | ≥99% |
| 格式准确率 | 格式合规记录数 / 总记录数 | ≥98% |
| 业务逻辑准确率 | 满足业务规则记录数 / 总记录数 | ≥95% |

### 3. 一致性（Consistency）

同一数据在不同表/系统中的取值是否一致。

| 检查项 | 计算方式 | 达标标准 |
|--------|---------|---------|
| 跨表一致性 | 一致记录数 / 对比记录数 | ≥99% |
| 跨系统一致性 | 一致记录数 / 对比记录数 | ≥98% |
| 编码一致性 | 符合统一编码记录数 / 总记录数 | 100% |

### 4. 时效性（Timeliness）

数据更新的及时程度。

| 检查项 | 计算方式 | 达标标准 |
|--------|---------|---------|
| 更新延迟 | 实际更新时间 - 预期更新时间 | ≤SLA约定 |
| 数据新鲜度 | 最新数据时间戳与当前时间差 | ≤业务要求 |
| 延迟到达率 | 延迟到达记录数 / 总记录数 | ≤1% |

### 5. 唯一性（Uniqueness）

数据记录无重复。

| 检查项 | 计算方式 | 达标标准 |
|--------|---------|---------|
| 主键唯一性 | 唯一主键数 / 总主键数 | 100% |
| 业务唯一性 | 唯一业务键数 / 总业务键数 | 100% |
| 重复率 | 重复记录数 / 总记录数 | 0% |

### 6. 合规性（Compliance）

数据是否符合安全、隐私和标准规范。

| 检查项 | 计算方式 | 达标标准 |
|--------|---------|---------|
| PII 合规率 | 已脱敏PII字段数 / PII字段总数 | 100% |
| 授权完备率 | 有授权记录数据集数 / 总数据集数 | 100% |
| 标准符合率 | 符合标准字段数 / 总字段数 | ≥95% |

## Inputs

- **数据集信息**（object）：
  - `name`：数据集/表名
  - `fields`：字段列表（含类型、约束，来自 `metadata-extract`）
  - `row_count`：总记录数
  - `sample_data`：样本数据（前 100~1000 行）
- **业务规则**（array）：业务校验规则（如"金额>0""下单时间≤支付时间"）
- **对照基准**（object，可选）：用于一致性检查的参照数据
- **SLA 要求**（object，可选）：时效性约定的 SLA

## Steps

### 步骤一：完整性检查

1. 检查记录数是否达到预期（与历史均值/源系统对比）
2. 检查核心字段空值率（NOT NULL 字段不应有空值）
3. 检查时间序列是否有断档（按日/小时分桶检查）
4. 量化输出：记录完整率、字段完整率

### 步骤二：准确性检查

1. 值域校验：数值字段是否在合理范围（如金额>0、年龄0-150）
2. 格式校验：手机号11位、邮箱格式、日期格式
3. 业务规则校验：执行用户提供的业务规则（如下单时间≤支付时间）
4. 量化输出：值域准确率、格式准确率、业务逻辑准确率

### 步骤三：一致性检查

1. 跨表对比：与关联表的关键字段（如 user_id → user_status）比对
2. 跨系统对比：如有参照数据源，比对同一实体的取值一致性
3. 编码一致性：检查编码字段是否使用统一编码表
4. 量化输出：跨表一致率、跨系统一致率

### 步骤四：时效性检查

1. 检查最新数据时间戳与当前时间的差值
2. 检查更新频率是否符合 SLA 约定
3. 检查延迟到达记录的比例
4. 量化输出：更新延迟、数据新鲜度

### 步骤五：唯一性检查

1. 主键唯一性：SELECT 主键 COUNT vs COUNT DISTINCT
2. 业务唯一键检查（如订单号、交易流水号）
3. 量化输出：重复率

### 步骤六：合规性检查

1. 扫描是否有 PII 字段未脱敏（调用 `pii-scan` 结果）
2. 检查数据采集是否有授权记录
3. 检查字段是否符合 GB/T 40685 元数据规范
4. 量化输出：PII 合规率、授权完备率、标准符合率

### 步骤七：生成质量报告

1. 计算各维度得分与总评得分
2. 汇总问题清单并按严重程度排序（阻断 > 严重 > 一般 > 建议）
3. 输出修复建议与准入结论
4. 如涉及挂牌评审，转交 `trading/listing-quality-scorer` 综合评分

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "这个数据质量怎么样？" | 执行六维检查，输出质量画像 |
| "能上线吗？" | 检查阻断级问题，有阻断项则不可上线 |
| "数据重复率高吗？" | 执行唯一性检查，输出重复率 |
| "字段空值多吗？" | 执行完整性检查，输出字段空值率 |
| "跟另一个表对得上吗？" | 执行跨表一致性检查 |
| "身份证号怎么存了？" | 触发合规性检查，标注为阻断级问题 |
