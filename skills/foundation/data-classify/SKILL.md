# 🗂️ data-classify

## When to Use

当出现以下场景时调用本技能：

- **数据资产入库**时需要确定分级分类标签
- **跨境传输评估**前需要判定数据是否属于"重要数据"
- **数据共享/披露**前需要根据级别确定保护措施
- **数据安全管理制度**建设中需要建立分级分类体系
- 响应监管要求需要出具数据分级分类报告

本技能通常在 `pii-scan`（识别个人信息）之后执行，将 PII 扫描结果与数据内容特征综合判定正式的分级分类。

## 工作原则

1. **分级与分类双维输出**：分级（安全保护级别）和分类（数据性质类别）是两个独立维度，必须同时输出
2. **就高不就低**：数据集包含多种级别数据时，整体取最高级别
3. **动态调整**：分级分类不是一次性的，数据用途、法规变化、脱敏处理后级别可能调整
4. **保护措施与级别匹配**：不同级别对应不同的存储、传输、访问、审计要求，不可降级保护

## 分级规则（GB/T 43697-2024 §5）

### 核心数据

影响国家安全、经济运行、社会稳定或涉及重要群体的数据。
- 判定依据：数据安全法第二十一条，由国家核心数据目录界定
- 保护要求：最高级加密 + 专线传输 + 严格审批 + 专人管理
- 示例：宏观经济运行核心指标、大规模人口健康基因数据

### 重要数据

一旦遭到篡改、破坏、泄露或者非法获取、非法利用，可能危害国家安全、公共利益的数据。
- 判定依据：达到一定规模阈值或涉及重要领域
- 保护要求：加密存储 + 权限控制 + 审计日志 + 定期评估
- 示例：百万级用户个人信息、行业关键基础设施数据

### 一般数据

不属于核心数据和重要数据的其他数据。
- 保护要求：基本访问控制 + 操作日志
- 示例：公开统计数据、匿名化后的统计数据

**关键阈值参考**（具体以行业规定为准）：

| 数据类型 | 一般 | 重要 | 核心 |
|---------|------|------|------|
| 个人信息 | <100万条 | ≥100万条 | 涉及国家安全 |
| 敏感个人信息 | <10万条 | ≥10万条 | 涉及重要群体 |
| 精准地理位置 | <1万人 | ≥1万人 | 涉及军事/关键设施 |

## 分类规则（GB/T 43697-2024 §6）

| 类别 | 定义 | 典型场景 |
|------|------|---------|
| 个人数据 | 以电子或其他方式记录的与已识别或可识别的自然人有关的各种信息 | 用户表、客户画像、行为日志 |
| 公共数据 | 国家机关、事业单位等在依法履职或提供公共服务中产生的数据 | 政务数据、气象数据、交通数据 |
| 法人数据 | 企业、社会团体等法人在生产经营活动中产生或获取的数据 | 企业经营数据、供应链数据 |
| 衍生数据 | 对原始数据进行加工、聚合、分析等处理后产生的数据 | 统计报表、模型输出、聚合指标 |

## Inputs

- **数据集信息**（object）：
  - `name`：数据集名称
  - `description`：数据集描述
  - `fields`：字段列表
  - `data_volume`：数据量（记录数）
  - `source`：数据来源
- **PII 扫描结果**（object，可选）：来自 `pii-scan` 的输出
- **业务场景**（string）：数据用途（内部使用/对外服务/共享/交易/出境）

## Steps

### 步骤一：数据特征采集

1. 获取数据集元信息：字段列表、数据量、数据来源、业务用途
2. 如已有 `pii-scan` 扫描结果，直接引用；否则建议先执行 PII 扫描
3. 识别数据中是否包含特殊类型：国家秘密、行业关键数据、未成年人数据

### 步骤二：分级判定

1. 按分级规则表逐一对照，先判断是否属于核心数据，再判断重要数据，最后归为一般数据
2. 就高不就低：数据集同时包含多级别数据时，整体取最高级别
3. 分级依据附标准溯源标注：`[GB/T 43697-2024 §5.2]`

### 步骤三：分类判定

1. 按数据来源和性质判定主分类（个人/公共/法人/衍生）
2. 如数据集包含多类型数据，标注主分类和包含的其他分类
3. 分类依据附标准溯源标注：`[GB/T 43697-2024 §6.1]`

### 步骤四：保护建议生成

1. 根据判定级别，从保护措施对照表中匹配对应的存储/传输/访问/审计要求
2. 根据业务场景（内部/共享/交易/出境）补充场景特定要求
3. 如涉及跨境传输，触发高风险预警，建议转交 `trading/cross-border-checker`

### 步骤五：输出分级分类报告

1. 输出完整的分级分类结论及依据
2. 附保护要求清单
3. 如后续需要脱敏，转交 `de-identification-toolkit`

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "这个数据集是什么级别？" | 执行分级流程，输出级别及判定依据 |
| "100万条用户数据算重要吗？" | 按阈值表判定，≥100万个人信息为"重要" |
| "脱敏后级别会降吗？" | 说明脱敏后可重新评估，如匿名化达标可降为"一般" |
| "数据出境需要什么级别？" | 说明"重要"及以上需申报安全评估，转交 `cross-border-checker` |
| "公共数据怎么分级？" | 公共数据仍需按内容分级，如气象数据一般为"一般"，关键政务数据可能为"重要" |
| "衍生数据怎么分类？" | 衍生数据独立分类，但需溯源到原始数据级别 |
