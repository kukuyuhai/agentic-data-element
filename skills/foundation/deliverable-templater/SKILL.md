# 📐 deliverable-templater

## When to Use

当出现以下场景时调用本技能：

- 用户请求某个**可交付物**（如"生成一份评估报告""出一份审计底稿""做一个数据战略画布"）但未提供模板
- 需要快速生成结构化文档骨架，避免手工排版
- 需要根据当前岗位的职责定义，选择正确的交付物模板
- 需要将多个交付物组合为完整的项目交付包

本技能读取当前 agent 的角色定义文件（`agents/*.md`）中的"📋 交付物模板"段落，生成可直接填充的结构化骨架。

## 工作原则

1. **模板来自角色定义**：所有模板源自 `agents/` 下对应岗位 md 文件的"交付物模板"段落，不自行编造
2. **骨架即用**：生成的骨架包含标题、章节结构、占位符提示，用户可直接填充内容
3. **多模板可选**：每个岗位通常有 3+ 交付物模板，根据用户请求选择匹配的模板
4. **可组合**：支持将多个交付物模板组合为项目交付包

## Inputs

- **当前 agent id**（string）：如 `dam-07`，用于定位角色定义文件
- **交付物名称**（string）：用户请求的交付物，如"评估报告""审计底稿"
- **可选·填充数据**（object）：用户已提供的部分数据，用于预填模板
- **可选·输出格式**（enum）：`markdown`（默认）/ `json` / `yaml`

## Steps

### 步骤一：定位角色定义文件

1. 根据当前 agent id（如 `dam-07`），找到对应的角色定义文件路径（如 `agents/data-asset-management/dam-07-data-asset-appraiser.md`）
2. 读取该文件的"📋 交付物模板"段落
3. 提取所有可用模板列表

### 步骤二：匹配用户请求

1. 将用户请求的交付物名称与模板列表匹配
2. 如匹配到多个，列出供用户选择
3. 如未匹配到，建议最接近的模板或提示用户描述更具体

### 步骤三：生成骨架

1. 从模板定义中提取章节结构
2. 为每个章节生成占位符提示（形如 `[待填写：XX说明]`）
3. 如用户提供了填充数据，将数据预填到对应位置
4. 保留模板中的固定措辞和标准引用

### 步骤四：附加溯源信息

1. 在模板末尾附溯源标注，标明模板来源和对应标准
2. 如该交付物需要配合其他技能（如评估报告需要 `income-approach-calc`），列出建议技能
3. 如该交付物需要其他岗位协同（如评估报告需确权信息），标注协作需求

### 步骤五：输出

1. 输出结构化骨架（Markdown/JSON/YAML）
2. 提示用户哪些占位符需要填写
3. 建议后续使用哪些技能来填充具体内容

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "生成一份评估报告" | 匹配 dam-07 的评估报告模板，生成骨架 |
| "出一份审计底稿" | 匹配 dam-09 的审计底稿模板，生成四大认定骨架 |
| "做一个数据战略画布" | 匹配 dam-01 的战略画布模板，生成六要素骨架 |
| "有 9 步交易清单吗？" | 匹配 dat-05 的 9 步清单模板，生成步骤卡 |
| "我自己有模板能用吗？" | 用户自带模板优先，本技能仅做骨架补全 |
| "能把几个模板组合吗？" | 支持组合为项目交付包（如评估+审计+保荐意见） |
