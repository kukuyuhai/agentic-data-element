# 🔍 pii-scan

## When to Use

当出现以下场景时调用本技能：

- **来源不明的数据集**接入前需要评估个人信息风险
- **数据共享/交易前**需要识别其中是否包含个人信息或敏感信息
- **数据脱敏前**需要先识别哪些字段需要脱敏、脱敏到什么程度
- **资产盘点**时需要标注每个数据集的 PII 风险等级
- **跨境传输**前需要判断是否涉及个人信息出境（配合 `trading/cross-border-checker`）

本技能是数据安全合规的**前置关卡**，在 `data-classify`（分级分类）之前执行，识别结果作为分级的输入。

## 工作原则

1. **宁可多识别不可遗漏**：敏感信息遗漏的代价远高于误报，对疑似字段标注"疑似"而非直接排除
2. **以个保法为准绳**：敏感信息认定以《个人信息保护法》第二十八条为准，GB/T 43697 提供技术分级框架
3. **上下文感知**：同一字段在不同上下文中敏感度不同（如"手机号"在用户表 vs 通话记录中），需结合数据集整体语义判断
4. **区分识别与脱敏**：本技能只负责识别和定级，脱敏执行转交 `role/data-asset-security-compliance/de-identification-toolkit`

## 敏感信息识别规则

### 一、个人信息（个保法第十三条）

以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息，包括但不限于：

| 类别 | 典型字段 | 识别特征 |
|------|---------|---------|
| 身份标识 | 身份证号、护照号、军官证号 | 唯一标识自然人身份 |
| 联系方式 | 手机号、邮箱、固话 | 可直接联系到自然人 |
| 位置信息 | GPS 坐标、IP 地址、MAC 地址 | 可定位自然人行踪 |
| 生物特征 | 人脸、指纹、声纹、虹膜 | 不可更改的生物标识 |
| 账号信息 | 用户 ID、账号名、设备 ID | 间接关联到自然人 |
| 行为数据 | 浏览记录、消费记录、搜索记录 | 反映自然人行为偏好 |

### 二、敏感个人信息（个保法第二十八条）

一旦泄露或者非法使用，容易导致自然人的人格尊严受到侵害或者人身、财产安全受到危害的个人信息：

| 类别 | 典型字段 | 风险说明 |
|------|---------|---------|
| 生物识别 | 人脸、指纹、基因、声纹 | 泄露后不可更改，身份冒用风险极高 |
| 宗教信仰 | 宗教信仰、宗教活动记录 | 歧视风险 |
| 特定身份 | 民族、政治面貌、工会会员 | 歧视风险 |
| 医疗健康 | 病历、体检报告、基因检测 | 隐私极高，泄露后果严重 |
| 金融账户 | 银行账号、支付密码、征信记录 | 财产安全风险 |
| 行踪轨迹 | 实时位置、出行记录、住宿记录 | 人身安全风险 |
| 不满 14 岁未成年人 | 儿童个人信息 | 特殊保护要求 |

### 三、敏感等级建议（GB/T 43697-2024）

| 等级 | 判定条件 | 保护要求 |
|------|---------|---------|
| 核心 | 涉及国家安全/经济运行/重要群体 | 最高级加密 + 专线传输 + 审批访问 |
| 重要 | 涉及敏感个人信息或大规模个人信息 | 加密存储 + 权限控制 + 审计日志 |
| 一般 | 涉及少量个人信息或非敏感数据 | 基本访问控制 + 日志记录 |

## Inputs

- **数据集描述**（object）：
  - `fields`：字段列表（字段名、类型、示例值或描述）
  - `source`：数据来源（自采/外购/共享/公开）
  - `context`：数据用途（内部使用/对外服务/交易/共享）
- **可选·样本数据**（array）：前 N 行样本数据（用于辅助识别）
- **可选·已知约束**（array）：已知的脱敏或匿名化处理情况

## Steps

### 步骤一：字段级扫描

1. 遍历所有字段，根据字段名、类型、示例值匹配 PII 识别规则
2. 对疑似字段标注"疑似"并说明判断依据
3. 识别组合标识符：单个字段不敏感，但多个字段组合可定位个人（如"邮编+生日+性别"）

### 步骤二：敏感等级判定

1. 对每个识别为 PII 的字段，按个保法第二十八条判定是否属于敏感个人信息
2. 根据数据集整体包含的 PII 数量和敏感度，判定数据集整体等级（核心/重要/一般）
3. 判定依据附标准溯源标注：`[GB/T 43697-2024 §5.2]` `[《个人信息保护法》第二十八条]`

### 步骤三：生成处理建议

1. 对每个 PII 字段给出脱敏建议（掩码/哈希/加密/泛化/删除）
2. 对数据集整体给出合规建议（同意获取/最小必要/访问控制/审计日志）
3. 如涉及跨境传输或大规模个人信息，触发高风险预警

### 步骤四：输出与流转

1. 输出扫描报告供用户确认
2. 如需要执行脱敏，转交 `de-identification-toolkit` 选择具体技术方案
3. 如需要分级分类，将扫描结果传递给 `data-classify` 执行正式分级

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "这批数据有没有个人信息？" | 执行字段级扫描，列出所有 PII 字段 |
| "手机号算不算敏感信息？" | 查个保法第二十八条，手机号属于个人信息但不一定属于敏感信息，需结合上下文判断 |
| "这个数据集能直接共享吗？" | 扫描后如含敏感 PII，建议先脱敏再共享 |
| "人脸数据怎么处理？" | 标注为生物识别敏感信息，等级"核心"，建议加密+审批 |
| "字段名看不出是不是 PII" | 结合示例值判断，如仍不确定标注"疑似" |
| "脱敏方案怎么做？" | 识别完成后转交 `de-identification-toolkit` 执行 |
