# 🔖 citation-attach

## When to Use

当 agent 产出的内容包含**能力性判断**或**合规结论**时，必须调用本技能在每条结论末尾附加溯源标注。典型场景：

- 产出**评审意见**（如合规评审、入表审核、挂牌准入评审）
- 产出**方案设计**（如交易结构设计、数据共享方案）
- 产出**审计底稿**或**评估报告**
- 产出**检查清单**的判定结论
- 用户明确要求"需要可追溯""标注依据"

本技能是**全员挂载的合规保障能力**，确保所有能力性输出可追溯到标准或法规条文。

## 工作原则

1. **结论即标注**：每一条带有"应当/不得/建议/推荐"等判断性措辞的结论，末尾必须附标注
2. **标注格式统一**：标准用 `[GB/T XXX §X.X]`，法规用 `[《法规名》第X条]`，岗位标准用 `[T/MIITEC 025-2024 §5.X.X]`
3. **标注必须真实**：引用的条款必须存在且内容匹配，不可编造条款号
4. **多重依据全列**：如结论同时依据标准与法规，全部列出，用分号分隔
5. **概括性陈述豁免**：纯描述性内容（如"数据资产是指……"）不需要标注，但规范性内容（如"应当分类为……"）必须标注

## 标注格式规范

### 标准（GB/T、T/）

```
数据分级应遵循最小必要原则。[GB/T 43697-2024 §5.1]
```

### 法律法规

```
处理个人信息应当取得个人同意。[《个人信息保护法》第十三条]
```

### 岗位标准

```
确权师应具备数据产权理论知识。[T/MIITEC 025-2024 §5.1.3]
```

### 多重依据

```
数据资产入表应按成本法初始计量。[财会〔2023〕11号 第四条；GB/T 42129-2022 §6.1]
```

### 引用交叉技能

```
出境合规判定转交 cross-border-checker 执行。[T/MIITEC 025-2024 §5.2.2]
```

## Inputs

- **待标注文本**（string）：agent 产出的结论性文本，可能是一段评审意见、一组检查项判定、一份方案设计
- **可选·标注范围**（enum）：`all`（全部结论，默认）/ `conclusions_only`（仅"应当/不得"类结论）/ `custom`（用户指定关键词触发）
- **可选·当前场景**（string）：当前产出类型（report/review/plan/audit），用于调整标注密度

## Outputs

```yaml
original: "数据资产应确认为无形资产，按使用寿命摊销。"
annotated: "数据资产应确认为无形资产，按使用寿命摊销。[财会〔2023〕11号 第四条、第六条]"

  - source: "财会〔2023〕11号"
    article: "第六条"

    excerpt: "使用寿命有限的无形资产，在使用寿命内系统合理摊销……"
unannotated_clauses: []
```

## Steps

### 步骤一：识别结论性语句

1. 扫描待标注文本，识别以下类型的语句：
   - **规范性结论**：含"应当""必须""不得""禁止""应"等措辞
   - **建议性结论**：含"建议""推荐""宜""可"等措辞
   - **判定性结论**：含"符合""不符合""通过""未通过""合规""不合规"等措辞
   - **分类性结论**：含"属于""归类为""分为""应确认为"等措辞
2. 纯描述性内容（定义解释、背景说明、操作步骤描述）不标注

### 步骤二：匹配溯源依据

1. 对每条结论性语句，通过 `standard-lookup` 和 `regulation-lookup` 检索对应条款
2. 匹配优先级：
   - 先匹配**直接依据**（结论直接引用的条款）
   - 再匹配**间接依据**（支撑该结论的相关条款）
3. 如果无法匹配到具体条款，标注 `[依据待补充]` 并提示用户确认

### 步骤三：插入标注

1. 在结论性语句末尾（句号前或句号后）插入标注
2. 多重依据用分号分隔，按效力层级排列（法律 > 法规 > 标准）
3. 保持原文流畅性，标注不破坏语句结构

### 步骤四：输出引用清单

在文本末尾附**引用清单**，列出所有标注的完整出处，便于审计追溯：

```
---
引用清单：
1. [财会〔2023〕11号 第四条] 数据资源成本构成……
2. [GB/T 43697-2024 §5.1] 数据分级原则……
```

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "这段意见标注一下依据" | 扫描全文，对规范性结论逐条附标注 |
| "报告需要可追溯" | 对全报告执行标注，并在末尾附引用清单 |
| "这个判断有没有依据？" | 单条检索，返回对应条款原文 |
| "标注太密了" | 切换为 `conclusions_only` 模式，仅标注"应当/不得"类结论 |
| "这条依据不确定" | 标注 `[依据待补充]`，建议用户人工确认或调用 standard-lookup 深入检索 |
