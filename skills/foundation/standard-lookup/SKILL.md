# 📚 standard-lookup

## When to Use

当出现以下场景时调用本技能：

- 用户询问"某项做法有没有标准依据""对应哪条条款"
- 需要在报告、评审意见、方案设计末尾附溯源标注（形如 `§5.X.X` 或 `GB/T XXX §X`）
- 需要校验当前做法是否满足标准要求
- 跨岗位协作时需要快速确认某岗位的标准章节归属

本技能是**全员挂载的基础能力**，不依赖外部 API，所有条文来自项目内置的标准知识库与公开资料。

## 工作原则

1. **原文优先**：返回的条款必须是标准原文片段，不可自行改写或概括后替代原文
2. **条款号必须带出**：每条引用必须附带标准编号 + 条款号（如 `GB/T 40685-2021 §6.2`），便于下游追溯
3. **多标准交叉引用**：同一主题可能涉及多个标准（如"数据分类分级"同时出现在 GB/T 43697 与 GB/T 40685），需列出所有相关出处
4. **区分强制与推荐**：国标（GB）中的"应"为强制要求，"宜"为推荐，"可"为可选，引用时需标注

## 标准索引

本技能覆盖以下标准的常用条款检索：

| 标准编号 | 标准名称 | 核心覆盖范围 |
|---------|---------|------------|
| T/MIITEC 025-2024 | 数据要素产业人才岗位能力要求 | 17 个岗位的能力定义、知识技能要求（§5.1.1~§5.2.6） |
| GB/T 40685-2021 | 信息技术 数据要素 术语 | 数据要素术语定义、数据资产分类、元数据最小集（§5~§7） |
| GB/T 36073-2018 | 数据管理能力成熟度评估模型（DCMM） | 8 大能力域、5 级成熟度、29 个能力项（§4~§6） |
| GB/T 43697-2024 | 数据安全技术 数据分类分级规则 | 分级（一般/重要/核心）、分类（个人/公共/法人/衍生）（§5~§7） |
| GB/T 36344-2018 | 信息技术 数据质量评价指标 | 六大质量维度：完整/准确/一致/时效/唯一/合规（§4） |
| GB/T 42129-2022 | 信息技术 大数据 数据资产评估 | 成本法/收益法/市场法三法适用条件与计算框架（§6~§7） |

## Inputs

- **查询关键词**（string）：用户想检索的主题，如"数据分类分级""无形资产确认条件""DCMM 成熟度等级"
- **可选·标准限定**（string）：用户指定只查某个标准，如 `GB/T 43697`
- **可选·岗位上下文**（string）：当前 agent id（如 `dam-03`），用于优先返回该岗位对应章节

```yaml
query: "数据分类分级"
```

## Steps

### 步骤一：解析查询意图

1. 从用户输入中提取关键词，判断查询类型：
   - **定义类**："XX 是什么意思" → 查术语标准（GB/T 40685）
   - **要求类**："XX 有什么要求" → 查能力标准（T/MIITEC 025-2024）或管理标准（DCMM）
   - **操作类**："XX 怎么做" → 查操作类标准（GB/T 43697 分类分级、GB/T 36344 质量评价）
   - **判定类**："XX 是否合规" → 查法规与标准交叉
2. 如果有岗位上下文，优先匹配该岗位在 T/MIITEC 025-2024 中对应的标准章节（§5.X.X）

### 步骤二：检索标准库

1. 在标准索引中按关键词匹配，定位到具体标准与条款
2. 如果查询涉及多个标准（如"分类分级"同时涉及 GB/T 43697 与 GB/T 40685），全部列出
3. 提取条款原文片段（≤ 200 字），保留"应/宜/可"等关键措辞

### 步骤三：构造引用格式

1. 每条结果必须包含：`标准编号` + `条款号` + `条款标题` + `原文片段` + `上下文说明`
2. 如该条款在其他技能中被引用过，标注关联技能路径（如 `→ management/dcmm-scoring`）
3. 如查询涉及法规（非标准），建议同时调用 `foundation/regulation-lookup` 交叉验证

### 步骤四：附溯源标注

将检索到的条款以标准格式输出供引用：

```
[GB/T 43697-2024 §5.1] 数据分级应遵循合法合规、分类分级、动态调整、最小必要原则
```

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "数据资产有没有标准定义？" | 查 GB/T 40685-2021 §5 术语，返回"数据资产"的原文定义 |
| "DCMM 有几个等级？" | 查 GB/T 36073-2018 §4.2，返回 5 级成熟度定义原文 |
| "确权师的岗位能力要求在哪？" | 查 T/MIITEC 025-2024 §5.1.3，返回该岗位的知识/技能/工程实践要求 |
| "数据质量有几个维度？" | 查 GB/T 36344-2018 §4，返回六大维度定义 |
| "数据评估该用哪个标准？" | 查 GB/T 42129-2022，返回三法适用条件条款 |
| "入表会计处理有标准吗？" | 提示这是法规而非标准，转交 `foundation/regulation-lookup` 查财会〔2023〕11号 |
| "我们的 DCMM 评了几分？" | 提示这是评分场景，转交 `management/dcmm-scoring` 执行打分 |
