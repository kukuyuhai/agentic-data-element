# 数据产品交付技术方案

> 数据交易的"最后一公里"——交付方式决定了安全性、效率和成本。

---

## 一、六大交付方式对比

| 交付方式 | 安全等级 | 适用场景 | 成本 | 技术门槛 |
|---------|---------|---------|------|---------|
| **API 接口** | 中 | 数据服务/核验/实时查询 | 低-中 | 中 |
| **数据包加密传输** | 中-高 | 数据集一次性交付 | 低 | 低 |
| **隐私计算** | 高 | 多源数据融合/联合建模 | 高 | 高 |
| **数据沙箱** | 高 | 数据可用不可见/分析 | 高 | 中 |
| **可信数据空间** | 高 | 多方安全计算/联邦学习 | 高 | 高 |
| **区块链存证交付** | 中 | 全流程溯源/防篡改 | 中 | 中 |

---

## 二、各交付方式详解

### 1. API 接口交付

**原理**：需方通过调用 API 获取查询结果，不直接获取原始数据。

**优点**：
- 供方对数据保持完全控制
- 可按次数/流量计费
- 可实时监控和切断访问

**缺点**：
- 不适合大数据量批量交付
- 需方需具备开发能力

**典型场景**：征信查询、天气查询、企业工商信息核验、反欺诈评分

**技术要点**：
- HTTPS + API Key 鉴权
- IP 白名单
- QPS 限流
- 调用日志全量记录

---

### 2. 数据包加密传输

**原理**：将数据导出为加密文件（CSV/JSON/Parquet），通过安全通道传输，需方用密钥解密。

**优点**：
- 简单直接
- 适合大数据量一次性交付
- 成本低

**缺点**：
- 需方获取数据后控制力弱
- 难以防止二次分发

**典型场景**：训练数据集、行业研究报告数据、历史数据

**安全措施**：
- AES-256 加密
- 密钥通过独立通道传输
- 约定使用后删除义务（合同中）
- 可考虑数字水印嵌入

---

### 3. 隐私计算（联邦学习 / 多方安全计算）

**原理**：数据不出库，算法在数据所在地运行，仅传输计算结果。

**优点**：
- 最高安全等级
- 数据完全不出域
- 符合"原始数据不出域、数据可用不可见"

**缺点**：
- 技术复杂、成本高
- 计算效率低于直接使用
- 适用场景有限

**典型场景**：跨机构联合风控建模、医疗机构联合科研、多方数据融合分析

**常见技术**：联邦学习（FATE/PaddleFL）、多方安全计算（MPC）、可信执行环境（TEE）

---

### 4. 数据沙箱

**原理**：供方在安全隔离环境中提供数据，需方在沙箱内进行分析和建模，分析结果可导出，原始数据不可导出。

**优点**：
- 数据安全保障好
- 需方可灵活分析使用
- 比隐私计算效率高

**缺点**：
- 需要额外的基础设施
- 分析工具可能受限

**典型场景**：数据分析和建模、AI 模型训练

**主流平台**：蚂蚁摩斯、华为可信智能计算、各地数据交易所提供的沙箱服务

---

### 5. 可信数据空间（TDS）

**原理**：基于分布式架构和数据使用控制技术，实现数据跨域安全流通的生态系统。

**核心特点**：
- 数据所有权和使用权分离
- 使用策略由数据提供方制定并强制执行
- 分布式架构、去中心化

**适用场景**：产业数据空间、行业数据共享

---

### 6. 区块链存证交付

**原理**：将交易合同、交付记录、使用记录上链存证，实现全链路可追溯。

**典型应用**：贵阳大数据交易所已在交易中采用区块链存证。

---

## 三、交付方式选择决策

```
第一步：看数据敏感度
  ├── 含个人信息/重要数据 → 隐私计算 或 数据沙箱（最高安全）
  ├── 含商业秘密 → API接口（实时控制）或加密包+水印
  └── 公开数据/脱敏后数据 → 加密包传输 或 API

第二步：看使用方式
  ├── 实时查询 → API
  ├── 批量分析/建模 → 沙箱 或 隐私计算
  ├── 一次性交付 → 加密包
  └── 持续使用 → API 或 沙箱

第三步：看成本预算
  ├── 预算充足 → 隐私计算/沙箱（安全最优）
  └── 预算有限 → API/加密包 + 合同约束

第四步：看技术能力
  ├── 双方技术能力强 → 隐私计算/联邦学习
  ├── 一方技术强 → 沙箱/API
  └── 技术能力都弱 → 加密包 + 专业服务商
```
