# 数据交易典型案例

> 覆盖不同行业、不同规模、不同交易模式的真实案例（已脱敏）。

---

## 案例一：金融风控数据交易（贵阳数交所）

**供方**：某金融科技公司
**需方**：某城商行
**数据产品**：中小微企业信用评分模型（API 接口）
**交易流程**：

1. **数据产品化**：供方基于自有征信数据和公开工商数据，开发企业信用评分模型，个人敏感信息已脱敏
2. **入驻/挂牌**：在贵阳大数据交易所完成主体登记 → 提交合规评估报告 → 数据产品上架
3. **定价**：使用贵阳数据产品交易价格计算器，输入行业（金融）、场景（风控）、质量（A级）、开发成本（500 万）→ 获得参考价区间 80-120 万元/年
4. **交易**：按年授权，以 API 方式交付，每次查询计费，年合同金额 103.9 万元
5. **交付**：HTTPS API + API Key 鉴权 + 调用日志

**关键启示**：
- 价格计算器发挥了"锚定效应"，降低议价成本
- API 交付使供方可按使用量持续收费，而非一次性卖断
- 合规评估报告是顺利挂牌的关键材料

## 案例二：气象数据授权运营（上海数交所）

**供方**：某省气象局（授权运营）
**需方**：某农业科技公司
**数据产品**：县域级气象预报数据（加密数据包，T+1 更新）
**交易流程**：

1. **确权**：气象局将公共数据授权给运营公司，完成公共数据授权运营协议
2. **产品化**：将气象站原始数据加工为县域级预报产品，进行质量评估
3. **挂牌**：上海数据交易所登记 → 提交合规评估和质量评估报告 → 挂牌
4. **定价**：公共数据，按政府指导价管理 → 运营服务费
5. **交易**：按季度授权，加密数据包交付，500 万元/年
6. **交付**：加密 CSV 包 + 独立密钥传输

**关键启示**：
- 公共数据的交易链上有"授权运营"这一额外环节
- 公共数据定价受政府指导价约束，不能完全市场化
- 上海数交所"先登记后挂牌"的流程适合合规要求高的公共数据场景

## 案例三：工业企业设备数据交易（西部数据交易中心）

**供方**：某汽车制造企业
**需方**：某设备预测性维护服务商
**数据产品**：发动机运行状态数据（数据沙箱）
**交易流程**：

1. **数据源**：企业自有 IoT 设备采集的发动机运行参数（温度、振动、转速等）
2. **权属确认**：数据由企业资产产生，权属清晰，完成数据确权登记
3. **挂牌**：在西部数据交易中心汽摩专区挂牌
4. **定价**：市场法，参考同类工业数据行情，协商定价
5. **交易**：按年授权，沙箱交付，300 万元/年
6. **交付**：数据不出库，需方在沙箱内训练预测模型，仅导出模型参数

**关键启示**：
- 工业数据有明确的产业场景，在行业专区内交易效率更高
- 沙箱交付平衡了数据安全和使用灵活性
- 西部数据交易中心的"1+6"规则体系为工业数据交易提供了规范框架

## 案例四：个人数据合规交易探索（深圳数交所）

**供方**：某个人数据管理平台（数据经纪人模式）
**需方**：某 AI 训练企业
**数据产品**：经用户授权的脱敏行为数据集
**交易流程**：

1. **授权收集**：平台通过清晰告知获取大量用户的明确授权（包括数据用途、接收方）
2. **脱敏处理**：对原始行为数据进行严格匿名化处理，确保无法反向识别个人
3. **合规审查**：提交用户授权记录、脱敏方案、PIA 报告 → 通过深圳数交所合规审查
4. **定价**：收益法 + 市场法，按数据量分级定价
5. **交易**：按数据集一次性交付，加密传输，500 元/千条有效记录

**关键启示**：
- 个人数据交易的关键在于"充分授权 + 严格脱敏"
- 数据经纪人（受托人）模式是个人数据流通的可行路径
- 深圳数交所对涉及个人数据的交易有专门的审查通道

## 案例五："数据资产入表 + 交易"联动案例

**企业**：某数字营销公司
**数据资产**：用户画像标签数据库（自研，已脱敏）

**第一步：入表**
- 已完成成本归集（1200 万）：数据采集系统开发 500 万 + 清洗标注外包 300 万 + 存储 200 万 + 合规审查 200 万
- 确权：企业自主经营产生，权属清晰
- 入表：作为无形资产－数据资源，以成本法初始计量 1200 万元
- 摊销：预计受益年限 5 年，直线法每年摊销 240 万元

**第二步：交易**
- 数据产品化：将标签数据包装为 API 查询服务
- 挂牌：贵阳大数据交易所
- 定价：参考同类数字营销数据，年授权费 500 万元/家
- 签约 3 家需要方，年交易收入 1500 万元

**关键启示**：
- 入表金额（成本法 1200 万）远低于交易收入（1500 万/年），充分体现了"成本法入账、市场法交易"的差异
- 数据资产入表为企业参与数据交易提供了会计基础——资产"有了账"
- 入表和交易互补：入表为融资/估值提供依据，交易为价值实现提供通道
