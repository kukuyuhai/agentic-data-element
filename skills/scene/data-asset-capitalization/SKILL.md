# 数据资产入表技能

## 核心能力

本技能提供数据资产入表（Data Asset Capitalization）的全流程支持，以财政部《企业数据资源相关会计处理暂行规定》（财会〔2023〕11号）为核心依据。

## 工作原则

1. **以暂行规定为锚点**：所有分析和建议必须回到暂行规定和相关会计准则的条文上，不可脱离法规自由发挥
2. **交互式诊断优先**：入表工作高度依赖企业具体情况（行业、数据来源、业务模式、成本体系），通过与用户递进式问答来定位问题，而非直接抛出长篇手册
3. **成本法为入账基准**：会计入账必须按历史成本原则，评估价值不能直接入账，需反复向用户强调这一关键区别
4. **合规先行**：数据来源合法性、权属清晰是入表的前提，合规问题未解决前不应推荐入表操作

## 工作流程

根据用户所处的阶段，选择对应的流程路径：

### 阶段一：资产识别与盘点

当用户问"哪些数据能入表""我的数据能不能入表"时：

1. 引导用户回答5个确认条件：
   - 合法性：数据来源是否合法？
   - 权属性：企业是否合法拥有或控制？
   - 可辨认性：能否单独识别和管理？
   - 经济利益：能否带来未来经济利益？
   - 成本可计量：成本能否可靠计量？

2. 引导用户按来源分类盘点数据资产：
   - 外购数据（含API、数据服务订阅）
   - 自行加工形成的数据（内部产生、开发）
   - 外购+加工混合形成的数据

3. 输出：数据资产候选清单，注明每项的初步判断（可入表/待确认/不可入表）

可加载 `references/policies.md` 获取5个确认条件的详细条文。

### 阶段二：确权与合规审查

当用户问"入表前需要做什么合规准备"时：

1. 逐项引导用户确认：
   - "三权分置"中企业拥有哪些权利？
   - 数据采集的授权文件是否齐全？
   - 是否涉及个人信息/敏感信息？
   - 是否存在权属争议？

2. 建议运行合规验证脚本：
   ```bash
   python scripts/validate_compliance.py
   ```
   或通过JSON输入：
   ```bash
   python scripts/validate_compliance.py --json input.json --output report.md
   ```

3. 可加载 `references/compliance-checklist.md` 获取完整审核清单。

### 阶段三：成本归集与计量

当用户问"成本怎么算""哪些费用可以资本化"时：

1. 区分研究阶段与开发阶段 — 这是资本化最关键的门槛：
   - 研究阶段 → 全部费用化（探索性、无明确应用目标）
   - 开发阶段 → 满足5条件后资本化（有明确目标、技术可行）

2. 引导用户逐项梳理成本：
   - 直接成本：数据采购、清洗、标注、合规审查
   - 开发成本：系统建设、算法研发（开发阶段部分）
   - 基础设施：服务器、存储（按实际使用量分摊）
   - 维持费用：日常运维（费用化）

3. 加载 `references/accounting-treatment.md` 获取详细的资本化/费用化判定和会计分录示例。

4. 如用户需要模板，加载 `assets/入表工作底稿模板.xlsx` 并根据用户信息填充。

### 阶段四：会计处理与披露

当用户问"入表后怎么做账""摊销多少年""怎么披露"时：

1. 确认资产分类：
   - 内部使用或对外提供服务 → 无形资产-数据资源
   - 日常持有、最终出售 → 存货-数据资源

2. 初始计量：按实际发生的、可可靠计量的成本入账

3. 后续计量：
   - 无形资产：按使用寿命摊销（常见3-10年），每年减值测试
   - 存货：不摊销，期末按成本与可变现净值孰低

4. 列报与披露：资产负债表中"数据资源"子项目，附注强制/自愿披露事项

加载 `references/accounting-treatment.md` 获取完整的会计分录、列报格式和披露模板。

## 估值与定价（辅助模块）

当用户问"数据资产值多少钱""怎么估值"时：

加载 `references/valuation-methods.md`，按以下优先级建议方法：
1. 成本法（重置成本 - 贬值）→ 最可靠，入账首选
2. 收益法（预期收益折现）→ 适用于有明确商业化路径的数据产品
3. 市场法（可比交易调整）→ 适用于有活跃交易市场的场景

**关键提醒**：评估结果不能直接作为入账金额，入账金额仍以实际成本为准。

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "什么是数据资产入表？" | 简要解释概念，引用暂行规定核心条文，然后问企业具体情况 |
| "我的数据能入表吗？" | 进入阶段一流程，逐个确认5个条件 |
| "入表需要什么条件？" | 列举5个确认条件，逐一解释，给出判定建议 |
| "入表流程是什么？" | 展示四阶段全貌，问用户当前处于哪个阶段，聚焦指导 |
| "数据资产怎么估值？" | 加载valuation-methods.md，强调成本法为入账基准 |
| "摊销多少年合适？" | 询问数据时效性和业务周期，给出3-10年区间建议 |
| "帮我审核入表方案" | 加载compliance-checklist.md，逐项对照输出审核结论 |
| "生成一份工作底稿" | 加载assets模板并根据用户信息填充，或提供空白模板 |
| "XX行业的案例有吗？" | 加载case-studies.md，匹配最接近案例 |
| "帮我跑一下合规检查" | 引导使用validate_compliance.py脚本 |
