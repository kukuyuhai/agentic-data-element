# 数据资产评估方法

> 基于《数据资产评估指导意见》（中国资产评估协会，2023年9月）

## 一、三大方法对比

| 维度 | 成本法 | 收益法 | 市场法 |
|------|--------|--------|--------|
| **原理** | 以形成/取得数据资产的合理成本为基础 | 以数据资产未来预期收益的现值为基础 | 以同类数据资产的市场交易价格为基础 |
| **适用场景** | 自用型数据、成本记录完整 | 商业化数据产品、可预测收益 | 有活跃交易市场、有可比案例 |
| **数据要求** | 成本明细账、工时记录 | 历史收入/用户数据、增长率预测 | 可比交易案例、市场报价 |
| **优点** | 客观、可验证、不易操纵 | 反映真实经济价值、考虑未来 | 反映市场定价、操作简单 |
| **缺点** | 成本≠价值，可能低估高价值资产 | 预测主观性强、不确定性大 | 市场不成熟、可比案例少 |
| **入表首选** | ✅ 入表时按成本法确认 | ❌ 仅用于评估，不入账 | ❌ 仅用于评估，不入账 |

> **重要**: 会计入账必须按照历史成本原则，即无论评估价值多高，入表金额以实际发生的、可可靠计量的成本为准。收益法和市场法的评估结果仅用于辅助决策、交易定价、减值测试，不能直接作为入账依据。

## 二、成本法详解

### 基本公式

```
数据资产评估值 = 重置成本 - 功能性贬值 - 经济性贬值

其中：
  重置成本 = 重新形成相同/类似功能的数据资产所需的合理成本
  功能性贬值 = 因技术落后、效率降低导致的价值减损
  经济性贬值 = 因外部因素（法规变化、市场需求下降）导致的价值减损
```

### 成本构成分析

**1. 直接成本**:
- 数据采集成本（API调用、传感器、人工采集）
- 数据存储成本（服务器/云存储、数据备份）
- 数据处理成本（清洗、标注、去重、脱敏）
- 专用软件/工具采购或开发摊销
- 数据质量检验成本
- 数据合规与安全审查成本

**2. 间接成本**:
- 项目管理人员薪酬
- 数据基础设施共享部分分摊
- 知识产权的法律费用
- 数据战略规划费用

**3. 机会成本**（评估时考虑，不入账）:
- 数据采集/开发期间的资本占用成本

### 功能性贬值判定因素

- 数据时效性衰减（如实时数据的即时贬值）
- 数据格式/标准过时
- 数据处理技术迭代
- 数据覆盖维度不足

### 经济性贬值判定因素

- 法律法规限制数据使用范围
- 行业需求变化导致数据价值下降
- 替代数据源出现
- 数据交易价格整体下跌

## 三、收益法详解

### 基本公式

```
数据资产评估值 = Σ (第t年预期收益 / (1 + 折现率)^t)
                + 永续期终值（如有）

其中：
  预期收益 = 增量收入 - 增量成本
  折现率 = 无风险利率 + 风险溢价
```

### 收益的三种来源识别

**1. 收入增量**: 数据产品/服务直接销售收入、数据驱动的精准营销增收
**2. 成本节约**: 通过数据优化运营降低的成本、自动化替代人工节省的费用
**3. 风险规避**: 因数据驱动的风控能力避免的损失

### 折现率确定

| 数据资产类型 | 参考折现率区间 | 风险因素 |
|-------------|:-----------:|---------|
| 高度垄断/独家数据 | 8%-10% | 数据稀缺性强，替代风险低 |
| 行业领先数据 | 10%-15% | 一定竞争优势，存在部分替代 |
| 通用/可替代数据 | 15%-20% | 竞争激烈，时效性风险高 |
| 个人行为/实时数据 | 20%-25% | 合规风险高，时效性极强 |

## 四、市场法详解

### 基本公式

```
数据资产评估值 = 可比交易案例价格 × 调整系数

调整系数考虑因素：
  - 数据质量差异（完整性、准确性、时效性）
  - 数据规模差异（数据量、覆盖范围）
  - 使用权限差异（排他性、使用次数、地域限制）
  - 数据格式/交付方式差异
  - 交易时点差异（市场变化）
```

### 可比交易信息来源

- 数据交易所场内交易记录（各大交易所公示数据）
- 数据服务商公开报价
- 行业报告中披露的交易案例
- 企业公开披露的数据资产交易信息

### 市场法常见调整系数

| 差异维度 | 调整方向 | 调整幅度参考 |
|---------|---------|:----------:|
| 排他使用权 vs 普通使用权 | 上调 | +30%~50% |
| 近实时数据 vs 月度更新 | 上调 | +15%~25% |
| 地域局限性 vs 全国覆盖 | 下调 | -10%~20% |
| 原始数据 vs 精加工数据 | 下调 | -20%~40% |

## 五、评估方法选择建议

```
决策流程：
1. 是否有活跃交易市场和足够可比案例？
   是 → 可尝试市场法，作为评估参考
   否 → 进入第2步

2. 数据资产是否有明确商业化路径，能否合理预测未来收益？
   是 → 可尝试收益法，作为评估参考
   否 → 进入第3步

3. 是否能清晰归集形成数据资产的各项成本？
   是 → 使用成本法进行评估和入账
   否 → 不具备入表条件，需先完善成本核算体系
```

> **入表实务建议**: 对绝大多数企业而言，成本法是最可行、最合规的入表基准。先按成本法完成入表，待数据资产运营成熟后，可辅以收益法/市场法进行后续评估和减值测试。
