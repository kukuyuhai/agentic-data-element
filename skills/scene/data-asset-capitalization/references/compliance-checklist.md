# 数据资产入表合规审核清单

> 每个入表项目开工前和完工后各审核一次。开工前审核侧重"能不能做"，完工后审核侧重"做得对不对"。

## 一、数据来源合法性审查（前置条件）

| # | 审核项 | 通过标准 | 状态 |
|---|--------|---------|:----:|
| 1.1 | 数据采集是否获得合法授权？ | 有用户知情同意记录或合同授权文件 | ☐ |
| 1.2 | 数据来源是否符合"最小必要"原则？ | 仅采集实现目的所必需的数据 | ☐ |
| 1.3 | 是否涉及个人敏感信息？ | 如涉及，已单独取得用户明示同意 | ☐ |
| 1.4 | 公共数据获取是否合规？ | 通过授权运营机构或公开渠道合法获取 | ☐ |
| 1.5 | 爬取/抓取数据是否合规？ | 已评估robots协议、用户协议和反爬政策 | ☐ |
| 1.6 | 是否涉及国家秘密或核心数据？ | 不涉及（涉及则不可入表） | ☐ |
| 1.7 | 跨境数据传输是否合规？ | 已完成安全评估/标准合同备案/认证 | ☐ |

## 二、数据权属审查

| # | 审核项 | 通过标准 | 状态 |
|---|--------|---------|:----:|
| 2.1 | 企业对该数据资源是否拥有合法权利？ | 持有权/使用权/经营权至少拥有一项 | ☐ |
| 2.2 | 是否与他方存在权属争议？ | 无潜在权属纠纷 | ☐ |
| 2.3 | 合作开发数据的权属约定是否明确？ | 有书面合同明确各方权属和利益分配 | ☐ |
| 2.4 | 数据授权是否有期限限制？ | 授权期限已知，不影响资产确认 | ☐ |
| 2.5 | 是否存在数据质押/担保情况？ | 如有，已在资产评估中考虑 | ☐ |

## 三、资产确认条件审查

| # | 审核项 | 通过标准 | 状态 |
|---|--------|---------|:----:|
| 3.1 | 数据资源能否为企业带来未来经济利益？ | 有明确的自用增效或对外交易路径 | ☐ |
| 3.2 | 成本能否可靠计量？ | 有完整的成本记录系统或合理分摊依据 | ☐ |
| 3.3 | 数据资源是否可单独识别和管理？ | 可与企业其他资产明确区分 | ☐ |
| 3.4 | 数据是否已投入使用或具备使用条件？ | 已完成必要的加工处理，可正常使用 | ☐ |
| 3.5 | 是否排除了"场景化依赖"？ | 数据不依赖单一系统/项目才产生价值 | ☐ |

## 四、成本核算审查

| # | 审核项 | 通过标准 | 状态 |
|---|--------|---------|:----:|
| 4.1 | 资本化与费用化划分是否合理？ | 研究阶段支出已费用化，开发阶段满足条件后资本化 | ☐ |
| 4.2 | 成本归集是否完整？ | 涵盖采集、清洗、标注、存储等全链路合理支出 | ☐ |
| 4.3 | 共同成本的分配方法是否合理？ | 有明确的分配标准（工时、使用量等） | ☐ |
| 4.4 | 是否有原始凭证支撑？ | 采购合同、发票、工时记录等原始凭证齐全 | ☐ |
| 4.5 | 成本是否存在人为虚增？ | 不存在将无关费用计入数据资产成本 | ☐ |

## 五、会计处理审查

| # | 审核项 | 通过标准 | 状态 |
|---|--------|---------|:----:|
| 5.1 | 资产分类是否正确？ | 内部使用→无形资产，最终出售→存货 | ☐ |
| 5.2 | 摊销年限确定是否合理？ | 有明确的判断依据（数据时效性、业务周期等） | ☐ |
| 5.3 | 摊销方法是否合理？ | 通常为直线法，有特殊情况的做了合理调整 | ☐ |
| 5.4 | 是否进行了减值测试？ | 至少每年末一次，有减值迹象时立即测试 | ☐ |
| 5.5 | 列报是否符合规定？ | 资产负债表分设"数据资源"子项目 | ☐ |
| 5.6 | 附注披露是否完整？ | 含类别、增减变动、摊销、减值等强制披露项 | ☐ |

## 六、高风险预警信号

以下任一情况出现时，**应暂停入表，先解决风险**：

- 🔴 数据来源明显违法违规（如非法爬取、未经授权的个人信息采集）
- 🔴 存在重大权属争议或潜在诉讼
- 🔴 数据无法为企业带来可预见的经济利益
- 🔴 成本记录严重缺失，无法追溯核实
- 🔴 涉及国家秘密或核心数据但未获得相应审批
- 🔴 采用的估值方法严重偏离行业惯例（如无依据地使用极高倍数）
