# 📜 contract-clause-library

## When to Use

当出现以下场景时调用本技能：

- **起草数据类合同**时，需要标准条款模板
- **审核合同**时，需要对照关键条款清单检查遗漏
- **三权分置交易**时，需要按权利类型选择对应条款
- **合同谈判**时，需要了解各条款的常见谈判点和替代方案

本技能是数据类合同的关键条款库，基于"数据二十条"三权分置框架组织条款。

## 工作原则

1. **三权对应**：条款按持有权/加工使用权/产品经营权三种权利类型分别组织
2. **交易方式适配**：同一权利的转让/许可/入股使用不同条款模板
3. **风险条款全覆盖**：每类合同必须包含标的界定、权利范围、对价、陈述保证、违约救济、争议解决六大核心条款
4. **法律审慎**：本技能提供条款模板和参考，不替代律师审核，高风险合同建议法律顾问终审

## 三权分置与合同类型

### 权利类型

| 权利类型 | 权利内容 | 典型交易场景 |
|---------|---------|------------|
| 数据资源持有权 | 控制和管理数据资源的权利 | 数据存储托管、数据资产入表确权 |
| 数据加工使用权 | 加工、分析、使用数据的权利 | 数据分析服务、数据建模合作 |
| 数据产品经营权 | 将数据加工为产品并经营的权利 | 数据产品销售、数据 API 授权 |

### 交易方式

| 方式 | 特征 | 合同类型 |
|------|------|---------|
| 转让 | 权利全部转移给受让方 | 数据权利转让合同 |
| 许可 | 权利有限度地授予被许可方 | 数据许可使用合同 |
| 入股 | 权利作价出资入股 | 数据权利作价入股协议 |

## 六大核心条款

### 1. 标的界定条款

**必须明确**：
- 数据资产名称、标识符、来源
- 数据范围（字段/记录/时间范围）
- 数据量、更新频率、数据格式
- 三权中转让/许可的是哪一权
- 是否包含衍生数据权

```
【示例·标的界定】
甲方将其合法拥有的"XX用户行为数据集"（资产编号：DA-XXXX，约500万条记录，
更新频率：日更）的数据加工使用权转让给乙方，乙方有权对该数据集进行加工、
分析、建模，但不得再许可第三方。衍生数据产品经营权归乙方所有。
```

### 2. 权利范围条款

**必须明确**：
- 使用目的限制（是否限定特定用途）
- 使用地域限制（境内/全球）
- 使用期限（永久/固定期限）
- 再许可权（是否允许转授权）
- 衍生数据归属（谁拥有加工后产生的新数据）

### 3. 对价条款

**必须明确**：
- 价格/费用金额
- 支付方式（一次性/分期/按量）
- 支付时间节点
- 税费承担
- 价格调整机制（如适用）

### 4. 陈述与保证条款

**必须保证**：
- 甲方对数据拥有合法权利（持有权/加工使用权/经营权）
- 数据来源合法，已取得必要授权
- 数据不侵犯第三方权益
- 数据不含国家秘密
- 个人信息已合规处理（脱敏/授权）

### 5. 违约救济条款

**必须明确**：
- 违约情形定义
- 违约金计算方式
- 合同解除条件
- 损失赔偿范围
- 数据返还/销毁义务

### 6. 争议解决条款

**必须明确**：
- 争议解决方式（诉讼/仲裁）
- 管辖法院/仲裁机构
- 适用法律
- 争议期间的合同效力

## Inputs

- **交易类型**（enum）：`transfer`（转让）/ `license`（许可）/ `equity`（入股）
- **权利类型**（enum）：`holding`（持有权）/ `processing`（加工使用权）/ `operation`（经营权）
- **交易信息**（object）：标的、双方、金额、期限
- **可选·已有合同草稿**（string）：用户已起草的合同文本（用于审核模式）

## Steps

### 步骤一：确定合同类型

1. 根据交易方式（转让/许可/入股）和权利类型（持有/加工/经营）确定合同类型
2. 选择对应的条款模板组合

### 步骤二：填充标的条款

1. 从 `asset-catalog-crud` 获取数据资产信息
2. 明确数据范围、量级、格式、更新频率
3. 明确转让/许可的具体权利

### 步骤三：填充权利范围条款

1. 与用户确认使用目的限制
2. 确认地域和期限限制
3. 确认再许可权和衍生数据归属

### 步骤四：填充对价条款

1. 确认价格和支付方式
2. 如涉及入股，需配合 `deal-structuring-3paths` 确定估值

### 步骤五：填充陈述保证与违约条款

1. 列出甲方必须保证的事项
2. 定义违约情形和救济方式
3. 明确数据返还/销毁义务

### 步骤六：审核完整性

1. 检查六大核心条款是否齐全
2. 如为审核模式（已有合同草稿），逐条对照检查遗漏
3. 输出风险警告和建议增加的条款

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "起草一份数据许可合同" | 确定权利类型和交易方式，填充六大条款 |
| "审核这份合同" | 逐条对照检查，输出遗漏和风险 |
| "加工使用权怎么写？" | 选择加工使用权许可条款模板 |
| "转让和许可有什么区别？" | 解释权利转移 vs 权利授予的区别 |
| "衍生数据归谁？" | 说明需在权利范围条款中明确约定 |
| "违约金怎么定？" | 建议按合同金额的 20-30% 或实际损失约定 |
