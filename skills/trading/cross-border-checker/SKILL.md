# 🌐 cross-border-checker

## When to Use

当出现以下场景时调用本技能：

- **数据跨境传输**前，判断是否触发安全评估申报义务
- **境外IP访问**境内数据，需要判断是否构成"数据出境"
- **跨国企业**内部数据共享，需要判断适用哪条出境路径
- **数据产品对外提供**给境外买家，需要完成出境合规
- **个人信息处理者**需要判断是否触发个人信息出境标准合同备案

本技能是高风险技能，涉及数据出境合规判定，结论影响企业法律责任。

## 工作原则

1. **保守判定**：出境合规判断取保守立场——如不确定是否触发，默认按触发处理
2. **三条路径**：数据出境有三条合规路径——安全评估、标准合同、认证，需判断适用哪条
3. **数量阈值敏感**：个人信息数量是触发出境申报的关键阈值，必须精确统计
4. **法规时效**：出境法规更新频繁，引用时需标注法规最新版本和施行日期

## 数据出境判定框架

### 第一步：是否构成"数据出境"？

| 情形 | 是否构成出境 | 说明 |
|------|------------|------|
| 数据物理传输到境外服务器 | 是 | 直接出境 |
| 境外IP访问/查询境内数据 | 是 | 间接出境 |
| 境外人员远程操作境内系统 | 是 | 间接出境 |
| 数据在境内但被境外实体控制 | 视情况 | 需判断是否构成"控制" |
| 数据已匿名化后传输 | 否 | 匿名化数据不适用出境规定 |
| 数据在境内境内传输 | 否 | 非出境 |

### 第二步：是否触发安全评估申报？

依据《数据出境安全评估办法》第四条，以下情形**应当申报**安全评估：

| 触发条件 | 阈值 | 法规依据 |
|---------|------|---------|
| 关键信息基础设施运营者 | 无门槛 | 第四条第一款 |
| 处理100万人以上个人信息 | ≥100万人 | 第四条第二款 |
| 累计向境外提供10万人以上个人信息 | ≥10万人 | 第四条第三款 |
| 累计向境外提供1万人以上敏感个人信息 | ≥1万人 | 第四条第三款 |
| 重要数据出境 | 无门槛 | 第四条第四款 |
| 国家网信部门规定的其他情形 | — | 第四条第五款 |

### 第三步：适用哪条合规路径？

| 路径 | 适用条件 | 流程 | 周期 |
|------|---------|------|------|
| 安全评估申报 | 触发第四条任一条件 | 向省级网信部门申报 → 国家网信部门评估 | 7个工作日+60工作日 |
| 标准合同备案 | 不触发安全评估，处理个人信息出境 | 签署标准合同 → 向省级网信部门备案 | 签署后10工作日内备案 |
| 个人信息保护认证 | 不触发安全评估，取得认证 | 通过认证机构认证 | 认证有效期3年 |

## Inputs

- **数据出境场景**（object）：
  - `data_type`：数据类型（个人信息/重要数据/一般数据/匿名化数据）
  - `data_volume`：数据量（记录条数）
  - `contains_sensitive_pii`：是否含敏感个人信息
  - `sensitive_pii_count`：敏感个人信息条数
  - `recipient_location`：接收方所在国家/地区
  - `recipient_identity`：接收方身份（关联公司/第三方）
  - `transfer_method`：传输方式（物理传输/远程访问/API调用）
- **数据处理者信息**（object）：
  - `is_ciio`：是否为关键信息基础设施运营者
  - `total_pii_processed`：处理的个人信息总量

## Steps

### 步骤一：判定是否构成数据出境

1. 分析数据流向：数据是否物理或逻辑地传输到境外
2. 判断接收方：接收方是否为境外实体或境外人员
3. 如不构成出境，输出"无需出境合规"并结束

### 步骤二：统计数据规模

1. 统计个人信息总量（是否≥100万）
2. 统计出境个人信息量（是否≥10万）
3. 统计敏感个人信息量（是否≥1万）
4. 判断是否包含重要数据

### 步骤三：检查触发条件

1. 逐一对照《数据出境安全评估办法》第四条的五项触发条件
2. 如有任何一项触发，标记为"需安全评估申报"
3. 如全部不触发，标记为"可选标准合同或认证"

### 步骤四：确定合规路径

1. 如触发安全评估 → 路径为"安全评估申报"
2. 如不触发但涉及个人信息出境 → 路径为"标准合同备案"或"认证"
3. 如为匿名化数据 → 无需出境合规
4. 如为重要数据 → 必须安全评估申报

### 步骤五：生成行动方案

1. 按合规路径生成步骤化行动方案
2. 标注高风险警告（未通过评估前不得出境）
3. 建议配合使用的技能（如需脱敏先调用 `de-identification-toolkit`）
4. 附标准溯源标注

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "数据能出境吗？" | 执行出境判定，输出是否触发申报 |
| "要申报吗？" | 检查触发条件，如触发则需申报 |
| "走标准合同行不行？" | 如未触发安全评估且涉及个人信息出境，可走标准合同 |
| "100万条数据要报吗？" | 如为个人信息≥100万，触发安全评估申报 |
| "匿名化后还用报吗？" | 匿名化数据不适用出境规定，无需申报 |
| "境外员工访问算出境吗？" | 境外IP访问境内数据构成间接出境 |
