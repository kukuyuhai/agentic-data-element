# 🪪 kyc-kyb-runner

## When to Use

当出现以下场景时调用本技能：

- **数据交易入场前**，对买方和卖方进行身份核验
- **数据资产登记**时，对申请主体进行资质审查
- **合同签署前**，确认对方主体合法性和签约资格
- **持续监控**中，定期复核交易方资质是否发生变化

本技能执行 KYC（Know Your Customer，个人客户身份识别）和 KYB（Know Your Business，企业客户身份识别）检查。

## 工作原则

1. **双向审查**：买方和卖方都需要通过 KYC/KYB 检查，不可只查一方
2. **证据留存**：所有检查结果必须留存证据（截图/文件），不可仅口头确认
3. **风险分级**：根据检查结果将交易方分为低/中/高风险，高风险需额外审查
4. **持续监控**：KYC/KYB 不是一次性的，建议定期复核（如每半年）

## KYC 检查清单（个人客户）

| 检查项 | 检查内容 | 通过标准 | 风险级别 |
|--------|---------|---------|---------|
| 身份验证 | 姓名 + 身份证号一致性核验 | 与公安系统一致 | 阻断 |
| 证件有效性 | 身份证是否在有效期内 | 有效 | 阻断 |
| 手机验证 | 手机号实名认证 | 与运营商记录一致 | 阻断 |
| 制裁名单 | 是否在国内外制裁名单中 | 不在名单中 | 阻断 |
| 信用记录 | 个人征信是否有严重不良 | 无严重不良 | 严重 |
| 涉诉记录 | 是否有未了结诉讼 | 无重大涉诉 | 一般 |

## KYB 检查清单（企业客户）

| 检查项 | 检查内容 | 通过标准 | 风险级别 |
|--------|---------|---------|---------|
| 工商登记 | 营业执照核验（统一社会信用代码） | 存续/在营 | 阻断 |
| 注册信息 | 注册资本、成立日期、经营范围 | 与声称一致 | 阻断 |
| 法定代表人 | 法人身份核验 | 与工商一致 | 阻断 |
| 受益人识别 | 实际控制人/最终受益人（UBO）穿透 | 已识别 | 阻断 |
| 制裁名单 | 企业及实控人是否在制裁名单 | 不在名单中 | 阻断 |
| 经营异常 | 是否被列入经营异常名录 | 未被列入 | 严重 |
| 严重违法 | 是否有严重违法失信记录 | 无 | 严重 |
| 股权结构 | 股权穿透至自然人 | 已穿透 | 一般 |
| 涉诉记录 | 重大诉讼/仲裁/执行案件 | 无重大涉诉 | 一般 |
| 资质许可 | 数据处理相关资质（如需） | 具备 | 一般 |

## 风险分级标准

| 风险等级 | 判定条件 | 处理方式 |
|---------|---------|---------|
| 低风险 | 全部检查通过，无异常 | 正常推进交易 |
| 中风险 | 有一般级检查项未通过 | 可推进交易，需加强监控 |
| 高风险 | 有严重级检查项未通过 | 需人工审批后推进 |
| 阻断 | 有阻断级检查项未通过 | 不可推进交易 |

## Inputs

- **检查对象**（enum）：`individual`（KYC）/ `enterprise`（KYB）
- **主体信息**（object）：
  - 个人：姓名、身份证号、手机号
  - 企业：企业名称、统一社会信用代码、法定代表人
- **检查深度**（enum）：`basic`（基础核验）/ `enhanced`（增强核验，含 UBO 穿透）
- **可选·已有材料**（array）：用户已提供的证照文件

## Steps

### 步骤一：确定检查类型

1. 根据交易方类型确定执行 KYC（个人）还是 KYB（企业）
2. 如为企业，确认是否需要增强核验（UBO 穿透）

### 步骤二：基础信息核验

**KYC（个人）**：
1. 核验姓名与身份证号一致性
2. 检查身份证有效期
3. 验证手机号实名

**KYB（企业）**：
1. 核验营业执照（统一社会信用代码）
2. 检查企业状态（存续/注销/吊销）
3. 核验法定代表人信息
4. 确认经营范围包含数据处理相关

### 步骤三：风险筛查

1. 制裁名单核查：OFAC（美国）、UN（联合国）、EU（欧盟）、中国制裁名单
2. 经营异常核查：经营异常名录、严重违法失信企业名单
3. 涉诉记录核查：是否有重大诉讼/仲裁/执行案件

### 步骤四：受益人识别（KYB 增强核验）

1. 股权穿透：追溯至最终受益人（UBO）
2. 识别持股 ≥25% 的自然人
3. 识别实际控制人

### 步骤五：风险分级与输出

1. 根据检查结果判定风险等级
2. 如有阻断项，标记为"不可推进"
3. 如有严重项，标记为"需人工审批"
4. 输出检查报告并建议下次复核日期

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "查一下这个企业" | 执行 KYB 基础核验 |
| "这个人能交易吗？" | 执行 KYC 检查 |
| "实控人是谁？" | 执行 KYB 增强核验，股权穿透 |
| "在制裁名单上吗？" | 查多边制裁名单 |
| "经营异常了吗？" | 查经营异常名录 |
| "什么时候再查？" | 建议半年复核一次 |
