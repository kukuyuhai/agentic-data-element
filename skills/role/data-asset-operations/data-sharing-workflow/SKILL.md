# 🤝 data-sharing-workflow

内部数据共享工作流覆盖数据共享全生命周期：申请→审批→发布→日志→审计，内置分级授权模板与脱敏策略模板，确保数据共享过程合规、可控、可追溯。

## When to Use

- **新消费方接入**：新部门/新团队申请使用已有数据资产（§8.1）
- **新建共享渠道**：新建API、数据推送、文件交换等共享渠道（§8.2）
- **共享范围调整**：现有消费方申请扩大/缩小数据范围或期限
- **合规检查**：定期或不定期的数据共享合规审查
- **到期续约**：共享授权到期前的续约评估
- **安全事件响应**：发现数据泄露风险时，紧急审查共享渠道

## 工作原则

1. **最小必要**：共享数据范围遵循最小必要原则，仅提供业务所需的最小数据集（数据安全法 §13）
2. **分级授权**：按数据安全级别匹配审批层级，高敏感数据需高级别审批
3. **期限明确**：每项共享必须设定有效期，到期自动回收权限（§8.3）
4. **全程留痕**：申请、审批、访问、操作全流程日志记录，不少于3年留存
5. **脱敏优先**：敏感数据共享前必须脱敏，脱敏策略与数据级别匹配（GB/T 43697-2024）
6. **定期审计**：每月审计共享日志，每季度复审共享授权合理性

## 共享工作流

### 1. 申请阶段

消费方发起数据共享申请，填写完整用途说明。

| 申请字段 | 类型 | 说明 | 示例 |
|---------|------|------|------|
| 消费方信息 | string | 申请部门/团队/负责人 | 市场部-赵六 |
| 数据范围 | array | 申请的资产ID列表 | AST-2026-00001 |
| 用途说明 | string | 详细业务用途 | 用户画像分析 |
| 使用期限 | date | 申请截止日期 | 2026-12-31 |
| 访问方式 | enum | API/数据库直连/文件 | API |
| 安全承诺 | boolean | 签署安全使用承诺书 | true |

### 2. 审批阶段

按数据安全级别匹配审批流程，多角色协同审核。

| 审批角色 | 职责 | 审核重点 |
|---------|------|---------|
| 数据所有者 | 确认数据可否共享 | 用途合理性、范围适当性 |
| 安全审核员 | 安全合规审查 | 脱敏策略、权限配置 |
| 合规检查员 | 法规合规检查 | 数据安全法合规性 |
| 分级审批人 | 按级别匹配审批权限 | 见分级授权模板 |

### 3. 发布阶段

审批通过后执行技术配置，正式开放共享。

| 发布步骤 | 操作内容 | 责任方 |
|---------|---------|--------|
| 脱敏处理 | 按脱敏策略对敏感字段处理 | 数据工程师 |
| 权限配置 | 配置消费方访问权限 | 平台运维 |
| 接口开通 | 开通API/数据库账号 | 平台运维 |
| 通知告知 | 通知消费方共享已就绪 | 数据所有者 |
| 文档归档 | 归档审批记录与配置文档 | 数据资产管理员 |

### 4. 日志阶段

全量记录共享过程中的访问与操作行为。

| 日志类型 | 记录内容 | 留存期 |
|---------|---------|--------|
| 访问日志 | 谁/何时/访问了什么数据 | ≥3年 |
| 操作日志 | 查询/导出/删除等操作 | ≥3年 |
| 数据日志 | 数据量/返回行数/耗时 | ≥1年 |
| 异常日志 | 越权访问/高频访问/异常时段 | ≥3年 |

### 5. 审计阶段

定期审计与异常告警，到期自动回收。

| 审计类型 | 频率 | 审计内容 |
|---------|------|---------|
| 定期审计 | 月度 | 访问日志审查、权限一致性检查 |
| 合规审计 | 季度 | 用途匹配性、脱敏有效性 |
| 异常告警 | 实时 | 越权访问、批量导出、异常时段 |
| 到期回收 | 自动 | 授权到期自动关闭权限 |

## 分级授权模板

按数据安全级别匹配审批层级与处理时效。

| 安全级别 | 审批层级 | 审批人 | 处理时效 | 附加要求 |
|---------|---------|--------|---------|---------|
| 公开 | 自动审批 | 系统 | 即时 | 无 |
| 内部 | 主管审批 | 数据所有者 | ≤2工作日 | 用途说明 |
| 敏感 | 总监审批 | 数据所有者+安全审核 | ≤5工作日 | 脱敏处理+安全承诺 |
| 核心 | VP审批 | 数据所有者+安全+合规+VP | ≤10工作日 | 脱敏+审批会+专项审计 |

## 脱敏策略模板

按数据级别匹配脱敏方式，调用 `` `role/data-asset-operations/de-identification-toolkit` `` 执行。

| 数据级别 | 字段类型 | 脱敏方式 | 示例 |
|---------|---------|---------|------|
| 公开 | — | 不脱敏 | 原文展示 |
| 内部 | 一般字段 | 部分遮蔽 | 手机号 138****5678 |
| 敏感 | PII字段 | 掩码/哈希 | 身份证号 SHA-256 |
| 敏感 | 统计字段 | 聚合/泛化 | 精确值→区间值 |
| 核心 | 全部字段 | 加密+脱敏 | AES-256 + 哈希 |
| 核心 | 标识字段 | 假名化 | 真实ID→假名ID映射 |

## Inputs

- `applicant_info`（object，必填）：消费方信息（部门/负责人/联系方式）
- `requested_assets`（array，必填）：申请资产ID列表
- `purpose`（string，必填）：详细用途说明
- `duration`（date，必填）：使用期限
- `access_method`（enum，必填）：访问方式
- `security_commitment`（boolean，必填）：是否签署安全承诺

**示例输入：**

```json
{
  "applicant_info": { "department": "市场部", "owner": "赵六" },
  "requested_assets": ["AST-2026-00001", "AST-2026-00002"],
  "purpose": "用户画像分析用于精准营销",
  "duration": "2026-12-31",
  "access_method": "API",
  "security_commitment": true
}
```

## Outputs

- `workflow_id`（string）：共享工作流ID
- `approval_status`（enum）：审批状态（待审批/已批准/已拒绝）
- `sharing_config`（object）：共享配置（脱敏策略/权限/接口）
- `logging_config`（object）：日志配置（日志类型/留存期）
- `audit_schedule`（object）：审计计划（频率/审计项）

## Steps

1. **申请受理**：消费方提交申请，系统校验申请完整性与资产有效性（§8.1.1）
2. **分类定级**：调用 `` `role/data-asset-operations/data-classify` `` 确认数据级别，匹配审批层级（§8.1.2）
3. **PII扫描**：调用 `` `role/data-asset-operations/pii-scan` `` 扫描申请资产，确定PII字段（§8.1.3）
4. **审批流转**：按分级授权模板流转审批，各级审批人审核并签署意见（数据安全法 §13）
5. **脱敏配置**：审批通过后，按脱敏策略模板配置脱敏规则，调用 `` `role/data-asset-operations/de-identification-toolkit` `` 执行（GB/T 43697-2024）
6. **权限开通**：配置访问权限、开通接口、设置有效期，通知消费方
7. **日志启动**：启动全量日志记录，配置异常告警规则
8. **审计计划**：配置定期审计计划与到期回收规则，进入持续监控

### 工作流流转图

```
申请 → 分类定级 → PII扫描 → 审批流转 → 脱敏配置 → 权限开通 → 日志启动 → 审计监控
                                                              │
                                                              └─ 到期自动回收
```

## 常见问法与处理策略

| 常见问法 | 处理策略 | 标准溯源 |
|---------|---------|---------|
| "新部门要用我们的数据怎么走流程？" | 按"申请→审批→发布→日志→审计"五步流程执行 | §8.1 |
| "敏感数据共享必须脱敏吗？" | 是，按脱敏策略模板执行，调用 `` `role/data-asset-operations/de-identification-toolkit` `` | GB/T 43697-2024 |
| "共享授权到期了怎么办？" | 系统自动回收权限，如需续约需重新发起申请 | §8.3 |
| "发现异常访问怎么处理？" | 触发异常告警，紧急冻结权限，启动安全调查 | 数据安全法 §29 |
| "核心数据可以共享吗？" | 可以但需VP审批+脱敏+专项审计，处理时效≤10工作日 | §8.2 |
| "共享范围可以临时扩大吗？" | 需重新发起申请走变更审批流程，不可临时扩大 | §8.1 |
| "审计发现问题怎么追责？" | 依据日志记录追溯责任，按安全承诺书约定追责 | 数据安全法 §52 |
| "跨公司数据共享怎么处理？" | 需额外签署数据共享协议，法务审核后方可执行 | 数据安全法 §22 |
