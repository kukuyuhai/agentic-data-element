# ⚖️ three-rights-analyzer

## When to Use

当存在以下任一场景时应启用本技能进行三权分置分析：

- 数据资产确权咨询：需要明确某一数据资源/产品上的权利归属主体；
- 数据合作协议或授权合同起草前：需要界定各方在数据流转中的权能范围；
- 数据资产登记申请前：登记机构要求明确申报三权归属；
- 数据交易前权属尽职调查：买方需确认卖方是否具备相关处分权能；
- 数据资产入表前：需确认会计主体对所入表数据资源的合法持有和收益基础；
- 存在多方主体对同一数据集主张权利，需进行权利切割与剥离。

注意：本技能是纯法律/合规分析能力，不产生任何代码或工程产物。其输出是结构化权属判断文书，供后续 `evidence-chain-builder` 和 `registration-cert-drafter` 使用。

## 工作原则

1. **依法合规**：以《数据二十条》确立的"淡化所有权、强调使用权"为指导思想，所有判断必须溯源至法规或标准条款。
2. **促进流通**：在不违背法律强制性规定的前提下，优先选择有利于数据流通和再利用的权利配置方案。
3. **举证责任清晰**：每个权能的归属判定均附带举证要点，帮助用户明确需要准备哪些证据材料。
4. **可剥离性**：三权具备可分性，分析时须各别判定归属，不能笼统地"归某方所有"。
5. **场景敏感**：同一数据集的权能归属可能因采集方式、加工投入、合同约定不同而差异显著，必须结合具体场景分析。

## 三权分置框架

### 一、数据资源持有权

**定义**：对数据资源的实际控制和管理权限，包括存储、备份、访问控制、防篡改、保密等事实性支配能力。持有权不要求对数据拥有所有权或知识产权，重在"合法持有"这一事实状态。

**归属判定要素**（`T/MIITEC 025-2024 §5.1.3`）：
- 谁直接控制数据的物理/逻辑存储？
- 谁承担数据的安全管理责任？
- 数据获取方式是否合法（合同授权、依法采集、公开收集）？
- 是否存在排他性占有或共享持有安排？

**举证要点**：
1. 数据存储架构文档（服务器/云平台/数据库位置）
2. 数据安全管理制度和责任人证明
3. 数据来源合法性文件（采购合同、用户授权、采集合规证明）
4. 数据持有协议（如多方共享持有时的约定）

**典型归属方**：
- 自采数据：采集者（企业自身）
- 委托加工数据：委托方（除非合同另有约定）
- 联合采集数据：依合作协议确定（通常为共有或按份）
- 公共数据授权运营：被授权运营主体

### 二、数据加工使用权

**定义**：对数据进行清洗、标注、聚合、分析、建模、衍生计算等加工处理，并基于加工后的结果开展内部使用的权利。

**归属判定要素**：
- 谁实际投入了加工劳动（人力、算法、算力）？
- 加工行为是否在持有方授权范围内？
- 加工成果是否形成新的衍生数据或数据产品？
- 加工合同中对加工方权利如何约定？

**举证要点**：
1. 数据加工授权书或合同中的授权条款
2. ETL 流程文档、血缘追踪记录（可调用 `management/lineage-graph` 生成）
3. 加工资源投入证明（人力工时、算法模型清单、计算资源账单）
4. 加工成果的技术文档或产品说明

**典型归属方**：
- 自有数据加工：持有方（企业自身）
- 受托加工：合同约定的加工方或委托方
- 联合建模/加工：依合作协议确定
- AI 训练场景：训练方通常享有模型使用权，但底料数据持有权不因此转移

### 三、数据产品经营权

**定义**：将数据以数据集、API、分析报告、数据模型等形式产品化后，向第三方提供、销售、许可使用或产生收益的权利。经营权是三权中市场化程度最高的权能。

**归属判定要素**：
- 谁投入了产品化设计、封装、定价和渠道建设？
- 产品是否涉及他人的数据持有权或加工使用权（需取得相应授权）？
- 是否存在限制经营的法定条件（如涉及个人信息、国家秘密）？
- 产品经营是否需要取得特许经营资质？

**举证要点**：
1. 数据产品说明书（产品形态、定价、目标市场）
2. 上游权源链证明（持有权授权 + 加工使用权授权）
3. 数据产品知识产权登记或备案文件
4. 经营资质证明（如涉及征信、金融等需持牌经营的领域）

**典型归属方**：
- 完整权链（持有→加工→产品化均由同一主体完成）：该主体
- 授权经营：持有方将经营权授权给第三方（需书面授权）
- 合作经营：依合作协议确定收益分配和经营权限
- API/平台模式：平台方享经营权，数据提供方享收益分成

## 权利剥离策略

三权可以全部归属同一主体，也可以分别归属不同主体。以下为常见剥离方案：

### 策略一：全部转让（三权合一）
- 适用场景：数据资产一次性买断交易
- 操作方式：通过买卖合同将持有权、加工使用权、经营权一并转移至买方
- 注意事项：需确保转让方确实拥有全部三权，否则构成无权处分

### 策略二：部分许可（保留持有权，许可加工使用+经营）
- 适用场景：数据 API 授权、数据集许可使用
- 操作方式：持有权仍归原主，许可相对方在约定期限和范围内加工使用并对外经营
- 注意事项：许可合同需明确许可范围（地域、期限、使用方式、是否可再许可）

### 策略三：三权分离（各归一方）
- 适用场景：政府公共数据授权运营、多方联合数据合作
- 操作方式：
  - 持有权归 A（如政府作为数据源提供方保持控制）
  - 加工使用权归 B（如授权给数据处理企业）
  - 经营权归 C（如委托运营平台）
- 注意事项：需通过三方合同明确权责边界和收益分配

### 剥离决策树

进行权利剥离时，按以下顺序依次判定：

```
开始
│
├─ 数据来源是否合法？（否 → 无法确权，终止分析）
├─ 持有权是否可以确定单一主体？
│  ├─ 是 → 记录持有权归属，继续
│  └─ 否 → 标记为"共有持有"或"权属待明确"
├─ 加工使用是否需要独立许可？
│  ├─ 是且合同有约定 → 按合同判定加工使用权归属
│  ├─ 是但合同无约定 → 提示补充约定
│  └─ 否（持有方自行加工）→ 加工使用权与持有权重合
├─ 是否涉及对外经营/交易？
│  ├─ 是 → 判断经营权归属（自有产品 vs 授权经营 vs 合作经营）
│  └─ 否 → 暂不涉及经营权的配置
└─ 生成权属分析报告
```

## Inputs

## Steps

**前置条件**：用户已提供待分析数据资产的描述和涉事主体信息。

1. **信息确认**：逐项确认 Inputs 中的必填字段是否齐全，若不齐则引导用户补充。
2. **来源合法性判定**：基于数据采集方式和已有合同，判定数据来源是否合法。若不合法，输出 `unable_to_confirm`，终止分析。
3. **持有权归属分析**：
   - 依据存储控制、安全管理责任和数据合法来源，判定持有权归属
   - 若多个主体共同持有，标记为"共有持有"并说明各自持有比例或范围
   - 溯源标注：`数据二十条（2022）`
4. **加工使用权归属分析**：
   - 识别加工劳动的投入主体
   - 审查加工相关合同条款，判定约定归属
   - 若无合同约定，提示风险并建议补充
   - 溯源标注：`T/MIITEC 025-2024 §5.1.3`
5. **经营权归属分析**：
   - 判断是否涉及对外经营/交易
   - 判权源链完整性：上游持有权 + 加工使用权是否向经营主体开放
   - 识别是否需要特殊资质
6. **整合权属画像**：将三权分析结果合并，标注各权利之间的关系（重合/分离/待定）。
7. **推荐剥离策略**：基于权属画像和 target_scenario，推荐全部转让/部分许可/三权分离中的一种。
8. **生成风险清单与后续建议**：汇总所有权属不确定性和风险点，给出交叉技能引用建议。
9. **输出结构化报告**：按 Outputs 格式整理输出。
10. 当需要构建证据链时，将控制权转交给 `foundation/handoff-router` 路由至 `evidence-chain-builder`。

## 常见问法与处理策略

| 问法 | 处理策略 |
|------|----------|
| "这个数据集归谁？" | 引导用户说明是对三权中的哪一项（持有/加工使用/经营）有疑问，三权可能分属不同主体。 |
| "数据是我们采集的，加工使用权当然也是我们的" | 这是常见误解。采集仅仅保证合法的持有权基础，加工使用权如果已通过合同许可他人，则加工方享有约定的使用权。 |
| "我们是委托方，加工出来的数据当然归我们" | 需要查看委托加工合同的具体约定。《数据二十条》鼓励通过合同约定加工成果归属，而非自动归于委托方。 |
| "数据二十条说淡化所有权，那到底谁有权卖数据？" | 淡化所有权不否定经营权。如果某一主体同时具备持有权+加工使用权且数据产品不涉及他人权益，则该主体可以依法经营。 |
| "政府开放数据能不能商业化？" | 取决于授权运营协议。持有权通常保留在政府，加工使用权和经营权可授权给运营方，运营收益需按约定分配。 |
| "个人信息可以交易吗？" | 原始个人信息不能直接交易。但经过匿名化处理的衍生数据（不指向特定个人、不可复原）可以作为数据产品经营。需先使用 `de-identification-toolkit` 确认去标识化达标。 |
| "三权分析结果和合同条款冲突怎么办？" | 以合同为准（契约优先原则），但需提醒：如果合同中约定的权属与事实状态不符（如约定转让持有权但数据仍在原控制方服务器中），则该约定可能无法实际履行。 |
| "这个分析结果有没有法律效力？" | 本技能输出的是结构化合规分析报告，不是法律意见书。如需法律效力，应委托执业律师出具正式法律意见。 |
| "跨境数据传输场景下三权怎么分析？" | 需额外增加数据出境合规分析维度。三权本身不受跨境影响，但行使各项权利（特别是经营权的产品出口/API 跨境调用）需符合《数据出境安全评估办法》等规定。此场景需转交至合规专项技能。 |
