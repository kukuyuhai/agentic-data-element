# 🏗️ deal-structuring-3paths

三条路径结构方案——全部转让、独家许可、作价入股的税务与治理影响对比与选择。

## When to Use

- 产权类数据交易方案设计阶段，需要选择交易路径时
- 买卖方对交易结构有不同诉求，需要对比方案优劣势时
- 税务筹划需要评估不同交易路径的税务影响时
- 交易涉及作价入股，需要评估股权结构和治理影响时

## 工作原则

1. **三权适配**：交易路径选择需匹配三权分置框架，明确转让/许可/入股的是哪些权利
2. **税务先行**：方案设计阶段同步评估税务影响，避免事后税务风险
3. **治理评估**：作价入股路径需评估对买方股权结构和数据治理的影响
4. **买方诉求导向**：路径选择应优先满足买方核心诉求（控制权/成本/灵活性）
5. **法律合规**：三条路径均需符合《数据二十条》和 T/MIITEC 025-2024 要求 `§5.2.6`

## 三条交易路径详解

### 路径一：全部转让

**定义：** 卖方将数据产权的全部权利（持有权/加工使用权/经营权）一次性永久转让给买方。

**交易结构：**

| 要素 | 内容 |
|------|------|
| 权利转移 | 三权全部转移，卖方不再享有任何权利 |
| 对价方式 | 一次性支付（现金为主） |
| 对价确定 | 评估作价（成本法/收益法/市场法） |
| 交付方式 | 数据资产整体移交 + 权属变更登记 |
| 后续关系 | 卖方与数据无任何权利关系（可能有保密义务） |

**优势与劣势：**

| 优势 | 劣势 |
|------|------|
| 交易关系简单清晰 | 卖方丧失数据全部权利 |
| 买方获得完全控制权 | 买方一次性资金压力大 |
| 无后续纠纷风险 | 卖方无法享受后续增值收益 |
| 适合卖方退出场景 | 税负可能较高（转让所得税） |

**税务影响：**

| 税种 | 税率 | 计算基础 | 备注 |
|------|------|---------|------|
| 企业所得税 | 25% | 转让收入 - 成本 - 税费 | 卖方缴纳 |
| 增值税 | 6%（现代服务） | 转让收入 | 卖方缴纳，买方可抵扣 |
| 印花税 | 0.05% | 合同金额 | 双方缴纳 |

**治理影响：**

- 买方获得数据完全控制权，可自主决定数据使用、加工、再交易
- 卖方退出数据治理体系，不再参与数据相关决策
- 买方需建立数据治理体系（如尚未建立）

### 路径二：独家许可

**定义：** 卖方保留数据持有权，将加工使用权和/或经营权独家授予买方，买方在约定期限和范围内享有排他性权利。

**交易结构：**

| 要素 | 内容 |
|------|------|
| 权利转移 | 不转移持有权，许可加工使用权和/或经营权 |
| 对价方式 | 许可费（一次性 + 按期/按量） |
| 对价确定 | 许可费评估（收益法为主） |
| 交付方式 | 数据使用授权 + API/数据集交付 |
| 后续关系 | 持续的许可关系，卖方保留持有权 |

**优势与劣势：**

| 优势 | 劣势 |
|------|------|
| 卖方保留数据持有权 | 买方无法获得完全控制权 |
| 买方资金压力小（可分期） | 存在许可到期后权利终止风险 |
| 卖方可享受持续收益 | 关系复杂，需持续管理 |
| 灵活性高，可定制许可范围 | 买方使用受许可条款限制 |

**税务影响：**

| 税种 | 税率 | 计算基础 | 备注 |
|------|------|---------|------|
| 企业所得税 | 25% | 许可收入 - 成本 | 卖方缴纳 |
| 增值税 | 6%（特许权使用费） | 许可费 | 卖方缴纳 |
| 印花税 | 0.05% | 合同金额 | 双方缴纳 |

**治理影响：**

- 买方获得排他性使用权，但非所有权
- 卖方保留持有权，需持续维护数据质量和安全
- 双方需建立许可管理机制（使用监督/到期处理）
- 买方衍生数据权属需在合同中明确约定

### 路径三：作价入股

**定义：** 卖方将数据产权评估作价，作为出资投入买方企业，获得买方股权。

**交易结构：**

| 要素 | 内容 |
|------|------|
| 权利转移 | 三权作价出资，转移至买方企业 |
| 对价方式 | 股权（买方公司股份） |
| 对价确定 | 评估作价（收益法为主，需第三方评估） |
| 交付方式 | 数据资产移交 + 股权变更登记 |
| 后续关系 | 卖方成为买方股东，参与公司治理 |

**优势与劣势：**

| 优势 | 劣势 |
|------|------|
| 买卖方利益深度绑定 | 交易结构复杂，周期长 |
| 买方无需现金支出 | 卖方承担买方经营风险 |
| 卖方可享受长期股权增值 | 需完成工商变更和评估 |
| 适合战略合作场景 | 治理结构需重构 |

**税务影响：**

| 税种 | 税率 | 计算基础 | 备注 |
|------|------|---------|------|
| 企业所得税 | 25% | 视同转让，按公允价值 - 成本 | 卖方缴纳（可递延纳税） |
| 印花税 | 0.05% | 股权变更金额 | 双方缴纳 |
| 增值税 | 6% | 视同转让 | 卖方缴纳（符合条件可免） |

> 注：作价入股符合条件时可申请递延纳税（企业所得税），即暂不缴纳，待股权转让时再缴纳。

**治理影响：**

- 卖方成为买方股东，享有股东权利（表决权/分红权/知情权）
- 买方股权结构变化，需修改公司章程
- 卖方可能获得董事会席位（视股权比例）
- 数据治理纳入公司治理体系，卖方有持续参与权

## 三路径对比表

| 维度 | 全部转让 | 独家许可 | 作价入股 |
|------|:-------:|:-------:|:-------:|
| 权利转移 | 三权全部 | 使用权+经营权 | 三权全部 |
| 对价方式 | 一次性现金 | 许可费（分期/按量） | 股权 |
| 资金压力 | 高（买方） | 低 | 无 |
| 税负水平 | 中高 | 中 | 中（可递延） |
| 关系复杂度 | 低 | 中 | 高 |
| 卖方控制权 | 完全丧失 | 保留持有权 | 转为股东参与 |
| 买方控制权 | 完全获得 | 排他使用 | 获得+卖方参与治理 |
| 适用场景 | 卖方退出 | 持续合作 | 战略绑定 |
| 交易周期 | 1-3月 | 1-2月 | 3-6月 |

## 路径选择指南

```
买方核心诉求是什么？
├── 完全控制数据资产
│   └── 买方资金充足？
│       ├── 是 → 全部转让
│       └── 否 → 作价入股（如卖方愿意持股）
├── 灵活使用，控制成本
│   └── 独家许可
└── 战略合作，深度绑定
    └── 作价入股

卖方核心诉求是什么？
├── 完全退出，一次性变现
│   └── 全部转让
├── 保留持有权，持续收益
│   └── 独家许可
└── 深度参与买方发展
    └── 作价入股
```

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 买方诉求 | Document | 买方对控制权/成本/灵活性的诉求 |
| 卖方诉求 | Document | 卖方对退出/收益/参与的诉求 |
| 尽调报告 | Document | `three-rights-due-diligence` 输出 |
| 评估报告 | Document | 数据资产评估报告（成本法/收益法/市场法） |
| 税务约束 | List | 买卖方的税务约束和优化诉求 |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 方案设计书 | Document | 推荐路径 + 交易结构 + 对价方案 |
| 对比评估表 | Document | 三路径优劣势对比和适配度评估 |
| 税务分析报告 | Document | 各路径的税务影响和优化建议 |
| 治理影响评估 | Document | 作价入股时的股权和治理影响 |

## Steps

1. 收集买卖方诉求、尽调报告、评估报告等输入
2. 评估三条路径的适配度，输出对比评估表
3. 选定推荐路径，设计交易结构和对价方案
4. 评估税务影响，提出税务优化建议
5. 评估治理影响（尤其作价入股路径）
6. 输出方案设计书 `§5.2.6`
7. 方案经买卖方确认后，传递给 `contract-key-clauses` 起草合同
8. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| 数据产权交易有几种方式？ | 三种：全部转让（一次性买断）、独家许可（保留持有权）、作价入股（换股权） |
| 哪种方式税负最低？ | 独家许可税负相对最低；作价入股可申请递延纳税；全部转让税负较高 |
| 作价入股怎么估值？ | 需第三方评估机构出具评估报告，通常用收益法（未来收益折现） |
| 独家许可到期后怎么办？ | 合同约定到期后权利终止，买方需停止使用并销毁数据；或约定续期条款 |
| 买方资金不足选哪条路？ | 独家许可（分期付许可费）或作价入股（用股权代替现金） |
| 三条路径能组合吗？ | 可以，如"部分转让 + 部分许可"或"许可 + 期权"，但结构更复杂，需法务参与 |
