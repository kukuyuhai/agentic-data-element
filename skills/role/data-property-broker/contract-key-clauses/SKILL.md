# 📜 contract-key-clauses

产权交易合同关键条款清单——六大核心条款的模板文本、要点解析与谈判技巧。

## When to Use

- 产权类数据交易合同起草阶段，需要参考标准条款模板时
- 合同审核阶段，需要逐条核查关键条款是否完备时
- 合同谈判阶段，需要了解各条款的谈判要点和让步空间时
- 交易结构方案（`deal-structuring-3paths`）确定后，需要转化为合同条款时

## 工作原则

1. **六条款完备**：标的界定、权利范围、对价、陈述保证、违约救济、争议解决六项缺一不可
2. **数据特殊性**：合同条款需体现数据资产的特殊性（非排他性/可复制性/衍生数据/三权分置）
3. **风险前置**：通过陈述保证和违约救济条款前置分配风险，而非事后争议
4. **可执行性**：条款内容必须具体可执行，避免"合理""适当"等模糊表述
5. **法律对齐**：条款内容对齐《民法典》合同编和《数据二十条》要求

## 六大关键条款详解

### 一、标的界定条款

**目的：** 精确界定交易标的的数据资产范围、量级和三权类型。

**模板文本：**

```
第一条 交易标的

1.1 本合同标的为以下数据资产（以下简称"标的资产"）：

    （1）资产名称：_______________
    （2）资产标识：_______________（数据资产登记编号）
    （3）数据范围：_______________（字段清单详见附件一）
    （4）数据量级：约 ___ 条/GB
    （5）时间范围：___ 年 ___ 月 至 ___ 年 ___ 月
    （6）数据格式：_______________
    （7）三权类型：卖方持有/加工使用/经营权的 ___ 权

1.2 标的资产的具体技术规格和质量标准详见附件一《数据资产规格书》。

1.3 卖方确认标的资产不含附件二《排除清单》中列明的内容。
```

**关键要点：**

| 要点 | 说明 | 谈判技巧 |
|------|------|---------|
| 资产标识 | 引用数据资产登记编号，确保唯一性 | 要求卖方提供登记证明 |
| 数据范围 | 字段清单作为附件，穷尽列举 | 买方应要求查看样本数据 |
| 量级标注 | 用"约"字留出合理偏差空间 | 约定偏差超过5%的处理方式 |
| 三权类型 | 明确交易的是哪些权利 | 与 `deal-structuring-3paths` 选定路径一致 |
| 排除清单 | 明确不含哪些内容 | 避免卖方用"等"字模糊范围 |

### 二、权利范围条款

**目的：** 界定买方获得权利的使用目的、地域、期限、再许可和衍生数据权属。

**模板文本：**

```
第二条 权利范围

2.1 使用目的：买方仅可将标的资产用于以下目的：
    _______________

2.2 地域范围：买方仅可在以下地域使用标的资产：
    [ ] 仅中华人民共和国境内
    [ ] 全球
    [ ] 其他：_______________

2.3 使用期限：
    [ ] 永久（适用于全部转让）
    [ ] ___ 年/月（适用于许可），自交付之日起算

2.4 再许可：买方 [可以/不可以] 将标的资产再许可给第三方。
    如可以再许可，需满足以下条件：_______________

2.5 衍生数据：买方基于标的资产加工产生的衍生数据，
    其权属归 [买方/卖方/共有/另行约定]。
    衍生数据的定义为：_______________

2.6 限制条款：买方不得将标的资产用于以下用途：
    （1）_______________
    （2）_______________
```

**关键要点：**

| 要点 | 说明 | 谈判技巧 |
|------|------|---------|
| 使用目的 | 明确具体用途，不可过于宽泛 | 买方争取宽泛，卖方要求具体 |
| 地域范围 | 涉及出境的需特别约定 | 与 `cross-border-checker` 结果对齐 |
| 再许可权 | 影响买方商业灵活性 | 买方争取再许可权，卖方限制 |
| 衍生数据权属 | 数据交易特有条款，极易争议 | 明确衍生数据定义和权属归属 |
| 限制条款 | 排除竞品使用等场景 | 卖方应争取限制条款 |

### 三、对价条款

**目的：** 约定交易金额、支付方式和价格调整机制。

**模板文本（摘要）：**

```
第三条 交易对价
3.1 交易总价：人民币 ___ 元（大写：_______________）。
3.2 支付方式：[一次性/分期（签合同__%/交付__%/验收__%）/按量计费]
3.3 价格调整：实际量级偏差超 ±5% 时，按 ___ 方式调整对价。
3.4 发票：卖方于收到款项后 ___ 个工作日内开具合规发票。
```

### 四、陈述保证条款

**目的：** 卖方对数据产权合法性、来源合法性、不侵权、PII合规等做出保证。

**模板文本：**

```
第四条 陈述与保证

4.1 卖方陈述与保证：
    （1）卖方合法持有标的资产的相关权利，有权进行本次交易；
    （2）标的资产来源合法，已取得所有必要的授权和同意；
    （3）标的资产不侵犯任何第三方的合法权益（包括但不限于知识产权、商业秘密）；
    （4）标的资产如涉及个人信息，已取得个人信息主体的合法授权并完成脱敏处理；
    （5）标的资产不存在质押、查封、冻结等权利限制；
    （6）卖方未隐瞒任何可能影响交易标的价值的重大信息。

4.2 买方陈述与保证：
    （1）买方具备签署和履行本合同的主体资格；
    （2）买方将按约定用途使用标的资产，不违反适用法律法规。

4.3 保证期限：本条陈述与保证在合同有效期内及终止后 ___ 年内持续有效。
```

**关键要点：**

| 要点 | 说明 | 谈判技巧 |
|------|------|---------|
| 权利合法 | 卖方保证有权交易 | 买方应要求提供权属证明 |
| 来源合法 | 卖方保证数据来源合法 | 与尽调报告交叉验证 |
| 不侵权 | 卖方保证不侵犯第三方权利 | 买方争取侵权赔偿条款 |
| PII合规 | 涉及PII时必须保证 | 与 `pii-scan` 结果对齐 |
| 保证期限 | 陈述保证的持续有效期 | 买方争取更长保证期（建议3-5年） |

### 五、违约救济条款

**目的：** 约定违约情形、违约金、解除权和数据销毁义务。

**模板文本（摘要）：**

```
第五条 违约救济
5.1 卖方违约：延迟交付按日 ___% 违约金；交付不符可要求整改或解除；
    违反陈述保证，买方可解除合同并要求退还对价+赔偿。
5.2 买方违约：延迟付款按日 ___% 违约金；超范围使用卖方可要求停止+赔偿；
    严重违约卖方可解除合同并要求销毁标的资产。
5.3 数据销毁：合同终止后买方 ___ 日内销毁全部标的资产及副本，出具销毁证明。
5.4 损失赔偿：违约方赔偿守约方直接损失，含律师费、诉讼费等维权合理费用。
```

### 六、争议解决条款

**目的：** 约定管辖法院、法律适用和仲裁方式。

**模板文本（摘要）：**

```
第六条 争议解决
6.1 法律适用：适用中华人民共和国法律（不含港澳台地区法律）。
6.2 争议解决：[诉讼→___地法院 / 仲裁→___仲裁委员会，___地仲裁，裁决终局]
6.3 争议期间义务：双方继续履行无争议部分。
```

## 与通用合同条款库的区别

| 维度 | 本技能（产权交易专用） | 通用合同条款库（`contract-clause-library`） |
|------|:-------------------:|:--------------------------------------:|
| 适用场景 | 数据产权交易（转让/许可/入股） | 通用数据交易 |
| 三权分置 | 有专门条款 | 无或简略 |
| 衍生数据 | 有详细约定 | 无或简略 |
| 数据销毁 | 有强制条款 | 可能有但不详细 |
| 保证期限 | 3-5年 | 通常1年 |
| 评估作价 | 有作价入股相关条款 | 无 |

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 交易结构方案 | Document | `deal-structuring-3paths` 输出的选定路径 |
| 尽调报告 | Document | `three-rights-due-diligence` 输出的尽调结果 |
| 评估报告 | Document | 数据资产评估报告 |
| 买卖方协商记录 | List | 双方谈判中达成的共识和分歧 |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 合同草案 | Document | 六大关键条款完整的合同草案 |
| 条款审核清单 | Document | 逐条审核的检查清单 |
| 谈判要点备忘 | Document | 各条款的谈判要点和让步空间 |

## Steps

1. 接收交易结构方案和尽调报告等输入
2. 根据选定交易路径选择对应条款模板
3. 逐条起草六大关键条款（标的/权利/对价/保证/救济/争议）
4. 填入交易具体信息（资产/价格/期限等）
5. 交叉验证条款与尽调报告、交易结构方案的一致性
6. 输出合同草案和条款审核清单
7. 买卖方谈判后修订合同
8. 合同定稿后进入签署流程
9. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| 数据产权合同必须有哪些条款？ | 六大关键条款：标的界定、权利范围、对价、陈述保证、违约救济、争议解决 |
| 衍生数据权属怎么约定？ | 在权利范围条款中明确衍生数据的定义和权属归属（买方/卖方/共有/另行约定） |
| 违约金怎么定？ | 延迟履行按日万分之五至千分之一；根本违约约定合同金额的10-30% |
| 数据销毁怎么执行？ | 合同终止后买方在约定期限内销毁全部数据及副本，出具销毁证明 |
| 选诉讼还是仲裁？ | 诉讼成本低但公开；仲裁保密性好但成本高；涉及商业秘密建议选仲裁 |
| 保证期限定多久？ | 建议陈述保证期限为3-5年，覆盖数据产权诉讼时效 |
| 和通用合同库有什么区别？ | 本技能专用于产权交易，有三权分置/衍生数据/数据销毁等数据产权特有条款 |
