# 🔎 three-rights-due-diligence

数据产权尽调——三权归属核查、权利瑕疵扫描、证据链完整性验证，保障产权交易安全。

## When to Use

- 产权类数据交易（全部转让/独家许可/作价入股）立项前，需要开展尽职调查时
- 数据资产入表前，需要确认产权归属和权利完整性时
- 买方对数据产权存在疑虑，要求提供产权尽调报告时
- 交易标的涉及复杂权属关系（如多方共有、委托加工、衍生数据），需要厘清时

## 工作原则

1. **三权分置**：严格按"持有权/加工使用权/经营权"三权分置框架核查，不混为一谈
2. **证据闭环**：每一项权属结论必须有证据支撑，形成"主张→证据→结论"闭环
3. **风险量化**：尽调发现的风险按高/中/低分级，高风险项一票否决
4. **独立客观**：尽调由独立第三方或平台合规团队执行，不受买卖方干预
5. **法定标准**：尽调标准对齐《数据二十条》和 T/MIITEC 025-2024 的要求 `§5.2.6`

## 尽调范围

### 一、三权归属核查

**核查目标：** 确认数据资源的持有权、加工使用权、经营权各归属谁，是否存在争议。

#### 1. 持有权核查

| 检查项 | 检查内容 | 证据要求 | 风险等级 |
|-------|---------|---------|:-------:|
| 持有权主体 | 谁合法持有该数据资源 | 数据采集协议/自有数据声明 | 高 |
| 持有合法性 | 持有方式是否合法（自有/采集/购买） | 采集授权/采购合同/自有证明 | 高 |
| 持有限制 | 是否存在限制持有的条款或约定 | 合同限制条款/法律法规限制 | 中 |
| 共有情况 | 数据是否与他人共有 | 共有协议/合作合同 | 中 |

#### 2. 加工使用权核查

| 检查项 | 检查内容 | 证据要求 | 风险等级 |
|-------|---------|---------|:-------:|
| 加工使用权主体 | 谁有权加工使用该数据 | 加工授权/委托加工合同 | 高 |
| 加工范围 | 加工使用的范围和限制 | 授权范围条款 | 中 |
| 加工成果归属 | 加工后的衍生数据归谁 | 合同约定/权属声明 | 高 |
| 转委托限制 | 是否允许转委托加工 | 合同条款 | 中 |

#### 3. 经营权核查

| 检查项 | 检查内容 | 证据要求 | 风险等级 |
|-------|---------|---------|:-------:|
| 经营权主体 | 谁有权经营（交易/处分）该数据 | 经营授权/权属声明 | 高 |
| 经营范围 | 可经营的具体权利范围（交易/许可/入股） | 授权文件 | 高 |
| 经营限制 | 经营的时间/地域/对象限制 | 合同限制条款 | 中 |
| 排他性 | 经营权是否排他（独家/非独家） | 授权协议 | 中 |

### 二、权利瑕疵扫描

**扫描目标：** 发现可能影响产权交易的权利瑕疵和潜在风险。

| 扫描项 | 扫描内容 | 发现方式 | 处理策略 |
|-------|---------|---------|---------|
| 限制条款 | 合同中是否存在限制交易/许可的条款 | 审查所有相关合同 | 有限制则需解除后方可交易 |
| 第三方权利 | 是否存在质押/查封/冻结等第三方权利 | 查询登记系统/法院公告 | 有第三方权利则不可交易 |
| 诉讼仲裁 | 是否存在涉及该数据产权的诉讼或仲裁 | 查询裁判文书网/仲裁机构 | 有未决诉讼则暂停交易 |
| 行政处分 | 是否受过行政处罚或被列入失信名单 | 查询信用中国/行政处罚公示 | 有严重处罚则评估风险 |
| 知识产权冲突 | 数据产品是否侵犯他人著作权/商业秘密 | 知识产权检索/比对 | 有冲突则需解决后方可交易 |
| 数据合规风险 | 数据采集和使用是否存在合规瑕疵 | 合规审查/ `compliance-review-sheet` | 有合规瑕疵则需整改 |

### 三、证据链完整性验证

**验证目标：** 确认从数据来源到当前权属状态的证据链完整无断裂。

| 证据节点 | 证据内容 | 完整性标准 |
|---------|---------|-----------|
| 来源证据 | 数据最初来源的合法证明 | 有采集授权/采购合同/自有证明 |
| 流转证据 | 数据每次流转的记录和协议 | 每次流转有合同/协议记录 |
| 加工证据 | 数据加工处理的记录和留痕 | 有加工日志/处理记录/脱敏记录 |
| 权属声明 | 当前权属主体的权属声明书 | 有书面权属声明且签署有效 |
| 授权链路 | 从数据源到卖方的完整授权链 | 链路无断裂，每环可追溯 |

**证据链断裂风险：**

| 断裂位置 | 风险描述 | 风险等级 |
|---------|---------|:-------:|
| 来源端断裂 | 无法证明数据最初来源合法 | 高（一票否决） |
| 中间断裂 | 某次流转缺少合同或记录 | 高（需补充证据） |
| 加工断裂 | 加工过程无留痕 | 中（需补充说明） |
| 授权断裂 | 授权链路某环缺失 | 高（一票否决） |

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 交易标的说明 | Document | 数据产品/资产的描述和范围 |
| 卖方权属声明 | Document | 卖方出具的数据产权声明书 |
| 相关合同文件 | Archive | 采集/采购/加工/授权等所有相关合同 |
| 合规审查结果 | Document | `compliance-review-sheet` 输出 |
| 信用/诉讼查询 | List | 信用中国/裁判文书网等查询结果 |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 尽调报告 | Document | 完整的产权尽调报告 |
| 风险清单 | List | 发现的风险项及等级 |
| 证据包 | Archive | 所有支撑尽调结论的证据材料 |
| 整改清单 | List | 附条件通过时的整改事项 |

## Steps

1. 确定尽调范围，收集交易标的说明和卖方权属声明
2. 执行三权归属核查（持有权/加工使用权/经营权）
3. 执行权利瑕疵扫描（限制条款/第三方权利/诉讼/处罚/知识产权/合规）
4. 验证证据链完整性（来源→流转→加工→授权链路）
5. 汇总发现，进行风险评级
6. 输出尽调报告和风险清单 `§5.2.6`
7. 如有高风险项，标注不通过；如有中低风险，输出整改清单
8. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| 数据产权尽调查什么？ | 三方面：三权归属（持有/加工使用/经营）、权利瑕疵（限制/诉讼/第三方权利）、证据链完整性 |
| 什么是三权分置？ | 《数据二十条》提出的持有权、加工使用权、经营权分置框架，确认各权利归属谁 |
| 证据链断裂怎么办？ | 来源端或授权链断裂为高风险（一票否决），中间断裂需补充证据，加工断裂需补充说明 |
| 尽调发现限制条款怎么办？ | 需解除限制后方可交易；无法解除的，交易不通过 |
| 尽调和合规审查什么关系？ | 合规审查（`compliance-review-sheet`）侧重数据合规，尽调侧重产权归属，两者互补 |
| 尽调报告有效期多久？ | 建议有效期为 6 个月，超期或权属状态变化时需重新尽调 |
