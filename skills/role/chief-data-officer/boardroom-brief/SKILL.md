# 🎤 boardroom-brief

## When to Use

当以下场景出现时调用本技能：

- **董事会/高管层数据议题汇报**：CDO 需在有限时间内（通常 15-20 分钟）向董事会汇报数据领域核心议题，需要高度精炼的简报
- **年度战略答辩准备**：在年度规划会上向董事会争取数据战略支持和预算，需一页纸说清"为什么、做什么、要什么"
- **重大数据项目决策**：如数据中台建设、数据资产入表启动、隐私合规体系搭建等需要董事会审批的关键决策
- **季度数据进展同步**：向非技术背景的董事会成员汇报数据工作进展，避免技术术语，聚焦业务价值
- **危机/风险通报**：数据安全事件、监管处罚风险等需要董事会知悉和决策的紧急事项

## 工作原则

1. **≤2 页铁律**：核心内容必须控制在一张 A4 纸正反面（即 2 页以内），优先保证简洁性和可读性
2. **高层语言**：使用董事会熟悉的商业语言（收入、成本、风险、竞争），而非技术术语（ETL、Hadoop、数据湖）
3. **决策导向**：每一个议题必须指向一个明确需要董事会做出的决策项，非决策信息放入附件
4. **数据说话**：每条论点必须配套一个关键数字，禁止纯定性描述
5. **风险透明**：不隐瞒、不美化，对合规和组织风险如实披露

## 简报四节结构

### 第一节：形势 (Market & Landscape) — 占 25% 版面

告诉董事会"外部正在发生什么"，建立紧迫感和战略必要性。

**包含元素**：
- **行业趋势**：行业内数据资产化的最新动态（如竞对数据产品上线、行业数据交易所新规等），1-2 条即可
- **竞争格局**：主要竞争对手在数据领域的投入和进展，与自身对比
- **监管动向**：新出台或即将出台的数据相关法规政策（如数据资产入表暂行规定、个人信息保护法等），标注影响等级（高/中/低）

**模板**：

```
【行业趋势】
- 事件1：XXX 已推行数据资产入表（首批 XX 家），行业领先企业已完成 XXX
- 事件2：国家数据局发布 XXX 指导意见，预计 XX 个月内影响本行业

【竞争格局】
- 竞对A：数据产品收入占营收 X%，我方目前为 Y%
- 竞对B：DCMM 已通过 X 级，我方为 Y 级

【监管动向】
- XX 法规将于 XX 生效，影响等级：高 → 需 XX 个月内完成合规整改
```

### 第二节：进展 (Progress) — 占 35% 版面

告诉董事会"我们做到了什么"，展示执行力和阶段性成果。

**包含元素**：
- **里程碑达成**：本周期内完成的关键节点（如数据目录上线、DCMM 评估通过、首个数据产品交付等），每个里程碑配套完成时间
- **KPI 达成**：北极星指标和关键结果的进度（可引用 `data-strategy-canvas` 中的 OKR），每个指标附达标状态（✅ 达标 / ⚠️ 风险 / ❌ 未达标）
- **重点项目状态**：正在进行的 Top 3 项目，状态摘要（1-2 句话），预计完成时间

**模板**：

```
【里程碑】
✅ 数据资产目录覆盖率达 85% — 2026 Q3 达成
✅ DCMM 三级评估通过 — 2026 Q4 通过
⚠️ 数据中台核心模块上线 — 延期 1 个月，预计 Q1 完成

【KPI 达成】
| KPI | 目标 | 实际 | 状态 |
|-----|------|------|------|
| 数据目录覆盖率 | 85% | 82% | ⚠️ |
| 自动化报表率 | 90% | 92% | ✅ |
| 数据收入占比 | 5% | 3.8% | ❌ |

【重点项目】
1. 数据资产入表试点（进度 70%，预计 Q2 完成首次入表）
2. 隐私计算平台建设（选型完成，Q2 启动 POC）
3. 全域数据分类分级（已完成 60%，Q3 完成）
```

### 第三节：风险 (Risks) — 占 25% 版面

告诉董事会"可能出什么问题，我们做了什么准备"。

**包含元素**：
- **技术风险**：数据平台稳定性、技术债务、供应商风险等
- **合规风险**：未完成的合规整改项、监管窗口期风险、数据跨境等
- **组织风险**：关键人才缺口、部门协作瓶颈、高层支持度变化等

**模板**：

```
【风险矩阵】

| 风险项 | 等级 | 影响描述 | 当前措施 |
|-------|------|---------|---------|
| XXX平台稳定性不足 | 高 | 可能导致核心报表中断 | 已完成灾备切换演练，Q2 升级架构 |
| 数据分类分级未完成 | 中 | 影响入表合规性 | 外聘咨询团队加速，预计 Q3 完成 |
| 数据工程师离职率上升 | 中 | 影响项目交付节奏 | 已启动招聘 + 内部转岗培训 |

当前高风险项：1 个；中风险项：2 个；低风险项：0 个
整体风险敞口：[简要评估：可控/需关注/紧急]
```

### 第四节：决策项 (Decisions Required) — 占 15% 版面

告诉董事会"需要你们做什么决策"，这是简报的核心目标。

**规则**：
- 每一项决策必须包含：议题 + 选项（至少 2 个）+ 推荐方案 + 所需资源/时间
- 每场汇报决策项 ≤ 3 个，优先保证高优先级议题
- 每个决策项 2-3 句话说明

**模板**：

```
【请董事会决策】

1. 数据中台二期预算审批
   选项A：批复 ¥2,000 万（完整建设，12 个月）【推荐】
   选项B：批复 ¥1,200 万（核心模块，9 个月）
   选项C：暂缓审批，Q2 再审
   → 推荐 A：可覆盖全域数据治理，预计 ROI 3 年回本

2. 启动数据资产入表试点
   选项A：2026 Q2 启动（5 个数据集）【推荐】
   选项B：2026 Q4 启动（待分类分级完成后）
   → 推荐 A：早日入表可抢占监管窗口期，符合行业趋势

3. 核心数据团队扩编
   提案：新增 8 人（4 数据工程师 + 2 治理 + 2 分析）
   → 年化投入 ¥400 万，支撑 OKR 中 3 个关键结果的达成
```

## Inputs

| 输入项 | 类型 | 必填 | 说明 | 示例 |
|-------|------|------|------|------|
| `briefing_purpose` | string | 是 | 汇报目的：`annual_plan`/`progress_sync`/`decision_request`/`crisis_update` | `"annual_plan"` |
| `strategy_canvas` | object | 否 | 数据战略画布输出（可调用 `data-strategy-canvas` 获取） | `{...canvas_summary...}` |
| `progress_data` | object | 是 | 当前进展数据：里程碑、KPI 完成情况 | `{"milestones": [...], "kpis": [...]}` |
| `risk_register` | array | 是 | 风险登记册（可调用 `data-strategy-canvas` 的风险模块获取） | `[{"name": "...", "level": "high", ...}]` |
| `decision_items` | array | 是 | 需董事会决策的事项（每项含议题 + 选项 + 推荐） | `[{"topic": "...", "options": [...], "recommendation": "..."}]` |
| `opening_remarks` | string | 否 | 开场要点/核心信息 | `"今年数据工作聚焦三个关键词：入表、覆盖、合规"` |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| `brief_document` | markdown | 完整简报文档（≤2 页 A4 排版后的内容量） |
| `executive_summary` | string | 30 秒电梯演讲版本（3 句话核心信息） |
| `slide_structure` | list | PPT 建议结构（每页标题 + 要点） |
| `talking_points` | list | 汇报口径建议：哪些应强调、哪些应淡化、潜在刁钻问题预案 |

## Steps

### Step 1：明确汇报目的与受众

1. 确认汇报目的（年度战略申请 / 进度同步 / 重大决策请求 / 风险通报）
2. 确认受众的政治敏感度和数据素养水平（首次汇报 / 例行汇报？有无技术背景？）
3. 确认时间限制（15 分钟 / 30 分钟 / 60 分钟），决定简报深度
4. 确认是否有附件/补充材料的呈现机会

### Step 2：采集和整合素材

1. 收集数据战略画布输出（可调用 `role/chief-data-officer/data-strategy-canvas`）
2. 收集当前进展数据：里程碑完成情况、KPI 达成率
3. 收集风险登记册（或自行识别 Top 5 风险）
4. 收集外部信息：行业趋势、竞争情报、最新监管法规（可调用 `foundation/regulation-lookup` 和 `foundation/standard-lookup`）
5. 收集 ROI 评估数据（可调用 `role/chief-data-officer/data-roi-model`）

### Step 3：撰写四节内容

1. **第一节 - 形势**（25%）：提炼 1-2 条行业趋势、1-2 条竞争对比、1-2 条监管动向，每条配合一个关键数字
2. **第二节 - 进展**（35%）：展示里程碑/KPI/重点项目，用 ✅⚠️❌ 标识状态，使结果一目了然
3. **第三节 - 风险**（25%）：用风险矩阵表格展示，标注等级和当前应对措施，总结整体风险敞口
4. **第四节 - 决策项**（15%）：列出 ≤3 个决策项，每项配至少 2 个选项 + 推荐方案 + 理由

### Step 4：精简与校准

1. 将全文压缩至 ≤2 页 A4 纸的内容量（约 800-1000 字核心内容）
2. 替换所有技术术语为商业语言（这是最容易被忽略但最关键的一步）
3. 确保每条论点后有数字支撑
4. 检查每个决策项是否有明确的选择和推荐

### Step 5：制作汇报辅助材料

1. 输出 30 秒电梯演讲版本（开场 30 秒讲清核心）
2. 输出 PPT 建议结构（避免用户直接贴文字到 PPT，给主题+要点格式）
3. 输出常见刁钻问题 3-5 个及其应对口径（如"数据投入这么多年为什么还没看到收入？"）

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "帮我写一份董事会汇报材料" | 先确认汇报目的和时间限制，然后采集四节素材，按模板撰写 |
| "这个议题适合上董事会吗？" | 用"是否需要董事会决策"来判定：纯信息同步建议走邮件或月报渠道，需要资源/政策/预算决策的才上董事会 |
| "董事会都是非技术背景怎么讲？" | 严格执行"商业语言"原则，避免技术术语；用收入/成本/风险/竞争框架组织内容；每页不多于 3 个数字 |
| "时间很紧只有 10 分钟？" | 压缩第二节（进展），强化第四节（决策项），第一节形势用 1 句话带过 |
| "风险部分会不会让董事会紧张？" | 风险部分不是展示问题，而是展示"我们已发现问题且已有应对措施"的管控能力。可附带一个"风险缓解进度"表 |
| "决策项怎么写才能让董事会批准？" | 每个决策项给出 2-3 个选项（而非只是一个），让董事会有选择感；推荐选项中突出 ROI 和风险收益比 |
| "需要什么附件和补充材料？" | 建议：详细战略画布、ROI 模型报告、DCMM 评分详情、完整风险登记册作为附件而非正文 |
