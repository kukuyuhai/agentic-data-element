# 💹 data-roi-model

## When to Use

当以下场景出现时调用本技能：

- **申请数据预算**：向财务委员会或 CFO 提交数据相关投入预算时，需要量化投入产出比证明资金合理性
- **复盘投入产出比**：季度/年度结束后，评估数据投入是否产生预期回报，指导下一周期预算分配
- **数据项目立项论证**：新建数据平台、购买数据产品、组建数据团队前的经济效益分析
- **战略资源决策**：在多个数据项目之间分配有限预算时，基于 ROI 做优先级排序
- **合规投入论证**：因监管要求产生的被动数据投入（如个人信息保护整改），需要证明其必要性

## 工作原则

1. **全面计量**：投入和产出均应覆盖直接和间接、当期和预期、可计量和难计量等多维度
2. **审慎归因**：产出归因需区分因果关系和相关性，避免将业务自然增长误判为数据投入的贡献
3. **动态更新**：ROI 模型不是一次性产物，应随投入产出数据积累持续校准
4. **避免过度金融化**：数据价值有其战略维度不可完全货币化，应保留定性分析补充
5. **标准溯源**：所有成本归集和资本化判断需溯源至 `T/MIITEC 025-2024 §5.1.1`

## ROI 模型框架

### 一、投入维度 (Input / Cost)

| 投入大类 | 明细项 | 计量方法 | 计量周期 |
|---------|-------|---------|---------|
| 人力成本 | 数据工程师、分析师、治理岗、CDO 团队等薪资福利 | 岗位数 × 平均薪酬（含社保公积金） | 年度 |
| 工具平台 | 数据目录、ETL、质量监控、血缘分析、BI 等 License + 实施 + 运维 | 合同金额 (CAPEX) + 年度运维 (OPEX) | 按合同周期分摊 |
| 合规成本 | 等保测评、个人信息保护审计、分类分级整改、合规咨询 | 按次/按年外部报价 | 年度或按需 |
| 培训与能力建设 | 数据素养培训、DCMM 贯标、认证考试 | 培训费 + 认证费 + 人力机会成本 | 年度 |
| 数据采购 | 外购数据、API 订阅、数据标注外包 | 合同金额 | 按订阅周期 |

### 二、产出维度 (Output / Benefit)

| 产出大类 | 明细项 | 量化方法 | 归因难度 |
|---------|-------|---------|---------|
| 直接收入 | 数据产品/服务销售收入、数据资产交易收入 | 财务系统中的数据相关收入科目 | ★☆☆（低） |
| 降本增效 | 报表自动化节省的人力、数据共享节约的重复采购 | 节省工时 × 人天成本 + 避免的重复采购额 | ★★☆（中） |
| 风控收益 | 提前预警减少的损失、反欺诈避免的资损 | 模型上线前后的损失率差值 × 业务规模 | ★★★（高） |
| 战略价值 | 数据驱动的精准决策、客户体验提升、合规风险降低 | 管理层定性评估 + 替代性量化指标 | ★★★（高） |

### 三、归因方法

由于数据投入的产出往往不是直接可见的，必须采用归因方法区分因果关系和相关性：

**1. 直接归因**：适用于数据产品销售、专项数据服务等可以直接标明"由数据团队产出"的场景。收入可直接计入数据团队账面。

**2. 间接归因**：适用于报表自动化、数据平台支撑等场景。采用以下方法：
- **贡献度折算法**：数据团队贡献权重 = 数据要素占总产出的占比（由业务侧评估确认）
- **替代成本法**：若无数据工具，业务侧需额外投入多少人力和时间（以市场替代成本计量）

**3. 对比实验**：适用于风控模型、推荐算法等场景。采用 A/B 测试或前后对比：
- 对照组（无数据优化）vs 实验组（有数据优化）的效果差值
- 需排除季节性、营销活动等混淆变量影响

### 四、ROI 计算公式

**基础公式**：

```
ROI = (Σ产出_i - Σ投入_j) / Σ投入_j × 100%
```

**分阶段公式**（考虑投入的前置性）：

| 阶段 | 公式 | 说明 |
|-----|------|------|
| 建设期（Year 1） | ROI₁ = (当期产出 - 当期投入) / 当期投入 | 该阶段产出通常为负或极低 |
| 成长期（Year 2） | ROI₂（累计）= (累计产出 - 累计投入) / 累计投入 | 开始出现正向产出 |
| 成熟期（Year 3+） | ROI₃⁺（累计）= (累计产出 - 累计投入) / 累计投入 | 基础设施摊销完成，产出放大 |

**关键提示**：数据投入具有典型的前置性和滞后性。建议至少以 3 年为周期评估 ROI，第一年的 ROI 通常为负值属于正常现象。

### 五、敏感性分析

识别影响 ROI 最大的单一变量：

1. 对每个投入/产出项设定 ±20% 的波动区间
2. 计算各变量波动时 ROI 的变化幅度
3. 识别高敏感性变量（ROI 变化 > 15%）

**示例**：

| 变量 | 基线 | -20% 时 ROI | +20% 时 ROI | 敏感性 |
|-----|------|------------|------------|-------|
| 人力成本 | ¥800万 | 35% | 8% | 高 |
| 直接收入 | ¥1,200万 | 5% | 42% | 高 |
| 工具采购 | ¥300万 | 28% | 18% | 中 |

对高敏感性变量的对策：人力成本 → 弹性用工（外包+自雇混合）；直接收入 → 多产品线分摊风险。

## Steps

### Step 1：采集投入数据

1. 引导用户按五大类（人力、工具、合规、培训、采购）逐项填报年度投入金额
2. 区分 CAPEX 和 OPEX，建议工具平台按使用寿命分摊
3. 确认投入金额的口径（含税/不含税、实际支付/承诺金额）
4. 若用户无法精确填报，提供行业参考比例：人力 40-50%、工具 25-35%、合规 10-15%、培训 5-10%

### Step 2：采集产出数据

1. 引导用户按四大类（直接收入、降本增效、风控收益、战略价值）填报产出
2. 对降本增效类产出，引导量化节省的人力和采购成本
3. 对风控收益类产出，建议获取对照数据（优化前后的损失率对比）
4. 对战略价值类产出，以管理层定性评估为主，辅以客户满意度等替代指标

### Step 3：归因分析与调整

1. 对每项产出选择合适的归因方法（直接归因/间接归因/对比实验）
2. 对间接归因项，与业务侧共同确认贡献度权重
3. 对对比实验项，检查是否排除了混淆变量
4. 根据归因分析结果，对产出金额进行调整（如贡献度权重 = 60%，则调整后产出 = 原始产出 × 60%）

### Step 4：ROI 计算

1. 计算当期 ROI 和累计 ROI（建议分年度展示）
2. 估算回本周期（累计投入/年均产出）
3. 分阶段（建设期/成长期/成熟期）标注不同特征
4. 如用户需要，可调用 `role/chief-data-officer/data-strategy-canvas` 将 ROI 结果嵌入战略画布

### Step 5：敏感性分析与建议

1. 选择 3-5 个高影响变量执行 ±20% 敏感性测试
2. 识别高敏感性变量并制定风险缓解建议
3. 输出最终 ROI 报告和预算调整建议

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "怎么算数据投入的 ROI？" | 展示四大投入维度和四类产出维度，引导用户逐项填报，然后计算 |
| "数据投入的产出怎么衡量？" | 区分直接收入和间接效益，介绍三种归因方法，帮助用户选择合适的归因路径 |
| "投入了很多年但看不到效果，正常吗？" | 说明数据投入的前置性和滞后性特征，建议以 3 年为评估周期，分阶段展示 ROI 变化趋势 |
| "风控收益怎么量化？" | 引导获取优化前后的损失率数据，用差值 × 业务规模计算，注意排除混淆变量 |
| "怎么看哪些投入是值得的？" | 按 ROI 从高到低排序各项投入，结合敏感性分析识别高杠杆投入项 |
| "怎么向 CFO 证明预算合理性？" | 输出结构化 ROI 报告：投入→产出→归因→敏感性→建议，建议一页纸摘要格式 |
| "培训投入有必要计入吗？" | 必须计入。数据能力建设核心在人，培训 ROI 虽然计量困难但长期回报显著（参考：1 元培训投入 ≈ 3-5 元效率提升） |
