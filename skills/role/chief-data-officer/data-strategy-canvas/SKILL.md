# 🗺️ data-strategy-canvas

## When to Use

当以下场景出现时调用本技能：

- **年度数据战略制定**：CDO 需要在年初输出全年数据战略蓝图，驱动组织数据能力建设
- **董事会/高管汇报前准备**：需将数据战略压缩为高层易懂的画布语言，支撑资源争取和战略对齐
- **数据治理体系规划**：新建或重构企业数据治理框架时，需要端到端的战略视图
- **跨部门数据责任划分**：业务部门、技术部门、合规部门之间的数据责任需要清晰界定
- **预算审批支持**：向财务委员会或 CFO 申请数据相关预算时提供结构化论证

## 工作原则

1. **愿景驱动**：一切行动从数据愿景出发，确保每个投入都有明确的战略指向
2. **可衡量**：所有关键结果必须有量化指标，禁止"提升数据质量"之类不可衡量的表述
3. **风险意识**：在设定目标时必须识别和标注技术、合规、组织三类风险
4. **责任闭环**：每个关键结果必须对应明确的负责主体（RACI 中的 A 角色）
5. **标准锚定**：所有能力判断需溯源至 `T/MIITEC 025-2024` 和 `GB/T 36073-2018` 相关条款

## 战略画布六要素

### 1. 愿景 (North Star)

定义企业数据战略的终极方向。需回答："3年后，数据在企业中扮演什么角色？"

**模板**：
```
通过数据[核心动作]，实现[业务/客户/效率/风控]等[价值维度]的[目标描述]。
```

**示例**："通过数据资产化运营，实现数据收入占比达到总营收的 15%，风控准确率提升至 99.5%。"

**关键检查**（`GB/T 36073-2018 §5.1`）：
- 愿景是否与组织业务战略一致？
- 是否覆盖数据战略八大能力域的核心目标？
- 是否可向下分解为可执行的年度目标？

### 2. 北极星指标 (One Metric)

在众多指标中选择唯一一个最能衡量数据战略推进效果的指标。该指标需满足：
- 可量化：有明确的数值和单位
- 可归因：指标变化可直接关联到数据战略行动
- 可追踪：有数据源支撑，可按月/季采集

**示例**："数据资产目录覆盖率（已编目数据资产数 / 全量数据资产数），年度目标从 40% 提升至 85%。"

### 3. 关键结果 (OKR)

围绕北极星指标分解 3-5 个关键结果，每个 KR 需包含：
- 目标值（含基线与达标线）
- 截止时间
- 衡量方式（数据来源和计算公式）
- 对应责任人

**模板**：

| KR | 基线值 | 目标值 | 截止时间 | 衡量方式 | 负责人 |
|----|-------|-------|---------|---------|-------|
| 数据目录覆盖率 | 40% | 85% | Q4 | 编目资产数/全量资产数 | 数据资产管理团队 |
| 核心报表自动化率 | 30% | 90% | Q3 | 自动生成的报表数/总报表数 | 数据平台团队 |
| DCMM 等级 | 2级 | 3级 | Q4 | GB/T 36073-2018 评估 | 数据治理委员会 |

**标准锚定**（`T/MIITEC 025-2024 §5.1.1`）：组织应建立与战略目标匹配的数据管理能力评估机制，每年至少开展一次自评估。

### 4. 责任矩阵 (RACI)

明确关键结果的责任分配。对每个 KR/行动项标注：

- **R** (执行者)：具体执行任务的角色
- **A** (审批者)：对结果最终负责的角色（每项有且仅有一个 A）
- **C** (咨询者)：需征求意见的角色
- **I** (知会者)：需被告知进度的角色

**模板**：

| 关键结果 | CDO | 数据治理委员会 | 数据平台团队 | 业务部门 | 合规法务 |
|---------|-----|-------------|-----------|--------|--------|
| 数据目录覆盖 85% | A | C | R | C | I |
| 自动化报表达 90% | A | I | R | C | I |
| DCMM 达 3 级 | A | R | C | C | C |

若责任矩阵复杂，可调用 `management/stakeholder-raci` 生成详细的干系人 RACI 矩阵。

### 5. 预算 (Budget)

按三个维度编制预算：

| 预算维度 | 内容 | 估算方法 |
|---------|------|---------|
| 人力成本 | 数据工程师、数据分析师、数据治理岗等 | 岗位数 × 市场薪酬 + 培训费 |
| 工具平台 | 数据目录、质量监控、血缘分析、ETL 等 | License + 实施 + 运维费 |
| 合规成本 | 等保测评、合规咨询、个人信息保护审计等 | 按次/按年外采报价 |

**注意事项**：
- 人力成本包含内部转岗和外部招聘两类
- 工具平台优先考虑已有基础设施延展，避免重复建设
- 合规成本按监管要求预留 15-20% 弹性空间

可调用 `role/chief-data-officer/data-roi-model` 将预算映射到 ROI 框架中验证投入合理性。

### 6. 风险 (Risk)

从三个维度识别战略执行风险：

**技术风险**：数据平台稳定性、技术选型变更、数据迁移失败、供应商锁定等
**合规风险**：个人信息保护新规、数据出境审核、版权争议、分类分级偏差等
**组织风险**：关键人才流失、部门协作摩擦、高层支持力度变化等

**风险评估模板**：

| 风险项 | 类型 | 概率 | 影响 | 缓解措施 |
|-------|------|------|------|---------|
| 核心数据工程师离职 | 组织 | 中 | 高 | 建立梯队、文档化、知识转移 |
| 数据分类分级不合规 | 合规 | 低 | 高 | 提前合规审计、聘请外部顾问 |
| 平台迁移数据丢失 | 技术 | 低 | 高 | 备份策略、灰度迁移、回滚方案 |

## Inputs

| 输入项 | 类型 | 必填 | 说明 | 示例 |
|-------|------|------|------|------|
| `current_state` | object | 是 | 当前数据管理现状描述 | `{"dcmm_level": 2, "catalog_coverage": "40%", "platform_maturity": "medium"}` |
| `business_strategy` | string | 是 | 企业年度业务战略简述 | "2026 年聚焦客户增长和风控升级" |
| `stakeholder_map` | object | 否 | 主要干系人及角色（可调用 stakeholder-raci 获取） | `{"cdo": "张三", "cto": "李四", "cfo": "王五"}` |
| `budget_ceiling` | number | 否 | 可支配预算上限（万元） | `1200` |
| `target_period` | string | 否 | 战略周期 | `2026-2028`，默认年度 |

## Steps

### Step 1：采集现状与战略上下文

1. 引导用户提供当前数据管理现状（可调用 `management/dcmm-scoring` 获取 DCMM 评估结果）
2. 确认企业业务战略方向和年度优先级
3. 识别关键干系人（可调用 `management/stakeholder-raci` 生成完整干系人地图）
4. 确认预算上限和时间周期

### Step 2：愿景提炼与北极星指标共识

1. 基于业务战略，引导用户用一句话描述数据愿景
2. 检查愿景是否满足 `GB/T 36073-2018 §5.1` 的战略对齐要求
3. 从数据价值链（采集→治理→分析→应用→变现）中识别最关键的转化环节
4. 与用户共同选定北极星指标，验证其可量化、可归因、可追踪性

### Step 3：关键结果分解

1. 将北极星指标拆解为 3-5 个关键结果（KR）
2. 为每个 KR 设定基线值、目标值、截止时间
3. 确定每个 KR 的数据源和计算公式
4. 明确每个 KR 的负责人（A 角色）

### Step 4：RACI 矩阵绘制

1. 列出所有 KR/行动项
2. 识别涉及的角色（CDO、数据治理委员会、平台团队、业务部门、合规法等）
3. 为每个 KR×角色交叉点标注 R/A/C/I
4. 检查确保每一项有且仅有一个 A

### Step 5：预算编制

1. 按人力、工具、合规三个维度逐项列支
2. 与预算上限校准，超出部分提出优先级排序建议
3. 标注可弹性调整的项目

### Step 6：风险识别与缓解

1. 从技术、合规、组织三个维度系统识别风险
2. 评估每个风险的概率和影响（高/中/低）
3. 为每个高风险项制定缓解措施和应急预案

### Step 7：画布整合与汇报准备

1. 将六要素整合为一页纸战略画布
2. 生成董事会级别的摘要版本（可调用 `role/chief-data-officer/boardroom-brief`）
3. 输出标准溯源清单，标注每条判断对应的标准条款

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "帮我制定年度数据战略" | 进入 Step 1，依次采集现状、业务战略、干系人信息，逐步填充六要素 |
| "画布是什么？怎么填？" | 解释六要素分别是什么，展示模板，引导用户逐个填写 |
| "北极星指标怎么选？" | 从数据价值链出发，引导用户识别最关键的转化环节，候选 2-3 个指标让用户选定 |
| "RACI 和普通分工表有什么区别？" | 解释 R/A/C/I 定义，强调"每项有且仅有一个 A"原则，输出 RACI 矩阵 |
| "预算怎么分项编制？" | 按人力/工具/合规三维度提供参考模板，结合企业规模和行业给出比例建议 |
| "风险怎么评估？" | 引导从技术、合规、组织三个维度逐一排查，用概率×影响矩阵输出风险等级 |
| "这个画布和董事会汇报什么关系？" | 说明画布是完整战略文档，可调用 `boardroom-brief` 压缩为董事会简报格式 |
