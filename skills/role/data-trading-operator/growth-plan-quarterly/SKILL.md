# 📈 growth-plan-quarterly

季度平台运营计划——GMV 目标分解、供需双侧规划、生态活动与预算编排。

## When to Use

- 每季度启动前（通常为上季度末月 15-25 日），制定下季度运营计划时
- 平台战略调整，需要重新规划季度运营方向时
- 季度中需要根据实际表现调整计划时（中期调整）
- 新人接手运营工作，需要了解季度运营全貌时

## 工作原则

1. **目标可拆解**：GMV 目标必须分解到月度、品类，避免"大而空"的目标
2. **供需双侧**：计划必须同时覆盖供给侧（卖方/产品）和需求侧（买方/订单），不可偏废
3. **数据驱动**：目标设定基于历史数据和预测（参考 `gmv-forecast-holt-winters`），而非拍脑袋
4. **预算匹配**：每项行动计划匹配对应预算，预算总额不超过季度运营预算上限
5. **可衡量可追踪**：每项目标设定明确指标和里程碑，支持周度追踪 `§5.2.3`

## 五大规划模块

### 一、GMV 目标规划

**目标分解链路：** 季度总 GMV → 月度分解 → 品类分解 → 周度跟踪

| 规划项 | 说明 | 示例 |
|-------|------|------|
| 季度总 GMV 目标 | 基于上季度实际 + 增长率预测 | 上季度 5000 万 → 本季度 6500 万（+30%） |
| 月度分解 | 考虑季节性波动，非均分 | M1: 2000万 / M2: 2200万 / M3: 2300万 |
| 品类分解 | 按品类贡献率分配目标 | 金融 40% / 医疗 25% / 交通 20% / 其他 15% |
| 增长来源 | 新增 vs 存量、新客 vs 老客 | 新客贡献 40%，老客复购贡献 60% |
| 风险缓冲 | 预留 10-15% 风险缓冲 | 目标 6500 万，内部冲刺目标 7400 万 |

**GMV 预测依据：** 调用 `gmv-forecast-holt-winters` 输出的时序预测值和置信区间。

### 二、供给侧规划

| 规划项 | 目标 | 关键动作 |
|-------|------|---------|
| 新增挂牌数 | 季度新增 ___ 款产品 | 卖方拓展、产品孵化、品类引入 |
| 活跃卖方数 | 季度活跃卖方 ___ 家 | 卖方激活、培训赋能、激励机制 |
| 供给质量 | 平均挂牌质量分 ≥ ___ 分 | 质量审核、劣质产品下架、优质奖励 |
| 品类覆盖 | 覆盖 ___ 个二级品类 | 空白品类填补、优势品类深耕 |
| 供给更新 | 活跃产品更新率 ≥ ___% | 数据更新提醒、过期产品清理 |

### 三、需求侧规划

| 规划项 | 目标 | 关键动作 |
|-------|------|---------|
| 新增买方 | 季度新增注册买方 ___ 家 | 精准获客、行业沙龙、渠道合作 |
| 活跃买方 | 季度活跃买方 ___ 家 | 买方激活、需求挖掘、1对1服务 |
| 复购率 | 季度复购率 ≥ ___% | 复购激励、满意度提升、售后优化 |
| 客单价 | 平均客单价 ≥ ___ 万元 | 高客单产品推荐、组合销售、增值服务 |
| 需求转化 | 注册→首购转化率 ≥ ___% | 新人优惠、试用体验、销售跟进 |

### 四、生态活动规划

| 活动类型 | 活动名称 | 时间 | 预期效果 | 预算 |
|---------|---------|------|---------|------|
| 促销活动 | 季度数据采购节 | M1 第2周 | GMV +800万 | 50万 |
| 行业沙龙 | 金融数据交易沙龙 | M2 第3周 | 新增买方30家 | 15万 |
| 卖方培训 | 数据产品上架培训 | M1 第1周 | 新增挂牌50款 | 5万 |
| 买方对接会 | 供需匹配对接会 | M3 第2周 | 成交2000万 | 10万 |
| 平台推广 | 行业大会参展 | M2 第4周 | 品牌曝光+获客 | 20万 |

### 五、预算规划

| 预算项 | 金额（万元） | 占比 | 说明 |
|-------|:----------:|:----:|------|
| 人员成本 | ___ | 40% | 运营团队人力 |
| 营销推广 | ___ | 30% | 广告投放、活动、渠道 |
| 技术支持 | ___ | 15% | 系统维护、工具采购 |
| 生态建设 | ___ | 10% | 培训、沙龙、补贴 |
| 应急储备 | ___ | 5% | 突发需求预留 |
| **合计** | **___** | **100%** | — |

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 上季度运营数据 | Document | 上季度 GMV、活跃数、转化率等实际值 |
| GMV 预测报告 | Document | `gmv-forecast-holt-winters` 输出的预测值 |
| 上季度复盘 | Document | `campaign-retrospective` 输出的复盘报告 |
| 看板数据 | Document | `platform-metric-dashboard` 当前指标 |
| 预算上限 | Float | 季度运营预算总额 |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 季度运营计划书 | Document | 五大模块完整规划 |
| 月度目标卡 | Document | M1/M2/M3 月度目标分解 |
| 活动排期表 | Document | 季度内所有活动的时间排期 |
| 预算明细表 | Document | 各项预算分配明细 |

## Steps

1. 收集上季度运营数据、GMV 预测、看板指标等输入
2. 设定季度 GMV 总目标，分解到月度和品类
3. 制定供给侧规划（新增挂牌、活跃卖方、质量目标）
4. 制定需求侧规划（新增买方、活跃买方、复购目标）
5. 规划生态活动（促销、培训、沙龙、推广）
6. 编制预算分配方案，确保总预算不超上限
7. 设定里程碑和周度跟踪指标
8. 评审通过后输出正式计划书 `§5.2.3`
9. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| 季度 GMV 目标怎么定？ | 基于上季度实际值 + GMV 预测（Holt-Winters/Prophet），结合增长率目标设定 |
| 供给和需求哪个更重要？ | 同等重要，供需双侧必须同步规划，避免有供给无需求或有需求无供给 |
| 预算不够怎么办？ | 优先保核心活动（高ROI），削减低效活动，申请追加预算或调整目标 |
| 季度中目标完不成怎么办？ | 每周追踪达成率，偏离 >15% 时启动中期调整，重新分配资源 |
| 活动效果怎么评估？ | 活动结束后调用 `campaign-retrospective` 执行复盘，评估 GMV/ROI/参与度 |
