# 📅 数据资产摊销计划表

## When to Use

当数据资产已确认为**无形资产**并完成初始计量后，需要生成年度摊销计划表时使用本技能。核心目标：根据摊销方法、摊销年限、残值等因素，输出完整的年度摊销金额表与账面价值曲线，支撑年度结账与财务报表编制。

典型触发场景：
- 数据资产入表后的首年摊销计划制定
- 年度结账时的摊销金额计算
- 摊销方法/年限变更后的计划重算
- 减值测试后的摊销调整

## 工作原则

1. **方法一致性**：同一数据资产在存续期内保持摊销方法一致，变更须有充分理由并追溯调整。
2. **年限合理性**：摊销年限须综合考量数据时效性、业务周期、法律期限、技术迭代等因素，不得随意设定。
3. **残值审慎性**：数据资产残值一般为零，除非有合同约定或活跃市场可证明残值存在。
4. **减值联动**：每年至少一次减值测试，触发条件后须重新计算摊销计划。
5. **准则溯源**：每条结论附 `§第六条` 等标准溯源标注。

## 摊销方法

### 1. 直线法（默认推荐）

每期摊销金额相等，适用于使用效益均匀分布的数据资产。

**公式：**
```
年摊销额 = (初始成本 - 残值) / 摊销年限
期末账面价值 = 期初账面价值 - 年摊销额
```

### 2. 加速法（余额递减法）

前期摊销金额较大，后期递减，适用于数据价值衰减较快、前期效益较高的资产。

**公式：**
```
年摊销额 = 期初账面价值 × 加速率
加速率 = 2 / 摊销年限 （双倍余额递减法）
```

> 注意：加速法最后两年须转为直线法，确保账面价值收敛至残值。

## 摊销年限确定因素

| 影响因素 | 说明 | 示例 |
|---------|------|------|
| 数据时效性 | 数据内容的有效期限 | 实时交易数据：1-3年；历史档案：10年以上 |
| 业务周期 | 数据资产服务的业务运营周期 | 营销数据：2-5年；基础设施数据：5-10年 |
| 法律期限 | 合同/法规约定的使用期限 | 授权数据集按合同年限；隐私数据按法规期限 |
| 技术迭代 | 技术更新导致的淘汰速度 | AI训练数据：3-5年；基础地理数据：10年以上 |

**综合判定原则：** 取上述四因素中**最短**的期限作为摊销年限上限，确保不高估资产价值。

## 残值确定

| 场景 | 残值判定 | 依据 |
|------|---------|------|
| 无合同约定且无活跃市场 | 残值 = 0 | CAS 6 默认处理 |
| 有明确处置合同 | 残值 = 合同约定处置价 | 合同条款 |
| 有活跃市场可比价格 | 残值 = 预计处置时市场价 | 市场法参考 |

> 数据资产残值通常为零，因数据不具有实物损耗特性，但技术淘汰会导致价值归零。

## 减值测试触发条件

| 触发条件 | 说明 | 处理 |
|---------|------|------|
| 市场环境重大变化 | 数据资产所处市场/技术环境发生重大不利变化 | 立即减值测试 |
| 使用方式重大变化 | 持有目的或使用方式发生变更 | 重新评估可收回金额 |
| 法律法规变化 | 新法规限制数据使用 | 重新评估剩余年限与价值 |
| 账面价值高于可收回金额 | 账面价值 > 公允价值-处置费用 或 > 使用价值 | 计提减值准备 |
| 年度强制测试 | 每年年末至少进行一次减值测试 | 常规年度减值检查 |

## 年度摊销计划表示例

**示例参数：** 初始成本 200 万元，摊销年限 5 年，残值 0，直线法

| 年度 | 期初账面价值 | 年摊销额 | 累计摊销 | 期末账面价值 |
|------|------------|---------|---------|------------|
| 第1年 | 200.00 | 40.00 | 40.00 | 160.00 |
| 第2年 | 160.00 | 40.00 | 80.00 | 120.00 |
| 第3年 | 120.00 | 40.00 | 120.00 | 80.00 |
| 第4年 | 80.00 | 40.00 | 160.00 | 40.00 |
| 第5年 | 40.00 | 40.00 | 200.00 | 0.00 |

**加速法对照（双倍余额递减，最后两年转直线法）：**

| 年度 | 期初账面价值 | 年摊销额 | 累计摊销 | 期末账面价值 |
|------|------------|---------|---------|------------|
| 第1年 | 200.00 | 80.00 | 80.00 | 120.00 |
| 第2年 | 120.00 | 48.00 | 128.00 | 72.00 |
| 第3年 | 72.00 | 28.80 | 156.80 | 43.20 |
| 第4年 | 43.20 | 21.60 | 178.40 | 21.60 |
| 第5年 | 21.60 | 21.60 | 200.00 | 0.00 |

## 账面价值曲线

直线法呈线性下降，加速法呈凸函数下降（前期降幅大）。选择方法时须匹配数据资产的价值消耗模式：

- 价值均匀消耗 → 直线法
- 前期价值消耗快 → 加速法
- 难以判断 → 默认直线法

## Inputs

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `asset_name` | string | 是 | 数据资产名称 |
| `initial_cost` | float | 是 | 初始成本（万元） |
| `useful_life_years` | int | 是 | 摊销年限 |
| `residual_value` | float | 否 | 残值（默认 0） |
| `amortization_method` | enum: `straight_line` / `declining_balance` | 否 | 摊销方法（默认 `straight_line`） |
| `acceleration_rate` | float | 否 | 加速率（默认 2.0，即双倍） |
| `start_date` | date | 是 | 开始摊销日期 |
| `impairment_trigger` | boolean | 否 | 是否触发减值测试 |

## Outputs

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `amortization_schedule` | array[object] | 年度摊销计划表 |
| `annual_amortization` | float | 每年摊销额（直线法） |
| `book_value_curve` | array[float] | 账面价值曲线 |

## Steps

1. **读取输入参数**：获取资产名称、初始成本、年限、残值、方法
2. **校验参数合理性**：年限 > 0、成本 > 残值、加速率 > 0
3. **确定摊销方法**：未指定时默认直线法
4. **计算年度摊销额**：
   - 直线法：`(成本 - 残值) / 年限`
   - 加速法：`期初账面 × 加速率 / 年限`，最后两年转直线法
5. **生成摊销计划表**：逐年计算期初账面、摊销额、累计摊销、期末账面
6. **生成账面价值曲线**：提取各年末账面价值形成曲线
7. **减值测试检查**：检查触发条件，必要时提示减值测试
8. **输出分录模板**：生成年度摊销会计分录
9. **标准溯源**：附 `§第六条` 标注

## 常见问法与处理策略

| 常见问法 | 处理策略 | 输出要点 |
|---------|---------|---------|
| "这批数据摊几年？" | 综合四因素判定最短期限 | 推荐年限 + 依据 |
| "用直线法还是加速法？" | 匹配价值消耗模式 | 推荐方法 + 理由 |
| "数据有残值吗？" | 默认残值为零，除非有合同/市场证据 | 残值判定 + 依据 |
| "今年摊销额多少？" | 查表输出对应年度摊销额 | 年度摊销额 + 分录 |
| "数据是不是该减值了？" | 检查减值触发条件 | 是否触发 + 可收回金额建议 |
| "摊销方法改了怎么办？" | 追溯调整，重新生成计划表 | 调整后计划 + 差异说明 |
| "年限比合同短怎么办？" | 取最短期限作为上限 | 修正年限 + 重算 |
| "年末账面价值是多少？" | 查表输出对应年度期末账面 | 期末账面价值 |
