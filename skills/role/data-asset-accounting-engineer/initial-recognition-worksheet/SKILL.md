# 📓 初始确认工作底稿

## When to Use

当新建数据资产需要判断是否满足入表条件、并进行初始确认时使用本技能。核心目标：依据财会〔2023〕11号第二条、第四条，系统化完成数据资产的可确认性判断、可靠计量、未来经济利益评估与成本归集明细，输出标准化初始确认工作底稿。

典型触发场景：
- 新建数据资产入表前的确认判断
- 数据资源开发完成后的资本化评估
- 审计/年报编制时补充初始确认底稿
- 外购数据资产的入表判断

## 工作原则

1. **五条件全满足**：可确认性判断须五条件全部满足，缺一不可。
2. **成本可追溯**：成本归集须有完整的原始凭证和费用分配依据。
3. **经济利益可量化**：未来经济利益评估须有量化分析，不能仅凭定性判断。
4. **底稿可审计**：工作底稿须完整保留，支持审计追溯。
5. **准则溯源**：每条结论附 `§第二条`、`§第四条` 等标准溯源标注。

## 工作底稿结构

### 第一部分：可确认性判断（五条件）

数据资产确认为资产须同时满足以下五个条件：

| 序号 | 确认条件 | 判定标准 | 判定结果 | 依据 |
|------|---------|---------|---------|------|
| 1 | 合法性 | 数据来源合法合规，已取得合法授权 | ✅/❌ | 数据来源审查文件 |
| 2 | 权属清晰 | 企业对该数据资源拥有所有权或控制权 | ✅/❌ | 权属证明/控制权说明 |
| 3 | 可辨认性 | 数据资源可从企业中分离或划分出来 | ✅/❌ | 可分离性分析 |
| 4 | 经济利益 | 很可能带来未来经济利益 | ✅/❌ | 经济利益评估报告 |
| 5 | 成本可计量 | 数据资源的成本能够可靠计量 | ✅/❌ | 成本归集明细表 |

**判定逻辑：** 五条件全部满足 → 可确认为资产；任一不满足 → 不可确认，费用化处理。

### 第二部分：可靠计量（成本归集）

成本归集分为四大类：

**2.1 直接成本**
- 数据采购费用：外购数据集的购买价款
- 采集费用：自采数据的采集人员薪酬、设备费用
- 数据加工费用：清洗、标注、结构化等加工人员薪酬

**2.2 开发成本**
- 开发人员薪酬：数据平台/算法开发人员工资
- 开发工具费用：专用软件/工具采购费
- 资本化条件判断：研究阶段→费用化；开发阶段→满足条件资本化

**2.3 基础设施费用**
- 存储费用：数据存储设备/云存储费用分摊
- 计算费用：算力资源费用分摊
- 网络费用：数据传输/网络带宽费用分摊

**2.4 维持费用**
- 日常维护：数据更新、质量维护人员薪酬
- 折旧分摊：相关固定资产折旧分摊
- 合理费用：为保持数据资产可用性发生的必要支出

### 第三部分：未来经济利益评估

| 评估维度 | 评估方法 | 量化指标 |
|---------|---------|---------|
| 自用场景 | 成本节约法/收益增量法 | 预计年节约成本/增加收益 |
| 对外服务 | 服务收入预测法 | 预计年服务收入 |
| 交易场景 | 市场比较法 | 预计交易价格区间 |
| 综合评估 | 加权平均 | 各场景加权经济利益 |

**经济利益可能性判断：**
- 很可能（>50%）：满足确认条件
- 可能（30%-50%）：需附加证据
- 极小可能（<30%）：不满足确认条件

### 第四部分：成本归集明细表模板

| 成本项目 | 明细说明 | 金额（万元） | 费用类型 | 资本化/费用化 | 凭证编号 |
|---------|---------|------------|---------|-------------|---------|
| 数据采购 | XX数据集购买费用 | 50.00 | 直接成本 | 资本化 | FP-001 |
| 采集人员 | 采集团队3人×6月薪酬 | 36.00 | 直接成本 | 资本化 | FP-002 |
| 数据清洗 | 清洗标注人员薪酬 | 24.00 | 直接成本 | 资本化 | FP-003 |
| 平台开发 | 数据平台开发人员薪酬 | 40.00 | 开发成本 | 资本化 | FP-004 |
| 云存储 | 12个月存储费用分摊 | 15.00 | 基础设施 | 资本化 | FP-005 |
| 算力资源 | 训练算力费用分摊 | 20.00 | 基础设施 | 资本化 | FP-006 |
| 日常维护 | 年度维护人员费用 | 15.00 | 维持费用 | 费用化 | FP-007 |
| **合计** | — | **200.00** | — | **资本化185/费用化15** | — |

## Inputs

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `asset_name` | string | 是 | 数据资产名称 |
| `data_source` | enum: `purchased` / `self_collected` / `developed` / `donated` | 是 | 数据来源类型 |
| `legality_docs` | array[string] | 是 | 合法性证明文件列表 |
| `ownership_proof` | object | 是 | 权属证明信息 |
| `separability_analysis` | string | 是 | 可辨认性分析说明 |
| `economic_benefit_assessment` | object | 是 | 经济利益评估（场景+金额+可能性） |
| `cost_collection_detail` | array[object] | 是 | 成本归集明细列表 |
| `capitalization_conditions` | object | 否 | 开发支出资本化条件判断（开发类适用） |

## Outputs

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `recognition_eligible` | boolean | 是否满足初始确认条件 |
| `five_conditions_result` | array[object] | 五条件逐条判定结果 |
| `total_capitalized_cost` | float | 资本化总成本 |
| `total_expensed_cost` | float | 费用化总成本 |
| `cost_breakdown` | object | 成本归集明细表 |
| `journal_entry` | object | 会计分录 |
| `worksheet_document` | string | 完整工作底稿文本 |
| `next_step_skill` | string | 下一步衔接技能路径 |

## Steps

1. **读取资产基本信息**：名称、来源类型、相关证明文件
2. **五条件逐条判定**：
   - 合法性 → 检查 `legality_docs`
   - 权属 → 检查 `ownership_proof`
   - 可辨认性 → 检查 `separability_analysis`
   - 经济利益 → 检查 `economic_benefit_assessment`
   - 成本可计量 → 检查 `cost_collection_detail`
3. **判定结果汇总**：五条件是否全部满足
4. **成本归集与分类**：按四类（直接/开发/基础设施/维持）归集
5. **资本化/费用化判定**：开发支出按资本化条件判断
6. **汇总成本**：计算资本化总额与费用化总额
7. **经济利益可能性评估**：判定"很可能"门槛
8. **生成会计分录**：根据资本化/费用化结果生成分录
9. **输出完整底稿**：生成可审计的工作底稿文本
10. **衔接后续技能**：满足确认条件 → `accounting-treatment-classifier`
11. **标准溯源**：附 `§第二条`、`§第四条` 标注

## 常见问法与处理策略

| 常见问法 | 处理策略 | 输出要点 |
|---------|---------|---------|
| "这批数据能不能入表？" | 执行五条件逐条判定 | 判定结果 + 不满足条件说明 |
| "数据来源合法吗？" | 检查合法性证明文件 | 合法性判定 + 依据 |
| "权属归我们吗？" | 检查权属证明/控制权说明 | 权属判定 + 依据 |
| "这批数据能带来多少收益？" | 执行经济利益量化评估 | 三场景评估 + 可能性判定 |
| "哪些成本可以资本化？" | 按四类归集，判断资本化条件 | 成本归集表 + 资本化/费用化分类 |
| "开发支出怎么区分研究/开发阶段？" | 按资本化条件逐条判断 | 阶段判定 + 资本化条件结果 |
| "基础设施费用怎么分摊？" | 按使用量/面积/工时合理分摊 | 分摊依据 + 分摊金额 |
| "不满足条件怎么办？" | 费用化处理，说明原因 | 费用化分录 + 原因说明 |
