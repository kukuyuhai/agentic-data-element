# 📐 计量模式选择器

## When to Use

当数据资产需要确定初始计量与后续计量所采用的计量模式时使用本技能。核心目标：依据企业会计准则——基本准则及财会〔2023〕11号，从**历史成本**、**重置成本**、**公允价值**、**可变现净值**四种模式中判定适用模式，并输出选择理由与适用条件。

典型触发场景：
- 数据资产初始确认时的计量模式选择
- 存货类数据资产期末计量（成本与可变现净值孰低）
- 无形资产类数据资产后续计量模式确认
- 会计政策变更时的计量模式重新评估

## 工作原则

1. **默认历史成本**：初始确认默认采用历史成本，除非准则另有规定。
2. **公允价值限制**：在中华人民共和国企业会计准则下，数据资产**不适用**公允价值计量模式进行后续计量（除非满足投资性房地产等特殊规定）。
3. **可变现净值仅限存货**：可变现净值模式仅适用于确认为存货的数据资产。
4. **一致性原则**：计量模式一经确定不得随意变更，变更须有充分理由并追溯调整。
5. **准则溯源**：每条判定附 `§第四条` 等标准溯源标注。

## 四种计量模式详解

### 1. 历史成本（默认·初始确认首选）

**定义：** 以取得数据资产时实际发生的成本作为计量基础。

**适用场景：**
- 数据资产初始确认（无形资产/存货）
- 成本可可靠计量且有完整成本归集记录

**计量要素：**
- 直接成本：采购价款、采集费用、加工费用
- 间接成本：分配的间接费用（基础设施折旧、管理人员薪酬等）
- 资本化条件：满足资本化条件的开发支出

**优点：** 可验证性强、客观可靠
**局限：** 不反映资产当前市场价值

### 2. 重置成本（罕见·特殊情况使用）

**定义：** 以当前重新获取相同数据资产所需支付的金额作为计量基础。

**适用场景：**
- 盘盈数据资产
- 接受捐赠数据资产（无原始成本记录）
- 历史成本无法可靠取得的特殊情况

**判定条件：** 必须同时满足：
- 历史成本无法可靠取得
- 可合理估算重新获取相同资产所需成本
- 经管理层审批

### 3. 公允价值（不适用于数据资产后续计量）

**定义：** 在计量日市场参与者在有序交易中出售资产所能收到的价格。

**重要提示：** 在中华人民共和国企业会计准则下，数据资产**不适用**公允价值模式进行后续计量。原因如下：

- 数据资产不属于以公允价值计量的资产类别（如交易性金融资产、投资性房地产等）
- 目前缺乏活跃且成熟的数据资产交易市场
- 财会〔2023〕11号未将公允价值列为数据资产的后续计量模式

**例外：** 公允价值可用于减值测试中的可收回金额计算（公允价值-处置费用），但非日常后续计量模式。

### 4. 可变现净值（仅限存货类数据资产）

**定义：** 数据资产预计售价减去预计销售费用和相关税费后的净额。

**适用场景：**
- 确认为存货的数据资产期末计量
- 成本与可变现净值孰低原则

**计算公式：**
```
可变现净值 = 预计售价 - 预计销售费用 - 相关税费
```

**计量规则：**
- 期末比较成本与可变现净值，取较低者
- 可变现净值低于成本时，计提存货跌价准备

## 适用性矩阵

| 计量模式 | 初始确认 | 后续计量 | 无形资产 | 存货 | 适用条件 |
|---------|---------|---------|---------|------|---------|
| 历史成本 | ✅ 首选 | ✅ 默认 | ✅ | ✅ | 成本可可靠计量 |
| 重置成本 | ⚠️ 特殊 | ❌ | ⚠️ | ⚠️ | 历史成本无法取得 |
| 公允价值 | ❌ | ❌ | ❌ | ❌ | 不适用数据资产 |
| 可变现净值 | ❌ | ✅ 存货专用 | ❌ | ✅ | 确认为存货且需孰低测试 |

## 计量模式决策树

```
是否为初始确认？
├── 是 ──→ 历史成本是否可可靠计量？
│          ├── 是 ──→ 【历史成本】（默认）
│          └── 否 ──→ 是否可估算重置成本？
│                     ├── 是 ──→ 【重置成本】（经审批）
│                     └── 否 ──→ 无法可靠计量，费用化处理
│
└── 否（后续计量）
     ├── 确认为无形资产 ──→ 【历史成本】+ 摊销 + 减值测试
     └── 确认为存货 ──→ 成本 vs 可变现净值孰低
                         ├── 成本 ≤ 可变现净值 ──→ 按成本计量
                         └── 成本 > 可变现净值 ──→ 按【可变现净值】计量，计提跌价准备
```

## Inputs

## Steps

1. **读取输入**：获取资产分类、计量阶段、成本信息
2. **判定计量阶段**：初始确认 or 后续计量
3. **初始确认分支**：
   - 历史成本可可靠取得 → 历史成本
   - 不可取得但重置可估算 → 重置成本（需审批）
   - 均不可得 → 费用化处理
4. **后续计量分支**：
   - 无形资产 → 历史成本 + 摊销（调用 `amortization-scheduler`）
   - 存货 → 成本 vs 可变现净值孰低
5. **可变现净值计算**（存货专用）：
   - `NRV = 预计售价 - 预计销售费用 - 相关税费`
   - 若 `NRV < 成本` → 计提跌价准备
6. **输出选择结果**：计量模式 + 准则条款 + 金额 + 分录
7. **公允价值排除说明**：明确标注公允价值不适用
8. **标准溯源**：附 `§第四条` 等标注

## 常见问法与处理策略

| 常见问法 | 处理策略 | 输出要点 |
|---------|---------|---------|
| "数据资产用什么计量模式？" | 默认历史成本，存货后续按可变现净值孰低 | 模式 + 准则依据 |
| "能不能用公允价值计量？" | 明确排除，附准则依据 | 不适用 + 原因说明 |
| "存货类数据期末怎么算？" | 成本与可变现净值孰低 | NRV 计算 + 跌价准备 |
| "没有原始成本记录怎么办？" | 评估重置成本可行性 | 重置成本 or 费用化 |
| "初始计量和后续计量一样吗？" | 区分说明两种阶段 | 初始历史成本 + 后续规则 |
| "跌价准备怎么计？" | 计算 NRV 与成本差额 | 跌价准备分录 |
| "计量模式能不能中途改？" | 一致性原则，变更需充分理由 | 变更条件 + 追溯调整说明 |
