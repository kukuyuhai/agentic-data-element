# 📚 数据资产会计处理判定器

## When to Use

当数据资产已完成初始确认判断、需要确定其入表科目落位时使用本技能。核心目标：依据财会〔2023〕11号及对应会计准则，判定数据资产应归入**无形资产**、**存货**还是**长期待摊费用/费用化**，并给出对应的会计处理建议。

典型触发场景：
- 数据资产入表前的科目判定
- 数据资产持有目的发生变化时的科目重分类
- 年末审计/年报编制时的科目复核

## 工作原则

1. **业务实质优先**：以企业持有数据资产的**业务实质**（自用 vs 持有出售）作为首要判定依据，而非仅依据合同形式。
2. **准则溯源**：每条判定结论须附标准条款溯源标注（形如 `§第二条`、`§第六条`），确保可审计。
3. **风险前置**：对于判定边界模糊的案例，强制提示风险并建议人工复核。
4. **交叉引用**：判定完成后自动衔接后续计量环节，调用相关技能完成闭环。

## 分类判定逻辑树

```
数据资产是否满足初始确认五条件？
├── 否 ──→ 费用化处理（当期损益）或长期待摊费用
│           （条件不满足，不确认为资产）
└── 是 ──→ 企业持有该数据资产的主要目的是什么？
           ├── 内部使用（生产/经营管理/对外服务）
           │     └── 确认为【无形资产】（CAS 6）
           │         ├→ 后续计量：摊销 + 减值测试
           │         └→ 调用 `amortization-scheduler` 生成摊销计划
           │
           ├── 持有待售（日常活动中持有以备出售）
           │     └── 确认为【存货】（CAS 1）
           │         └→ 后续计量：成本与可变现净值孰低
           │
           └── 兼具自用与出售
                 ├── 主要用途为自用 ──→ 无形资产
                 └── 主要用途为出售 ──→ 存货
```

## 判定标准对照表

| 判定维度 | 无形资产 | 存货 | 费用化/长期待摊 |
|---------|---------|------|----------------|
| 持有目的 | 内部使用、对外提供服务 | 持有以备出售 | 不满足确认条件 |
| 适用准则 | CAS 6 无形资产 | CAS 1 存货 | 财会〔2023〕11号 |
| 初始计量 | 成本法（历史成本） | 成本法（历史成本） | 当期费用化 |
| 后续计量 | 摊销 + 减值测试 | 成本与可变现净值孰低 | 不适用 |
| 摊销处理 | 按确定年限摊销 | 不摊销 | 不适用 |
| 报表列报 | 资产负债表—无形资产 | 资产负债表—存货 | 利润表—当期损益 |
| 附注披露 | 类别/摊销方法/年限/减值 | 类别/成本/可变现净值 | 费用化说明 |

## 各类别会计处理

### 1. 确认为无形资产

**初始确认分录：**
```text
借：无形资产——数据资源（XX数据集）
    贷：银行存款 / 应付职工薪酬 / 累计折旧等
```

**后续计量要点：**
- 按使用寿命是否确定分类：有限寿命摊销 / 不确定寿命不摊销但每年减值测试
- 摊销方法：直线法为主，加速法为辅（详见 `amortization-scheduler`）
- 每年年末进行减值测试，触发条件见下文

### 2. 确认为存货

**初始确认分录：**
```text
借：存货——数据资源（XX数据产品）
    贷：银行存款 / 应付职工薪酬 / 累计折旧等
```

**后续计量要点：**
- 期末按成本与可变现净值孰低计量
- 可变现净值 = 预计售价 - 预计销售费用和相关税费
- 计量模式选择详见 `measurement-model-selector`

### 3. 费用化/长期待摊

**费用化分录：**
```text
借：管理费用——数据资源开发费
    贷：银行存款 / 应付职工薪酬等
```

**长期待摊分录：**
```text
借：长期待摊费用——数据资源
    贷：银行存款 / 应付职工薪酬等
```

## 判定示例

**示例 1：客户画像数据集（自用）**
- 背景：某银行投入 200 万元构建客户画像数据集，用于内部风控模型训练
- 判定：持有目的为内部使用 → 确认为**无形资产**
- 分录：借 无形资产 200 万 / 贷 银行存款等 200 万

**示例 2：行业数据库产品（待售）**
- 背景：某数据服务商投入 150 万元构建行业数据库，用于对外打包出售
- 判定：持有目的为出售 → 确认为**存货**
- 分录：借 存货 150 万 / 贷 银行存款等 150 万

**示例 3：探索性数据采集（未达确认条件）**
- 背景：某公司投入 30 万元采集实验数据，尚未确定可辨认性，无法可靠计量经济利益
- 判定：不满足确认条件 → **费用化处理**
- 分录：借 管理费用 30 万 / 贷 银行存款 30 万

## Inputs

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `asset_name` | string | 是 | 数据资产名称 |
| `acquisition_purpose` | enum: `internal_use` / `hold_for_sale` / `mixed` | 是 | 持有目的 |
| `recognition_conditions_met` | boolean | 是 | 是否满足初始确认五条件 |
| `cost_collection` | object | 否 | 成本归集明细（来自 `initial-recognition-worksheet`） |
| `mixed_purpose_ratio` | float | 否 | 当 `acquisition_purpose=mixed` 时，自用占比（0-1） |

## Outputs

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `classification_result` | enum: `intangible_asset` / `inventory` / `expense` / `long_term_prepaid` | 判定结果 |
| `applicable_standard` | string | 适用准则（如 "CAS 6"） |
| `journal_entry` | object | 会计分录（借/贷科目+金额） |
| `subsequent_measurement` | string | 后续计量方式 |
| `cross_ref_skill` | string | 衔接技能路径（如 `amortization-scheduler`） |
| `risk_warning` | string | 风险提示（如有） |

## Steps

1. **读取输入**：获取资产名称、持有目的、确认条件满足情况
2. **确认条件校验**：若不满足五条件 → 费用化/长期待摊，输出分录并结束
3. **持有目的判定**：根据 `acquisition_purpose` 进入对应分支
4. **混合用途拆分**：当 `mixed` 时按 `mixed_purpose_ratio` 判定主要用途
5. **科目落位**：输出判定结果 + 适用准则 + 会计分录
6. **后续计量衔接**：根据判定结果调用后续技能
   - 无形资产 → `amortization-scheduler`
   - 存货 → `measurement-model-selector`（可变现净值模式）
   - 费用化 → 结束
7. **风险标注**：边界模糊案例附风险提示，建议人工复核
8. **标准溯源**：每条结论附 `§第X条` 标注

## 常见问法与处理策略

| 常见问法 | 处理策略 | 输出要点 |
|---------|---------|---------|
| "这批数据应该入到哪个科目？" | 执行完整判定逻辑树 | 分类结果 + 准则 + 分录 |
| "我们的数据是自用还是出售？" | 追问业务场景，判断持有目的 | `acquisition_purpose` 判定 |
| "既自用又对外卖怎么处理？" | 按 `mixed_purpose_ratio` 判定主要用途 | 主要用途 + 科目落位 |
| "数据不满足确认条件怎么办？" | 走费用化/长期待摊分支 | 费用化分录 + 说明 |
| "持有目的变了要重新分类吗？" | 触发科目重分类流程 | 重分类分录 + 后续计量调整 |
| "数据集用来训练AI模型算什么？" | 判定为内部使用 → 无形资产 | 无形资产分录 + 摊销衔接 |
| "这个数据产品是给别人买的算存货吗？" | 确认持有待售目的 → 存货 | 存货分录 + 可变现净值衔接 |
