# 📄 数据资产评估报告草稿生成器

## When to Use

当数据资产评估项目需要交付正式《数据资产评估报告》时使用本技能。核心目标：依据 GB/T 42129-2022 及《数据资产评估指导意见》，整合收益法、成本法、市场法三法估值结果，生成结构完整的评估报告草稿，包含适用性说明、三法计算过程、敏感性分析、结论区间与限制条件。

典型触发场景：
- 数据资产评估项目结项交付
- 交易定价/出资入股/质押融资等场景需要正式评估报告
- 三法估值已完成，需要整合报告
- 监管/审计要求提交评估报告

## 工作原则

1. **三法并重**：报告须完整呈现三种估值方法的计算过程，不得选择性披露。
2. **结论审慎**：最终结论以区间形式呈现，附加权依据，避免单一数值误导。
3. **限制充分**：须完整披露评估假设、限制条件和使用范围。
4. **格式规范**：报告结构须符合 GB/T 42129-2022 要求，可审计可追溯。
5. **准则溯源**：每条结论附 `§X.X` 标准溯源标注。

## 报告结构（八大章节）

### 第一章：评估对象概述

**内容包括：**
- 数据资产基本信息（名称、类型、数据量、字段数）
- 数据来源与形成过程
- 数据质量描述（完整性、准确性、一致性、时效性）
- 权利状况说明（所有权/使用权/控制权）
- 账面价值（如有）

### 第二章：评估目的与基准日

**内容包括：**
- 评估目的（交易定价/出资入股/质押融资/财务报告等）
- 评估基准日及选择依据
- 评估假设（持续经营假设/公开市场假设等）
- 评估报告使用范围与限制

### 第三章：评估方法适用性分析

**三法适用性判断表：**

| 评估方法 | 适用性判定 | 判定理由 | 是否采用 |
|---------|----------|---------|---------|
| 收益法 | 适用/不适用 | 有/无明确商业化路径 | ✅/❌ |
| 成本法 | 适用/不适用 | 成本记录完整/不完整 | ✅/❌ |
| 市场法 | 适用/不适用 | 有/无可比案例 | ✅/❌ |

### 第四章：评估过程（三法分别计算）

**4.1 收益法计算**
- 引用 `income-approach-calc` 结果
- 现金流预测表
- 折现率选取过程
- 收益法估值结论

**4.2 成本法计算**
- 引用 `cost-approach-calc` 结果
- 成本归集明细表
- 贬值调整明细
- 成本法估值结论

**4.3 市场法计算**
- 引用 `market-approach-lookup` 结果
- 可比案例列表
- 可比性调整与倍数
- 市场法估值结论

### 第五章：评估结论（结论区间 + 加权）

**三法结果汇总表：**

| 评估方法 | 估值结论（万元） | 权重 | 加权值（万元） |
|---------|----------------|------|--------------|
| 收益法 | 861.13 | 50% | 430.57 |
| 成本法 | 147.86 | 20% | 29.57 |
| 市场法 | 165.90 | 30% | 49.77 |
| **加权结论** | — | 100% | **509.91** |

**结论区间：**
```
估值结论区间：148.00 - 861.00 万元
加权估值结论：509.91 万元
建议结论区间：450.00 - 570.00 万元
```

**权重确定依据：**
- 收益法权重最高：数据资产有明确商业化路径，收益预测可靠性较高
- 市场法次之：有可比交易案例但可比性有限
- 成本法最低：历史成本不能完全反映未来价值

### 第六章：敏感性分析

**关键变量敏感性汇总：**

| 敏感性变量 | 变化幅度 | 收益法影响 | 成本法影响 | 市场法影响 | 综合 |
|-----------|---------|----------|----------|----------|------|
| 折现率 | ±1% | ±12% | 不适用 | 不适用 | ±6% |
| 收入预测 | ±10% | ±15% | 不适用 | 不适用 | ±7.5% |
| 预测期限 | ±1年 | ±8% | 不适用 | 不适用 | ±4% |
| 可比倍数 | ±10% | 不适用 | 不适用 | ±10% | ±3% |

### 第七章：限制条件与假设

**7.1 评估假设**
- 持续经营假设
- 公开市场假设
- 数据资产合法合规假设
- 数据质量稳定假设

**7.2 限制条件**
- 评估结论仅用于指定用途
- 评估基准日后事项不调整
- 数据资产特定风险已在折现率中体现
- 评估不构成交易建议

### 第八章：评估师声明

- 评估师具备相应资质
- 评估过程遵循独立、客观、公正原则
- 评估师与评估对象无利益关系
- 评估报告内容真实、准确、完整

## Inputs

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `asset_name` | string | 是 | 数据资产名称 |
| `valuation_purpose` | string | 是 | 评估目的 |
| `valuation_date` | date | 是 | 评估基准日 |
| `income_approach_result` | object | 否 | 收益法结果（来自 `income-approach-calc`） |
| `cost_approach_result` | object | 否 | 成本法结果（来自 `cost-approach-calc`） |
| `market_approach_result` | object | 否 | 市场法结果（来自 `market-approach-lookup`） |
| `weighting_scheme` | object | 否 | 权重方案（如未提供，按适用性自动分配） |
| `asset_overview` | object | 是 | 评估对象概述信息 |

## Outputs

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `report_document` | string | 完整评估报告文本 |
| `valuation_summary` | object | 三法结果汇总 |
| `weighted_conclusion` | float | 加权估值结论 |
| `conclusion_range` | object | 结论区间 |
| `sensitivity_summary` | object | 敏感性分析汇总 |
| `assumptions` | array[string] | 评估假设清单 |
| `limitations` | array[string] | 限制条件清单 |

## Steps

1. **读取输入信息**：资产概述、评估目的、基准日、三法结果
2. **生成第一章**：评估对象概述（基本信息、数据质量、权利状况）
3. **生成第二章**：评估目的与基准日、评估假设
4. **生成第三章**：三法适用性分析表
5. **生成第四章**：整合三法计算过程
   - 收益法 → 引用 `income-approach-calc` 结果
   - 成本法 → 引用 `cost-approach-calc` 结果
   - 市场法 → 引用 `market-approach-lookup` 结果
6. **生成第五章**：三法汇总 + 权重分配 + 加权结论 + 区间
7. **生成第六章**：整合三法敏感性分析
8. **生成第七章**：限制条件与假设
9. **生成第八章**：评估师声明
10. **格式化报告**：按报告模板组装完整文本
11. **输出报告**：完整报告 + 汇总数据 + 结论区间
12. **标准溯源**：附 `§X.X` 标注

## 常见问法与处理策略

| 常见问法 | 处理策略 | 输出要点 |
|---------|---------|---------|
| "生成评估报告" | 整合三法结果，按八章节生成 | 完整报告文本 |
| "三法结果差异大怎么办？" | 分析差异原因，合理分配权重 | 差异分析 + 权重方案 + 区间结论 |
| "权重怎么分配？" | 按适用性和可靠性分配 | 权重表 + 分配依据 |
| "结论是给一个数还是区间？" | 以区间呈现，附加权结论 | 区间 + 加权值 |
| "报告需要哪些假设？" | 列示标准评估假设 | 假设清单 |
| "限制条件写什么？" | 列示标准限制条件 | 限制条件清单 |
| "只用了两种方法行吗？" | 允许，但须说明未采用方法的理由 | 适用性说明 + 替代分析 |
