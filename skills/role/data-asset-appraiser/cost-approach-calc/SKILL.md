# 🧮 数据资产成本法估值

## When to Use

当数据资产评估需要采用成本法进行估值时使用本技能。核心目标：依据 GB/T 42129-2022 §7.1 及《数据资产评估指导意见》，通过历史成本归集、重置成本计算与贬值调整，输出数据资产的成本法估值结论。成本法特别适用于资产新建、缺乏可比市场数据、且收益难以预测的场景。

典型触发场景：
- 数据资产评估报告需要成本法结论
- 新建数据资产，缺乏活跃市场可比交易
- 自用型数据资产，未来收益难以独立预测
- 收益法和市场法估值结论的交叉验证

## 工作原则

1. **成本完整性**：成本归集须涵盖直接成本、间接成本、开发成本、基础设施费用等全部成本要素。
2. **贬值审慎性**：贬值调整须有充分依据，三类贬值分别评估，不可遗漏。
3. **重置合理性**：重置成本须反映当前市场价格水平，而非历史价格。
4. **方法独立性**：成本法独立计算，不依赖收益法或市场法结论。
5. **准则溯源**：每条结论附 `§7.1.X` 标准溯源标注。

## 成本法估值方法

### 1. 历史成本法

**定义：** 以数据资产开发/获取过程中实际发生的成本作为估值基础。

**成本构成：**
```
历史成本 = 直接成本 + 间接成本 + 合理利润

直接成本 = 采购成本 + 采集成本 + 加工成本
间接成本 = 基础设施分摊 + 管理费用分摊
合理利润 = 行业平均利润率 × (直接成本 + 间接成本)
```

**成本明细表：**

| 成本类别 | 成本项目 | 计算依据 | 金额（万元） |
|---------|---------|---------|------------|
| 直接成本 | 数据采购 | 采购合同 | 50.00 |
| 直接成本 | 采集人员薪酬 | 人数×月薪×月数 | 36.00 |
| 直接成本 | 数据加工 | 清洗标注人员薪酬 | 24.00 |
| 间接成本 | 平台开发 | 开发人员薪酬 | 40.00 |
| 间接成本 | 云存储分摊 | 月费×月数 | 15.00 |
| 间接成本 | 算力分摊 | 按使用量分摊 | 20.00 |
| 合理利润 | 行业利润率 15% | (直接+间接)×15% | 27.75 |
| **合计** | — | — | **212.75** |

### 2. 重置成本法

**定义：** 以当前重新获取或开发相同数据资产所需的成本作为估值基础。

**计算公式：**
```
重置成本 = 重置直接成本 + 重置间接成本 + 合理利润
```

**与历史成本法的差异：**

| 对比维度 | 历史成本法 | 重置成本法 |
|---------|----------|----------|
| 价格基准 | 取得时的实际价格 | 当前市场价格 |
| 技术参数 | 按当时技术条件 | 按当前技术条件 |
| 适用场景 | 成本记录完整 | 资产盘盈/捐赠/评估 |
| 时间调整 | 不调整 | 按当前价格调整 |

**重置成本调整因素：**

| 调整因素 | 调整方法 | 说明 |
|---------|---------|------|
| 价格变动 | 按价格指数调整 | CPI/PPI 数据处理价格指数 |
| 技术效率 | 技术系数调整 | 当前技术效率 vs 开发时效率 |
| 规模效应 | 规模系数调整 | 数据量变化对单位成本影响 |

### 3. 贬值调整

数据资产的贬值分为三类：

**3.1 实体性贬值**

| 贬值类型 | 评估方法 | 数据资产特点 |
|---------|---------|------------|
| 实体性贬值 | 观察法/使用年限法 | 数据无物理损耗，通常为零 |

> 数据资产为无形资产，不具有实体磨损，实体性贬值通常为零。

**3.2 功能性贬值**

| 贬值类型 | 评估方法 | 典型原因 |
|---------|---------|---------|
| 功能性贬值 | 功能对比法 | 数据质量下降、覆盖范围缩减、精度降低 |

**计算公式：**
```
功能性贬值 = (原功能价值 - 现功能价值) / 原功能价值 × 重置成本
```

**功能性贬值评估表：**

| 功能指标 | 原始状态 | 当前状态 | 贬值率 |
|---------|---------|---------|-------|
| 数据覆盖率 | 100% | 85% | 15% |
| 数据精度 | 99% | 95% | 4% |
| 数据时效性 | 实时 | T+1 | 10% |
| 综合贬值率 | — | — | 9.7% |

**3.3 经济性贬值**

| 贬值类型 | 评估方法 | 典型原因 |
|---------|---------|---------|
| 经济性贬值 | 外部环境分析法 | 市场需求下降、竞争加剧、法规限制 |

**计算公式：**
```
经济性贬值 = (设计产能 - 实际/预期利用) / 设计产能 × 重置成本
```

**经济性贬值评估表：**

| 经济因素 | 评估依据 | 影响程度 |
|---------|---------|---------|
| 市场需求 | 行业报告/市场调研 | 利用率下降 15% |
| 竞争状况 | 竞品分析 | 价值折损 5% |
| 法规限制 | 合规审查 | 使用范围受限 3% |
| 综合经济贬值率 | — | 20.8% |

### 4. 成本法综合估值公式

```
成本法估值 = 重置成本 - 实体性贬值 - 功能性贬值 - 经济性贬值
```

## 示例计算

**估值对象：** XX客户画像数据集

**参数：**
- 重置成本：212.75 万元（含合理利润）
- 实体性贬值：0 万元（数据无实体损耗）
- 功能性贬值：212.75 × 9.7% = 20.64 万元
- 经济性贬值：212.75 × 20.8% = 44.25 万元

**估值计算：**
```
成本法估值 = 212.75 - 0 - 20.64 - 44.25 = 147.86 万元
```

## 适用性判断

**成本法适用条件：**

| 序号 | 适用条件 | 判定标准 |
|------|---------|---------|
| 1 | 成本记录完整 | 有完整的成本归集凭证和费用分配记录 |
| 2 | 无活跃市场 | 缺乏可比的市场交易案例 |
| 3 | 自用型资产 | 主要用于内部经营，无明确商业化路径 |
| 4 | 贬值可量化 | 三类贬值可合理评估 |

## Inputs

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `asset_name` | string | 是 | 数据资产名称 |
| `cost_method` | enum: `historical` / `replacement` | 否 | 成本方法（默认 `replacement`） |
| `cost_detail` | array[object] | 是 | 成本明细列表 |
| `profit_rate` | float | 否 | 合理利润率（默认取行业平均） |
| `functional_depreciation_params` | object | 否 | 功能性贬值评估参数 |
| `economic_depreciation_params` | object | 否 | 经济性贬值评估参数 |
| `price_index` | float | 否 | 价格变动指数（重置成本法适用） |

## Outputs

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `valuation_result` | float | 成本法估值结论（万元） |
| `cost_breakdown` | object | 成本明细表 |
| `depreciation_detail` | object | 三类贬值明细 |
| `applicability_check` | object | 适用性判断结果 |
| `calculation_process` | string | 完整计算过程 |
| `risk_disclosure` | string | 风险提示 |

## Steps

1. **读取输入参数**：资产名称、成本方法、成本明细、贬值参数
2. **适用性判断**：检查成本法四项适用条件
3. **成本归集**：按直接成本、间接成本、合理利润分类汇总
4. **重置成本计算**（若选择重置法）：按价格指数/技术系数调整
5. **功能性贬值评估**：按功能对比法计算贬值率
6. **经济性贬值评估**：按外部环境分析法计算贬值率
7. **实体性贬值**：默认为零，附说明
8. **综合估值计算**：重置成本 - 三类贬值
9. **输出估值结论**：估值结果 + 成本明细 + 贬值明细
10. **标准溯源**：附 `§7.1.X` 标注

| "历史成本和重置成本选哪个？" | 根据资产状态选择 | 方法推荐 + 理由 |

| "成本法和收益法差异大？" | 分析差异原因，建议加权 | 差异分析 + 加权建议 |
