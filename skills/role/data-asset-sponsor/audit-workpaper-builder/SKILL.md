# 📋 audit-workpaper-builder

数据资产审计底稿模板构建工具。依据《中国注册会计师审计准则》和财会〔2023〕11号要求，按照**存在性、权属、计量、披露**四大审计认定，为企业接受外部审计前准备规范化的审计工作底稿，覆盖审计程序、证据清单、常见发现与底稿范本。

## When to Use

- 年报审计或专项审计前，需要准备数据资产相关审计底稿。
- 保荐机构（`dam-09`）要求企业提供数据资产审计工作底稿以供核查。
- 企业内部审计部门开展数据资产专项审计。
- 数据资产入表后首次接受外部审计，需建立标准化底稿体系。

## 工作原则

1. **认定导向**：以审计四大认定（存在性、权属、计量、披露）为框架组织底稿。
2. **证据充分**：每条认定均列出充分且适当的审计证据类型及获取方法。
3. **程序规范**：审计程序参照《中国注册会计师审计准则》相关准则条款。
4. **风险分层**：按高/中/低风险标注各认定项的固有风险和控制风险。
5. **底稿可复核**：每张底稿留出审计结论/调整建议/复核意见栏位。

## 一、存在性认定

**核心问题**：账面记录的数据资产是否真实存在，且具备可验证的物质或逻辑载体。

### 审计程序

| 序号 | 程序 | 方法 | 证据来源 | 风险 |
|------|------|------|----------|------|
| EA-1 | 盘存核实 | 实地查验数据存储设施、访问数据库确认数据规模与结构 | 数据仓库清单、存储设备清单、数据库元数据 | 中 |
| EA-2 | 系统穿透测试 | 从财务系统穿行至数据中台/数仓，验证数据资产实体的可访问性 | 系统截图、API 调用日志 | 中 |
| EA-3 | 数据量一致性核验 | 将入表的数据量（记录数/TB等）与数据源系统元数据进行比对 | 数据资产台账 vs. 数据源统计报表 | 高 |
| EA-4 | 抽样验证 | 抽取样本数据资产，验证其是否可被正常访问、查询和使用 | 查询结果截图、数据样本文件 | 低 |
| EA-5 | 期后事项检查 | 检查资产负债表日后数据资产是否仍在使用或已废弃 | 使用日志、数据产品迭代记录 | 中 |

### 证据要求

- 数据资产**台账/清单**（名称、类别、存储位置、容量大小、状态）
- 数据存储**物理/逻辑位置证明**（服务器IP、数据库名、表名）
- 数据**可访问性证明**（不少于5个数据集的查询耗时和返回行数/大小）
- 年度内**数据增量记录**（采集量、处理量、归档量趋势）

### 常见审计发现

| 发现 | 严重程度 | 处理建议 |
|------|----------|----------|
| 数据已物理删除但未核销账面 | 高 | 立即核销，追溯调整前期报表 |
| 账实不符，账面数量/规模大于实际 | 高 | 查明差异原因，补提减值或核销 |
| 部分数据无法正常访问 | 中 | 评估可收回性，必要时计提减值 |
| 数据资产台账信息不全（缺存储位置） | 低 | 补充台账信息，强化台账管理 |

## 二、权属认定

**核心问题**：企业对所记录的数据资产是否拥有合法权利，即所有权/控制权/使用权等三权权属。

### 审计程序

| 序号 | 程序 | 方法 | 证据来源 | 风险 |
|------|------|------|----------|------|
| OA-1 | 权属文件审阅 | 检查数据采集合同、购买协议、用户授权协议、委托加工合同 | 合同原件/扫描件、授权书 | 高 |
| OA-2 | 三权确认 | 逐项确认数据资源持有权、数据加工使用权、数据产品经营权 | 法律意见书、合规审查意见 | 高 |
| OA-3 | 授权链条追溯 | 追溯数据采集至加工利用的全链条授权文件，确认无断点 | 采集协议 → 存储授权 → 加工协议 → 利用许可 | 中 |
| OA-4 | 来源合法审查 | 核查数据来源是否违反《数据安全法》《个人信息保护法》 | 数据来源声明、个人信息授权记录 | 高 |
| OA-5 | 知识产权检索 | 检索数据是否侵犯第三方知识产权或商业秘密 | 版权数据库检索结果、专利检索 | 中 |

### 证据要求

- 数据**采购合同**/数据**委托加工合同**/数据**交易协议**
- 个人信息采集**授权书**及**隐私政策**公示截屏
- **三权清单**（列明每项数据资产对应的持有权、加工权、经营权）
- **律师法律意见书**（针对争议性权属）
- **知识产权检索报告**
- 跨境数据**安全评估报告**（如适用）及**出境申报回执**

### 常见审计发现

| 发现 | 严重程度 | 处理建议 |
|------|----------|----------|
| 无明确授权采集的个人信息用于商业开发 | 高 | 暂停使用，补办授权或删除数据 |
| 数据采购合同权属条款不清 | 高 | 补充协议明确权利归属 |
| 数据来源涉及爬虫，未获被爬方同意 | 高 | 评估合法性风险，必要时核销 |
| 三权清单不完整，部分资产未明确权属 | 中 | 补充三权确认 |

## 三、计量认定

**核心问题**：数据资产的**成本是否可靠计量**、**摊销是否合理**、**减值是否充分**。

### 审计程序

| 序号 | 程序 | 方法 | 证据来源 | 风险 |
|------|------|------|----------|------|
| MA-1 | 成本归集复核 | 复算自行加工/外购/委托加工的数据资产成本归集口径，验证资本化时点 | 研发台账、工时记录、采购发票、合同 | 高 |
| MA-2 | 摊销合理性测试 | 独立重算本期摊销额，评估摊销年限、方法和残值率是否合理 | 摊销计算表、行业对标数据 | 中 |
| MA-3 | 减值测试复核 | 复算或独立执行减值测试，评估可收回金额，验证减值准备计提的充分性 | 收益法/市场法估值模型、减值测试底稿 | 高 |
| MA-4 | 成本资本化/费用化划分 | 审核满足资本化条件时点前后支出的划分是否正确 | 立项文件、结项报告、工时分配表 | 中 |
| MA-5 | 汇兑损益/借款费用 | 如涉及外币计价或专门借款，审核相关处理 | 外币合同、借款合同 | 低 |

### 证据要求

- **成本归集明细表**（分项目列示人工、服务器、外购软件、其他直接费用）
- **资本化审批文件**（立项/里程碑/验收）
- **摊销计算表**（含方法、年限、残值率选择依据）
- **减值测试工作底稿**（来自 `amortization-scheduler`）
- **同行业数据对比**（摊销年限、减值比例等基准）

### 常见审计发现

| 发现 | 严重程度 | 处理建议 |
|------|----------|----------|
| 研发支出资本化时点判定不当，提前或延后 | 高 | 更正资本化时点，调整前期报表 |
| 摊销年限偏长，与数据实际使用寿命不符 | 中 | 重新评估摊销年限，加速摊销 |
| 成本归集不全，遗漏了必要直接支出 | 中 | 补充归集，调整账面价值 |
| 未对明显减值迹象执行减值测试 | 高 | 立即执行减值测试，即期确认减值 |

## 四、披露认定

**核心问题**：财务报告附注中数据资源的**披露是否完整**、**分类是否正确**。

### 审计程序

| 序号 | 程序 | 方法 | 证据来源 | 风险 |
|------|------|------|----------|------|
| DA-1 | 披露完整性检查 | 逐条比对财会〔2023〕11号第八条披露要求与附注实际内容 | 财务报告附注草稿 vs. 准则清单 | 中 |
| DA-2 | 分类准确性核验 | 核验无形资产与存货的分类是否与 `accounting-treatment-classifier` 结果一致 | 分类结果 vs. 附注分类 | 中 |
| DA-3 | 数字勾稽 | 附注表格内数字与科目余额表、总账进行勾稽核对 | 附注数字 vs. 科目余额表 | 高 |
| DA-4 | 自愿披露评估 | 评估自愿披露内容的完整性及是否有误导性陈述 | 自愿披露段 vs. 实情 | 低 |
| DA-5 | 风险披露评估 | 评估风险披露段是否完整覆盖主要风险面 | 风险段 vs. 风险评估报告 | 中 |

### 证据要求

- 财会〔2023〕11号第八条**披露清单对照表**（逐项打勾）
- **附注草稿**（来自 `disclosure-note-drafter`）
- **科目余额表**及**总账**对应数据
- **董事会/管理层关于披露完整性的声明**

### 常见审计发现

| 发现 | 严重程度 | 处理建议 |
|------|----------|----------|
| 遗漏披露无形资产摊销方法 | 中 | 补充披露摊销政策 |
| 存货与无形资产分类错误 | 高 | 更正分类，追溯调整 |
| 附注表格金额与账面不一致 | 高 | 查明差异并调整至一致 |
| 未披露本期重大增加的原因 | 中 | 补充变动原因说明 |

## Inputs

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `audit_type` | enum | 是 | 审计类型 | `"年报审计"` / `"专项审计"` / `"IPO审计"` |
| `financial_data` | object | 是 | 数据资产科目余额及变动数据 | 含无形资产/存货的期初期末余额 |
| `accounting_classification` | object | 是 | 分类结果 | 来自 `accounting-treatment-classifier` |
| `amortization_result` | object | 是 | 摊销与减值结果 | 来自 `amortization-scheduler` |
| `disclosure_note` | string | 否 | 附注草稿 | 来自 `disclosure-note-drafter` |
| `ownership_documents` | array | 否 | 权属证据文件清单 | `[{文件: "采购合同.pdf", 资产: "用户画像数据"}]` |
| `cost_details` | object | 否 | 成本归集明细 | 分项目的成本明细 |
| `materiality_threshold` | number | 否 | 重要性水平 | 默认取税前利润的 5% |

## Outputs

| 字段 | 类型 | 说明 |
|------|------|------|
| `workpaper_package` | object | 完整审计底稿包（四份认定底稿 + 汇总表） |
| `existence_workpaper` | object | 存在性认定底稿 |
| `ownership_workpaper` | object | 权属认定底稿 |
| `measurement_workpaper` | object | 计量认定底稿 |
| `disclosure_workpaper` | object | 披露认定底稿 |
| `findings_summary` | array | 审计发现汇总 |
| `risk_assessment_matrix` | object | 风险评估矩阵 |
| `audit_conclusion` | string | 综合审计结论（无保留/保留/否定/无法表示意见） |

## Steps

1. **确认审计类型和范围**：明确是年报审计、专项审计还是IPO审计，确定审计覆盖面。
2. **设定重要性水平**：以税前利润或净资产的合适比例设定重要性水平，未输入时默认5%。
3. **构建存在性底稿（EA）**：执行 EA-1~EA-5 审计程序，获取证据，记录发现。
4. **构建权属底稿（OA）**：执行 OA-1~OA-5 审计程序，获取并审核权属证据文档。
5. **构建计量底稿（MA）**：执行 MA-1~MA-5 审计程序，独立复算成本归集、摊销和减值。
6. **构建披露底稿（DA）**：执行 DA-1~DA-5 审计程序，对照准则逐条检查披露完整性。
7. **汇总审计发现**：将四份底稿的发现汇总为 findings_summary，按严重程度排序。
8. **形成审计结论**：根据发现的性质和金额，形成综合审计结论。
9. **标注标准溯源**：每条程序末尾标注引用的审计准则编号。
10. **生成底稿包**：整合为可交付的审计工作底稿包。

## 常见问法与处理策略

| 问法 | 处理策略 | 注意事项 |
|------|----------|----------|
| "帮我们准备数据资产的审计底稿" | 收集 financial_data 和分类结果后按模板生成四份底稿 | 先确认审计类型，不同类型的取证深度不同 |
| "数据资产盘点不出来怎么办" | 建议先调用 IT 审计程序，导出全量数据集清单，再由业务部门逐个认领确认 | 存在性认定必须先于其他三项 |
| "权属文件不全，部分数据采购时没签合同" | 标记为高风险发现，建议①要求对方补签合同；②法律部门出具补充意见；③评估是否需核销该资产 | 权属瑕疵可能触发一票否决 |
| "减值测试太复杂，能不能不做" | 必须做！若存在减值迹象却不测试属于重大错报风险；可引入 `amortization-scheduler` 辅助 | 这是审计师无法退让的底线 |
| "内部审计和外部审计底稿有什么区别" | 内部审计底稿可用于自评和整改，外审底稿增加独立取证和第三方证据要求 | 内审底稿可作为外审的参考 |
| "计量认定和披露认定有什么区别" | 计量关注数字对不对（金额/摊销/减值），披露关注该说的有没有说全 | 金额正确不代表披露完整 |
