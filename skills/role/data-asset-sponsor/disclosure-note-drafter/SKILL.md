# 📝 disclosure-note-drafter

数据资产附注披露段起草工具。根据《企业数据资源相关会计处理暂行规定》（财会〔2023〕11号）第八条要求，为企业年报/半年报编制**数据资源披露附注段落**，涵盖会计政策说明、期初期末余额表、变动原因分析及自愿披露内容。

## When to Use

- 编制年度财务报告/半年度财务报告时，需要撰写数据资产相关附注披露段落。
- 审计师要求企业提供数据资源附注披露草稿供审阅。
- 保荐机构（`dam-09`）在入表报送前需准备规范化的披露材料。
- 企业拟自愿披露未确认为资产的数据资源信息。

## 工作原则

1. **准则遵循**：严格遵循财会〔2023〕11号第八条的披露要求，不遗漏必披项目。
2. **分类列报**：数据资源无形资产与存货**分类列报**，不得混同。
3. **数字勾稽**：期初余额 + 本期增加 - 本期减少 = 期末余额，表格内数字互为勾稽。
4. **变动有因**：对企业自行加工、外购、合并增加、出售、报废、失效等变动逐一说明原因。
5. **风险透明**：披露数据资源相关风险，包括技术迭代、法律合规、市场需求变化等。

## 附注披露结构

### 一、会计政策

披露企业对数据资源的**会计政策选择**，包括：

- **确认条件**：符合无形资产/存货定义且满足可辨认性、有关经济利益很可能流入、成本可靠计量等条件（参见 `accounting-treatment-classifier`）。
- **计量基础**：初始计量采用历史成本，包括外购价款、自行加工成本、使数据资源达到预定用途前发生的合理必要支出。
- **摊销方法**：无形资产的摊销方法（直线法/加速法）、摊销年限及其确定依据。
- **存货计价**：发出数据资源存货的计价方法（个别计价/加权平均）。
- **减值政策**：减值测试频率、可收回金额确定方法、减值迹象判断标准。

### 二、数据资源无形资产

编制**数据资源无形资产变动表**：

| 项目 | 行次 | 期初余额 | 本期增加 | 本期摊销 | 本期减值 | 本期处置 | 其他减少 | 期末余额 |
|------|------|----------|----------|----------|----------|----------|----------|----------|
| 一、账面原值合计 | 1 | - | - | - | - | - | - | - |
| 其中：外购 | 2 | - | - | - | - | - | - | - |
| 自行加工 | 3 | - | - | - | - | - | - | - |
| 其他方式取得 | 4 | - | - | - | - | - | - | - |
| 二、累计摊销合计 | 5 | - | - | - | - | - | - | - |
| 三、减值准备合计 | 6 | - | - | - | - | - | - | - |
| 四、账面价值合计 | 7=1-5-6 | - | - | - | - | - | - | - |

**本期增加明细**：
- 外购新增数据资源 [金额] 元，主要系 [事项说明]
- 自行加工新增 [金额] 元，主要系 [研发项目/加工活动说明]
- 合并增加 [金额] 元，系 [合并对象说明]

**本期减少明细**：
- 本期摊销 [金额] 元
- 本期处置/报废 [金额] 元，原因：[技术过时/合同到期/数据失效等]
- 本期计提减值 [金额] 元，参考 `amortization-scheduler` 的减值测试结果

### 三、数据资源存货

编制**数据资源存货变动表**，格式与无形资产相同：

| 项目 | 行次 | 期初余额 | 本期增加 | 本期减少 | 其中：转销 | 期末余额 |
|------|------|----------|----------|----------|------------|----------|
| 一、账面余额合计 | 1 | - | - | - | - | - |
| 其中：外购 | 2 | - | - | - | - | - |
| 自行采集加工 | 3 | - | - | - | - | - |
| 二、存货跌价准备合计 | 4 | - | - | - | - | - |
| 三、存货账面价值合计 | 5=1-4 | - | - | - | - | - |

**本期变动说明**：
- 本期新增存货 [金额] 元，用途：[对外出售/内部使用]
- 本期减少存货 [金额] 元，原因：[实现销售/失效/转作无形资产]

### 四、未确认为资产的数据资源

对于**未确认为资产但对决策有重要影响的数据资源**，鼓励自愿披露：

- 数据资源名称/类别：[例如：用户行为数据/设备日志数据等]
- 未确认原因：[不满足可辨认性/成本无法可靠计量/经济利益不确定等]
- 对企业的潜在价值描述：[描述]
- 相关费用化金额：[研发支出费用化金额/采集成本费用化金额]

### 五、数据资源相关风险

- **技术迭代风险**：数据存储/处理技术更新可能使现有数据资源贬值。
- **法律合规风险**：《数据安全法》《个人信息保护法》合规要求变化。
- **市场风险**：数据交易市场供求变化导致公允价值波动。
- **安全风险**：数据泄露、篡改、丢失可能影响数据资产价值。

## Inputs

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `reporting_period` | string | 是 | 报告期间 | `"2025-01-01 至 2025-12-31"` |
| `report_type` | enum | 是 | 报告类型 | `"年报"` / `"半年报"` |
| `intangible_assets` | object | 否 | 无形资产分类账数据 | `{期初余额: 5000000, 本期增加: 2000000, ...}` |
| `inventory_assets` | object | 否 | 存货分类账数据 | `{期初余额: 1500000, 本期增加: 800000, ...}` |
| `amortization_policy` | object | 是 | 摊销政策 | `{方法: "直线法", 年限: 5, 残值率: 0}` |
| `impairment_result` | object | 否 | 减值测试结果 | 来自 `amortization-scheduler` 的输出 |
| `accounting_classification` | object | 是 | 分类结果 | 来自 `accounting-treatment-classifier` 的输出 |
| `voluntary_disclosure` | array | 否 | 自愿披露项 | `[{名称: "用户日志", 未确认原因: "..."}]` |
| `risk_items` | array | 否 | 自定义风险项 | `[{类别: "技术迭代", 描述: "..."}]` |

## Outputs

| 字段 | 类型 | 说明 |
|------|------|------|
| `disclosure_note` | string | 格式化的附注披露全文（Markdown 或 Word-ready 文本） |
| `accounting_policy_section` | string | 会计政策段落 |
| `intangible_table` | object | 无形资产变动表（结构化） |

## Steps

1. **确认报告期和类型**：读取 `reporting_period` 和 `report_type`，确定会计期间范围。
2. **获取分类结果**：调用 `accounting-treatment-classifier` 获取数据资源的分类（无形资产/存货/费用化），确认入表的数据资产清单。
3. **获取摊销与减值数据**：调用 `amortization-scheduler` 获取本期摊销金额和减值测试结果。
4. **编写会计政策段**：基于企业实际采用的会计政策，撰写 "重要会计政策——数据资源" 段落。
5. **编制无形资产变动表**：汇总期初余额、本期增加（外购/自行加工/合并）、本期减少（摊销/减值/处置），计算期末余额。
6. **编制存货变动表**：汇总期初余额、本期增加（外购/自行采集加工）、本期减少（销售/失效/转出），计算期末余额。
7. **撰写变动原因说明**：对每项重大变动（>10% 或超过重要性水平）撰写原因说明。
8. **编写自愿披露段**：对于未入表但重要的数据资源，撰写自愿披露内容。
9. **编写风险提示段**：梳理技术、法律、市场、安全等方面的数据资源风险。
10. **交叉检查勾稽关系**：验证表格内数字勾稽，确保期初+增加-减少=期末。
11. **标注标准溯源**：每条判断性表述末尾标注溯源（如 `§第八条`）。
12. **整合输出**：将各段整合为完整的财务附注披露文档。

## 常见问法与处理策略

| 问法 | 处理策略 | 注意事项 |
|------|----------|----------|
| "帮我写一份数据资产附注披露" | 要求提供 `reporting_period`，调用 `accounting-treatment-classifier` 后按模板生成 | 需确认报告类型是年报还是半年报 |
| "我们有些数据没有入表，要不要披露" | 分析该数据资源的重要性，若对决策有影响则建议自愿披露，并注明未确认原因 | 自愿披露需说明原因，不可误导使用者 |
| "摊销年限怎么确定" | 评估数据资源的经济利益预期消耗方式、技术迭代周期、合同或法定期限，参考同行业实践 | 变更摊销年限属于会计估计变更，需追溯调整 |
| "变动表数字不平怎么办" | 逐项检查：①期初余额是否与上年期末一致；②本期增加是否录入完整；③本期减少是否遗漏摊销或减值 | 差异需查出原因并调整 |
| "附注要披露什么风险" | 至少包含技术迭代、法律合规、市场需求、数据安全四类，可根据企业实际情况补充 | 风险描述应具体，不可套话 |
| "这是半年度报告，和年报有什么不同" | 半年度报告披露要求简化：可不披露完整变动表，但需披露主要变动和会计政策；减值测试仍需要 | 参考《企业会计准则第32号——中期财务报告》 |
