# 🎯 matching-scorecard

供需匹配打分卡——五维度加权评分，为买方输出 Top-N 候选卖方推荐清单。

## When to Use

- 买方提交 intake 表单后，需要从挂牌产品中匹配候选卖方时
- 买方需求变化，需要重新匹配候选时
- 经纪人向买方推荐产品，需要量化推荐依据时
- 平台自动推荐引擎，需要打分排序时

## 工作原则

1. **需求驱动**：打分以买方 intake 表单为基准，所有维度对标买方需求
2. **五维均衡**：覆盖度、时效性、价格、合规、售后五维度加权评分，不可偏废
3. **量化打分**：每个维度按明确标准打分（0-100），避免主观判断
4. **Top-N 推荐**：输出前 3-5 名候选，而非全部匹配结果，降低买方选择成本
5. **可解释**：每个推荐必须附打分明细，解释"为什么推荐这个" `§5.2.5`

## 五维度打分卡

### 一、覆盖度（权重 30%）

**评估目标：** 卖方产品对买方数据需求的覆盖程度。

| 评分项 | 满分 | 评分标准 | 得分 |
|-------|:----:|---------|:----:|
| 字段覆盖率 | 40 | 覆盖买方需求字段的百分比 × 40 | ___ |
| 记录量达标 | 30 | 达到买方需求量级 = 30，差一档 = 15，差两档 = 0 | ___ |
| 时间范围覆盖 | 30 | 完全覆盖 = 30，覆盖≥80% = 20，覆盖<80% = 0 | ___ |
| **覆盖度小计** | **100** | 加权汇总 | **___** |

### 二、时效性（权重 20%）

**评估目标：** 数据更新频率和交付时效是否满足买方需求。

| 评分项 | 满分 | 评分标准 | 得分 |
|-------|:----:|---------|:----:|
| 更新频率达标 | 50 | 满足/超出需求 = 50，低一档 = 25，低两档 = 0 | ___ |
| 交付时效 | 50 | 满足期望日期 = 50，延迟≤7天 = 30，延迟>7天 = 0 | ___ |
| **时效性小计** | **100** | 加权汇总 | **___** |

### 三、价格（权重 20%）

**评估目标：** 产品价格是否在买方预算内及性价比。

| 评分项 | 满分 | 评分标准 | 得分 |
|-------|:----:|---------|:----:|
| 预算内 | 50 | 在预算范围内 = 50，超预算≤10% = 30，超预算>10% = 0 | ___ |
| 性价比 | 50 | 单位价格数据量优于市场均价 = 50，持平 = 30，低于 = 10 | ___ |
| **价格小计** | **100** | 加权汇总 | **___** |

### 四、合规（权重 20%）

**评估目标：** 卖方及产品的合规状况。

| 评分项 | 满分 | 评分标准 | 得分 |
|-------|:----:|---------|:----:|
| 合规审查通过 | 40 | 已通过 `compliance-review-sheet` = 40，未通过 = 0 | ___ |
| PII 处理达标 | 30 | 满足买方PII要求 = 30，部分满足 = 15，不满足 = 0 | ___ |
| 出境合规 | 30 | 满足出境要求 = 30，不涉及出境 = 30，不满足 = 0 | ___ |
| **合规小计** | **100** | 加权汇总 | **___** |

### 五、售后（权重 10%）

**评估目标：** 卖方的售后保障能力。

| 评分项 | 满分 | 评分标准 | 得分 |
|-------|:----:|---------|:----:|
| SLA 承诺 | 40 | 明确SLA且达标 = 40，有SLA但不完善 = 20，无SLA = 0 | ___ |
| 技术支持 | 30 | 7×24支持 = 30，工作日支持 = 20，无支持 = 0 | ___ |
| 口碑评分 | 30 | 历史评分≥4.5 = 30，≥3.5 = 20，<3.5 = 10 | ___ |
| **售后小计** | **100** | 加权汇总 | **___** |

## 综合评分与推荐

**综合得分公式：**

```
TotalScore = 覆盖度×0.30 + 时效性×0.20 + 价格×0.20 + 合规×0.20 + 售后×0.10
```

**Top-N 推荐逻辑：**

| 条件 | 处理 |
|------|------|
| 综合得分 ≥ 80 | 推荐为"强烈推荐"，排列在 Top-N 前列 |
| 综合得分 60-79 | 推荐为"推荐" |
| 综合得分 40-59 | 推荐为"可选" |
| 综合得分 < 40 | 不推荐 |
| 合规小计 = 0 | 一票否决，不推荐（无论总分多少） |

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 买方 intake 表单 | Document | `deal-intake-form` 输出的结构化需求 |
| 挂牌产品清单 | List | 平台所有在架产品及其属性 |
| 合规审查结果 | List | 各产品的 `compliance-review-sheet` 结果 |
| 卖方口碑数据 | List | 卖方历史评分和售后记录 |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 打分卡 | Document | 五维度打分明细 + 综合得分 + 排名 |
| Top-N 推荐清单 | Document | 前 3-5 名候选产品及推荐说明 |
| 匹配报告 | Document | 匹配过程说明和推荐理由 |

## Steps

1. 接收买方 intake 表单，提取需求关键词
2. 从挂牌产品库中初筛匹配产品（基于数据类型、领域、量级）
3. 对初筛产品逐个执行五维度打分
4. 计算综合得分并排名
5. 执行一票否决检查（合规=0 的产品排除）
6. 输出 Top-N 推荐清单和打分卡 `§5.2.5`
7. 将推荐清单提交经纪人，由经纪人向买方推荐
8. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| 怎么给买方推荐产品？ | 基于 intake 表单执行五维度打分（覆盖度/时效性/价格/合规/售后），输出 Top-5 推荐 |
| 权重可以调吗？ | 可以，不同买方关注点不同（如金融买方更重合规可调高合规权重），但默认权重适用大多数场景 |
| 合规为什么是一票否决？ | 合规不通过的产品存在法律风险，无论其他维度多优秀都不能推荐 |
| 匹配不到合适的产品怎么办？ | 扩大搜索范围（降低量级要求/放宽时间范围），或标记为"需求未满足"，触发卖方拓展 |
| Top-N 取几名？ | 默认 Top-5，买方要求精简时取 Top-3，买方要求广泛选择时取 Top-10 |
| 打分卡能自动化吗？ | 可以，对接挂牌库和 intake 系统，实现自动初筛+打分+排名，经纪人只需复核 |
