# 💰 pricing-tier-designer

数据产品定价分层设计工具，支持订阅制、按次计费、包月制、阶梯定价四种模式，内置定价要素分析与毛利测算，帮助产品经理制定科学合理的定价策略，确保毛利率目标达成。

## When to Use

- **新品定价**：新数据产品上架前制定定价方案（§5.1.6.6）
- **老品调价**：现有产品价格调整，涨价或降价决策
- **定价模式切换**：从按次计费切换为订阅制，或反向切换
- **竞争应对**：竞品调价后评估是否跟进调整
- **毛利优化**：毛利率不达标时重新设计定价分层
- **促销设计**：设计限时折扣、新客优惠等促销定价

## 工作原则

1. **价值定价**：以客户感知价值为定价基础，而非单纯成本加成（§5.1.6.6）
2. **分层覆盖**：多档次定价覆盖不同消费能力与用量场景
3. **毛利保障**：所有定价方案的毛利率目标 ≥ 40%，不达标需调整
4. **竞争感知**：定价需参考竞品价格带，避免过高或过低
5. **弹性测试**：关键产品定价调整前做需求弹性测试
6. **动态优化**：定期回顾定价效果，按市场反馈动态调整

## 定价模式详解

### 1. 订阅制

按周期收费，适合稳定使用的高频产品。

| 档次 | 月费（元） | 年费（元） | 包含内容 | 适用场景 |
|------|-----------|-----------|---------|---------|
| 基础版 | 2,000 | 20,000 | 1万次/月+标准支持 | 小型企业 |
| 专业版 | 5,000 | 50,000 | 5万次/月+优先支持 | 中型企业 |
| 旗舰版 | 12,000 | 120,000 | 20万次/月+专属支持 | 大型企业 |

**特点：** 收入可预测、客户粘性高、需控制用量超限风险

### 2. 按次计费

按调用次数收费，适合低频或用量波动大的产品。

| 计费单元 | 单价（元） | 批量折扣 | 说明 |
|---------|-----------|---------|------|
| 1-1万次 | 0.50/次 | — | 标准价 |
| 1万-10万次 | 0.40/次 | 8折 | 批量折扣 |
| 10万-100万次 | 0.30/次 | 6折 | 大客户折扣 |
| 100万次以上 | 0.20/次 | 4折 | 战略客户 |

**特点：** 低门槛入门、用多少付多少、收入波动大

### 3. 包月制

按月固定费用，不限次或有限次，适合中等频率产品。

| 档位 | 月费（元） | 调用上限 | 超限单价 | 适用场景 |
|------|-----------|---------|---------|---------|
| 不限次包月 | 8,000 | 不限 | — | 高频重度用户 |
| 有限次包月 | 3,000 | 3万次 | 0.30/次 | 中频用户 |
| 轻量包月 | 800 | 5千次 | 0.50/次 | 轻度用户 |

**特点：** 收入稳定、用户有预算安全感、超限收入是增量

### 4. 阶梯定价

按用量分档，价格递减，适合用量差异大的产品。

| 用量区间 | 单价（元/次） | 累计计算方式 | 月费示例（10万次） |
|---------|-------------|-------------|------------------|
| 0-1万次 | 0.50 | 阶梯计费 | 5,000 |
| 1万-5万次 | 0.40 | 阶梯计费 | 16,000 |
| 5万-20万次 | 0.30 | 阶梯计费 | 45,000 |
| 20万次以上 | 0.20 | 阶梯计费 | — |

**特点：** 大客户优惠明显、激励用量增长、计算复杂度高

## 定价要素分析

| 要素 | 说明 | 影响方向 |
|------|------|---------|
| 成本 | 数据采购+计算+存储+人力+分摊 | 成本越高，定价越高 |
| 竞争 | 竞品价格带与差异化程度 | 竞品越低，定价受限 |
| 价值 | 客户感知价值与ROI | 价值越高，定价空间越大 |
| 需求弹性 | 价格变动对需求的影响 | 弹性大，调价需谨慎 |
| 数据稀缺性 | 数据独有性与可替代性 | 越稀缺，定价越高 |
| 合规成本 | 合规要求带来的额外成本 | 合规成本高，定价需覆盖 |

## 毛利测算

### 测算公式

```
毛利 = 收入 - 直接成本
毛利率 = 毛利 / 收入 × 100%
直接成本 = 数据采购 + 计算存储 + API网关 + 人力分摊
```

### 测算示例（订阅制专业版）

| 项目 | 月金额（元） | 说明 |
|------|------------|------|
| 收入 | 5,000 | 月费 |
| 数据采购 | 800 | 第三方数据源费用 |
| 计算存储 | 500 | 云资源分摊 |
| API网关 | 200 | 调用费用分摊 |
| 人力分摊 | 1,000 | 运维+支持分摊 |
| **直接成本** | **2,500** | — |
| **毛利** | **2,500** | — |
| **毛利率** | **50%** | 达标（≥40%） |

### 毛利率目标

| 产品类型 | 毛利率目标 | 说明 |
|---------|-----------|------|
| 核心产品 | ≥50% | 高毛利支撑公司利润 |
| 增长产品 | ≥40% | 兼顾增长与盈利 |
| 探索产品 | ≥30% | 允许低毛利换取市场 |

## Inputs

- `product_id`（string，必填）：数据产品ID
- `pricing_model`（enum，必填）：订阅/按次/包月/阶梯
- `cost_data`（object，必填）：成本数据（采购/计算/人力）
- `competitor_prices`（array，可选）：竞品价格列表
- `margin_target`（number，必填）：毛利率目标，默认0.4
- `customer_segments`（array，可选）：目标客户分群

**示例输入：**

```json
{
  "product_id": "DP-001",
  "pricing_model": "订阅",
  "cost_data": {
    "data_procurement": 800,
    "compute_storage": 500,
    "api_gateway": 200,
    "labor": 1000
  },
  "competitor_prices": [3000, 5500, 11000],
  "margin_target": 0.45,
  "customer_segments": ["小型企业", "中型企业", "大型企业"]
}
```

## Outputs

- `pricing_scheme`（object）：定价方案（分档/价格/内容）
- `margin_analysis`（object）：毛利测算结果
- `price_tier_table`（object）：价格分层表
- `competitor_comparison`（object）：竞品对比分析
- `recommendation`（string）：定价建议与理由

## Steps

1. **成本核算**：核算数据采购、计算存储、API网关、人力分摊等直接成本（§5.1.6.6.1）
2. **竞品调研**：调研竞品定价模式与价格带，识别差异化定位空间
3. **价值评估**：评估客户感知价值与ROI，确定价值定价上限
4. **模式选择**：根据产品特性与使用场景选择定价模式（订阅/按次/包月/阶梯）
5. **分档设计**：设计价格档次，每档明确包含内容与适用场景
6. **毛利测算**：按测算公式计算各档毛利率，确保达标（≥40%）
7. **弹性评估**：对关键产品做需求弹性测试，评估调价影响
8. **方案输出**：输出完整定价方案，调用 `` `role/data-asset-operations-planner/data-product-canvas` `` 关联产品画像
9. **评审发布**：定价方案提交评审，通过后配置到产品并上架

### 定价设计流程图

```
成本核算 → 竞品调研 → 价值评估 → 模式选择 → 分档设计 → 毛利测算 → 弹性评估 → 方案输出 → 评审发布
                                                         │
                                                         └─ 毛利率不达标 → 回到分档设计
```

## 常见问法与处理策略

| 常见问法 | 处理策略 | 标准溯源 |
|---------|---------|---------|
| "新品怎么定价？" | 按九步流程执行，先核算成本再设计分档，确保毛利率≥40% | §5.1.6.6 |
| "按次还是订阅好？" | 高频用订阅、低频用按次、中等频率用包月，参考客户使用习惯 | §5.1.6.6.2 |
| "毛利率不达标怎么办？" | 降低成本或提高价格或调整档次内容，优先优化成本结构 | §5.1.6.6.1 |
| "竞品降价要不要跟进？" | 评估自身差异化价值，有差异化可维持价格，无差异化需跟进 | §5.1.6.6.3 |
| "老品涨价会不会流失客户？" | 做需求弹性测试，涨幅控制在弹性范围内，提前通知并给过渡期 | §5.1.6.6.4 |
| "阶梯定价怎么设计用量区间？" | 按客户用量分布设计，P25/P50/P75/P90作为分档点 | §5.1.6.6.2 |
| "大客户要定制价怎么办？" | 设战略客户档次，低于标准价但毛利率仍≥30%，签年框锁定 | §5.1.6.6.2 |
| "促销折扣怎么设计？" | 限时折扣≤7折、新客首月5折、年付8折，促销期后恢复原价 | §5.1.6.6.4 |
