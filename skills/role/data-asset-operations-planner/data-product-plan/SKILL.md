# 🖼️ data-product-plan

数据产品季度/年度规划工具，从商业目标出发，串联产品矩阵、路线图、运营投入与KPI体系，形成完整的数据产品组合运营规划方案，支撑数据产品商业化运营决策。

## When to Use

- **年度规划**：制定年度数据产品组合战略与运营目标（§5.1.6.1）
- **季度滚动**：季度回顾调整产品矩阵与路线图，滚动更新规划
- **新品立项**：新数据产品立项前的组合定位与资源评估
- **产品评审**：产品组合定期评审，识别增长/衰退产品
- **预算分配**：年度预算分配时评估各产品投入产出比
- **战略调整**：市场环境变化时调整产品组合战略方向

## 工作原则

1. **目标导向**：规划从商业目标出发，产品矩阵与路线图服务于目标达成（§5.1.6.1）
2. **组合思维**：用矩阵视角管理产品组合，平衡核心/增长/探索三类产品
3. **节奏可控**：路线图按季度设定里程碑，每个里程碑有明确验收标准
4. **投入产出**：运营投入与预期产出匹配，ROI是资源配置的核心依据
5. **量化管理**：KPI体系覆盖商业/用户/产品/财务四维，可量化可追踪
6. **动态调整**：季度滚动调整，不固守年初计划，响应市场变化

## 规划结构

### 1. 商业目标

从顶层商业目标出发，定义数据产品组合要支撑的业务成果。

| 目标维度 | 指标 | 示例目标 | 衡量周期 |
|---------|------|---------|---------|
| 营收目标 | GMV | 年GMV 5000万 | 月/季/年 |
| 用户增长 | 活跃消费方数 | 年新增200家消费方 | 月/季 |
| 市场占有率 | 行业渗透率 | 目标行业渗透率15% | 季/年 |
| 品牌影响 | NPS | NPS ≥ 50 | 季度 |
| 利润目标 | 毛利率 | 组合毛利率 ≥ 45% | 月/季 |

### 2. 产品矩阵

用二维矩阵管理产品组合，横轴为产品定位，纵轴为生命周期阶段。

|  | 新品 | 成熟品 | 衰退品 |
|--|------|--------|--------|
| **核心产品** | 企业征信API v3（战略投入） | 工商信息查询（稳定运营） | 旧版风控模型（计划退场） |
| **增长产品** | 行业图谱API（重点推广） | 企业画像标签包（扩大覆盖） | — |
| **探索产品** | 大模型知识库API（MVP验证） | — | — |

**矩阵策略：**
- 核心产品×成熟品：稳定现金流，持续优化
- 增长产品×新品：重点投入资源，加速增长
- 探索产品×新品：小步快跑，MVP验证后决定加注或止损
- 衰退品：制定退场计划，平稳迁移用户

### 3. 路线图

按季度设定里程碑，标注依赖关系与验收标准。

| 季度 | 里程碑 | 依赖项 | 验收标准 |
|------|--------|--------|---------|
| Q1 | 企业征信API v3上线 | 数据源扩容 | API可用性≥99.9% |
| Q2 | 行业图谱API正式发布 | 图谱构建完成 | 覆盖10个行业、1000万实体 |
| Q3 | 企业画像标签包v2 | 标签体系重构 | 标签数≥500、准确率≥95% |
| Q4 | 大模型知识库API GA | 模型评测通过 | 响应时间≤2s、准确率≥90% |

### 4. 运营投入

按人力、技术、市场、预算四维规划投入。

| 投入维度 | 分配项 | 年度预算 | 占比 |
|---------|--------|---------|------|
| 人力 | 产品经理×3、工程师×8、运营×2 | 800万 | 50% |
| 技术 | 云资源、数据采购、模型训练 | 400万 | 25% |
| 市场 | 推广、展会、客户活动 | 240万 | 15% |
| 预留 | 弹性预算 | 160万 | 10% |

### 5. KPI体系

四维KPI体系覆盖商业/用户/产品/财务，支撑组合运营管理。

| KPI维度 | 指标 | 目标值 | 跟踪频率 |
|---------|------|--------|---------|
| 商业 | GMV | 5000万/年 | 月 |
| 商业 | 收入增长率 | ≥30% YoY | 季 |
| 用户 | 活跃消费方数 | ≥500 | 月 |
| 用户 | 转化率 | ≥15% | 月 |
| 用户 | NPS | ≥50 | 季 |
| 产品 | API调用量 | ≥1亿次/月 | 日 |
| 产品 | 可用性 | ≥99.9% | 实时 |
| 财务 | 毛利率 | ≥45% | 月 |
| 财务 | 客单价 | ≥3万/年 | 季 |

## Inputs

- `planning_period`（string，必填）：规划周期，如 `2026年度` 或 `2026Q3`
- `business_goals`（object，必填）：商业目标（GMV/用户/市占率/利润）
- `current_portfolio`（array，可选）：现有产品组合列表
- `market_analysis`（object，可选）：市场分析与竞争情报
- `budget_ceiling`（number，必填）：预算上限

**示例输入：**

```json
{
  "planning_period": "2026年度",
  "business_goals": {
    "gmv": 50000000,
    "new_customers": 200,
    "market_share": 0.15,
    "gross_margin": 0.45
  },
  "current_portfolio": ["企业征信API", "工商信息查询"],
  "budget_ceiling": 16000000
}
```

## Outputs

- `plan_document`（object）：完整规划文档（五大模块）
- `product_matrix`（object）：产品矩阵分析结果
- `roadmap`（object）：季度路线图与里程碑
- `budget_allocation`（object）：预算分配方案
- `kpi_dashboard`（object）：KPI看板配置
- `risk_assessment`（array）：规划风险评估

## Steps

1. **目标对齐**：与管理层对齐年度商业目标，确认GMV/用户/市占率/利润目标（§5.1.6.1）
2. **组合盘点**：盘点现有产品组合，评估各产品生命周期阶段，调用 `` `role/data-asset-operations-planner/data-product-canvas` `` 生成产品画像
3. **矩阵构建**：按核心/增长/探索 × 新品/成熟品/衰退品构建产品矩阵，识别组合缺口
4. **路线图制定**：按季度制定里程碑，标注依赖关系与验收标准
5. **投入规划**：按人力/技术/市场/预算四维分配资源，确保ROI合理
6. **KPI设定**：建立商业/用户/产品/财务四维KPI体系，设定目标值与跟踪频率
7. **定价规划**：对新品的定价策略进行预规划，调用 `` `role/data-asset-operations-planner/pricing-tier-designer` ``
8. **上架规划**：新品上架计划，调用 `` `role/data-asset-operations-planner/api-marketplace-config` `` 配置上架方案
9. **评审发布**：规划文档提交管理层评审，通过后发布执行

### 规划流程图

```
目标对齐 → 组合盘点 → 矩阵构建 → 路线图制定 → 投入规划 → KPI设定 → 定价规划 → 上架规划 → 评审发布
              │                                  │
              └─ 产品画像                        └─ 预算分配
```

## 常见问法与处理策略

| 常见问法 | 处理策略 | 标准溯源 |
|---------|---------|---------|
| "今年数据产品怎么规划？" | 按五步规划流程执行，输出完整规划文档 | §5.1.6.1 |
| "新品应该定位核心还是探索？" | 根据市场验证程度判断，未验证放探索，已验证放核心或增长 | §5.1.6.2 |
| "路线图怎么排优先级？" | 按商业价值×可行性打分，高价值高可行性优先排期 | §5.1.6.3 |
| "预算不够怎么办？" | 优先保障核心产品投入，削减探索产品，预留弹性预算 | §5.1.6.4 |
| "KPI怎么定才合理？" | 参考历史数据+行业基准，设定有挑战但可达成的目标 | §5.1.6.5 |
| "衰退品什么时候退场？" | 连续两季度负增长且无改善趋势，启动退场流程 | §5.1.6.2 |
| "产品矩阵有缺口怎么办？" | 识别缺口后纳入新品立项评估，纳入下季度路线图 | §5.1.6.2 |
| "规划需要多久调整一次？" | 季度滚动调整，年度大规划+季度微调 | §5.1.6.1 |
