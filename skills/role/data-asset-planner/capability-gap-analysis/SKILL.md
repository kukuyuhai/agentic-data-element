# 📊 capability-gap-analysis

## When to Use

当以下场景出现时调用本技能：

- **规划 DCMM 升级路径**：企业当前处于某一 DCMM 等级，计划提升到更高等级，需要系统分析差距和制定行动方案
- **数据治理专项自查**：企业希望了解自身数据管理能力的薄弱环节，针对性投入改进
- **供应商/合作伙伴评估**：评估合作方的数据管理能力是否满足企业的准入标准
- **年度数据能力盘点**：作为年度数据战略回顾的一部分，系统性诊断当前能力不足
- **三/五年规划编制**：在制定中长期数据资产建设路线图之前，先完成能力基线评估和差距分析

## 工作原则

1. **全量评估，聚焦短板**：覆盖 DCMM 八大能力域，但输出重点放在差距最大的 2-3 个能力域
2. **定量+定性结合**：DCMM 评分提供定量基准，但改进建议需要结合企业业务场景做定性分析
3. **优先级驱动资源分配**：改进主题的优先级不只看差距大小，还需综合考虑业务影响度和实施可行性
4. **可落地**：每个改进主题必须包含具体的行动建议、估算投入和验收指标
5. **标准锚定**：所有等级判定和差距分析必须以 `GB/T 36073-2018` 为准绳

## DCMM 八大能力域简介

根据 `GB/T 36073-2018`：

| 能力域 | 关键能力项 | 评估要点 |
|-------|----------|---------|
| 1. 数据战略 | 战略规划、组织建设 | 是否有正式的数据战略文件？数据管理组织是否建立？ |
| 2. 数据治理 | 治理体系、治理实施 | 是否有数据治理委员会？治理流程是否运转？ |
| 3. 数据架构 | 架构设计、实施管理 | 数据模型是否统一？架构评审机制是否有效？ |
| 4. 数据标准 | 标准体系、标准执行 | 是否建立了数据标准？标准在开发和运维中是否贯彻？ |
| 5. 数据质量 | 质量管理、绩效评估 | 是否有质量规则库？质量问题是否闭环处理？ |
| 6. 数据安全 | 安全策略、技术防护 | 安全等级是否分类？是否有审计和追溯机制？ |
| 7. 数据应用 | 分析建模、数据服务 | 数据是否支持业务决策？是否有数据产品？ |
| 8. 数据生存周期 | 数据管理流程、归档销毁 | 是否有完整的数据生命周期管理流程？ |

## 差距分析方法论

### 第一步：获取当前 DCMM 评估结果

调用 `management/dcmm-scoring` 获取当前企业 DCMM 评估结果，应包含：
- 综合等级（1-5 级）
- 八个能力域各自得分（每域满分 5 分）
- 各能力域内的子项评分明细

**示例评估结果**：

| 能力域 | 当前得分 | 当前等级 |
|-------|---------|---------|
| 数据战略 | 2.3 | 2 级 |
| 数据治理 | 1.8 | 1 级 |
| 数据架构 | 2.5 | 2 级 |
| 数据标准 | 1.5 | 1 级 |
| 数据质量 | 2.0 | 2 级 |
| 数据安全 | 2.8 | 2 级 |
| 数据应用 | 2.2 | 2 级 |
| 数据生存周期 | 1.6 | 1 级 |
| **综合** | **2.1** | **2 级** |

### 第二步：定义目标等级

根据企业战略目标和行业对标，设定目标综合等级和各能力域目标得分。

**规则**：
- 目标综合等级对应的各域得分均不应低于该等级的基础线（如目标 3 级 → 各域 ≥ 3.0）
- 个别能力域可以有更高目标（如数据安全域 4 级，因为涉及核心风控竞争力）

### 第三步：差距量化

计算每个能力域的差距：

```
Gap_i = 目标得分_i - 当前得分_i (i = 1..8)
```

**差距等级判定**：

| 差距值 | 等级 | 标签 | 处置策略 |
|-------|------|------|---------|
| Gap ≥ 1.5 | 严重 | 🔴 | 需设立专项，重点投入 |
| 0.7 ≤ Gap < 1.5 | 中等 | 🟡 | 需制订专项改进计划 |
| 0 < Gap < 0.7 | 轻度 | 🟢 | 在日常工作中渐进改善 |
| Gap ≤ 0 | 达标 | ✅ | 保持即可 |

### 第四步：差距归因与改进主题识别

对每个差距较大的能力域，识别具体的薄弱子项和改进方向。

**示例归因**：

| 能力域 | 差距 | 薄弱子项 | 改进主题 |
|-------|------|---------|---------|
| 数据治理 | 1.2 (1.8→3.0) | 治理组织缺位、治理流程无运转 | **建立数据治理委员会和运转机制** |
| 数据标准 | 1.5 (1.5→3.0) | 无数据标准体系、标准未宣贯 | **建立数据标准体系和强制执行机制** |
| 数据生存周期 | 1.4 (1.6→3.0) | 数据归档无流程、销毁无记录 | **建立数据生命周期管理流程和审计** |

### 第五步：优先级排序

综合考虑三个维度确定改进优先级：

| 维度 | 权重 | 说明 |
|-----|------|------|
| 差距大小 | 30% | 差距越大优先级越高 |
| 业务影响度 | 40% | 该域改进对业务价值的直接贡献 |
| 实施可行性 | 30% | 资源可得性、技术难度、组织接受度 |

**优先级排序矩阵**：

```
                    业务影响度
                  高    中    低
               ┌─────┬─────┬─────┐
           大  │ P0  │ P1  │ P2  │
差距大小   中  │ P1  │ P2  │ P3  │
           小  │ P2  │ P3  │ P3  │
               └─────┴─────┴─────┘
```

### 第六步：投入估算

对每个 P0/P1 改进主题，估算所需投入：

| 改进主题 | 优先级 | 投入估算（万元） | 预计周期 | 主要资源 |
|---------|-------|----------------|---------|---------|
| 建立数据治理委员会 | P0 | 30-50 | 3 个月 | 管理层时间 + 1 咨询顾问 |
| 建立数据标准体系 | P0 | 50-80 | 6 个月 | 2 数据治理岗 + 业务域代表 |
| 数据分类分级完成 | P1 | 40-60 | 4 个月 | 1 安全工程师 + 1 合规顾问 |
| 数据质量闭环机制 | P1 | 60-100 | 8 个月 | 1 质量工程师 + 规则引擎平台 |

## Inputs

| 输入项 | 类型 | 必填 | 说明 | 示例 |
|-------|------|------|------|------|
| `current_dcmm_scores` | object | 是 | 当前 DCMM 八大域得分（可调用 `dcmm-scoring` 获取） | `{"data_strategy": 2.3, "data_governance": 1.8, ...}` |
| `target_dcmm_level` | number | 是 | 目标 DCMM 综合等级 | `3` |
| `target_domain_scores` | object | 否 | 各域目标得分（未指定则默认为目标等级对应基础线） | `{"data_security": 4.0}` |
| `business_priorities` | array | 否 | 业务优先关注的能力域，影响改进优先级 | `["数据安全", "数据应用"]` |
| `budget_available` | number | 否 | 可用预算（万元），用于投入估算的约束条件 | `500` |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| `gap_radar` | chart | 八域差距雷达图（文本版或 Mermaid 雷达图） |
| `gap_analysis_table` | table | 差距分析总表（每域当前/目标/差距/标签） |
| `improvement_themes` | list | 改进主题清单（含优先级、薄弱子项、行动建议） |
| `priority_matrix` | table | 优先级排序矩阵（差距×业务影响度） |
| `cost_estimation` | table | 投入估算总表（含分项和人天） |
| `timeline_recommendation` | object | 建议时间线和里程碑 |

## Steps

### Step 1：获取当前评估基线

1. 调用 `management/dcmm-scoring` 获取当前 DCMM 各域评分
2. 若评估结果不完整或缺失某些域，引导用户补充或使用替代评估方法
3. 确认评估结果的新鲜度（评估时间距今超过 1 年的，建议重新评估）

### Step 2：定义目标态

1. 确认用户的目标 DCMM 综合等级
2. 检查是否有特定能力域的更高要求（如数据安全域因监管要求需达 4 级）
3. 参考同行业对标企业的 DCMM 等级作为参考锚点

### Step 3：量化差距分析

1. 逐域计算 Gap = 目标得分 - 当前得分
2. 按差距值标注等级标签（🔴 严重 / 🟡 中等 / 🟢 轻度 / ✅ 达标）
3. 生成八域差距雷达图或表格视图
4. 识别差距最大的 Top 3 能力域（重点分析对象）

### Step 4：薄弱子项识别与改进主题生成

1. 对每个差距 ≥ 0.7 的能力域，深入分析具体的薄弱子项
2. 结合企业业务场景，为每个薄弱子项生成 1-2 个改进主题
3. 每个改进主题需包含：当前状态描述、目标状态描述、关键行动建议

### Step 5：优先级排序

1. 对每个改进主题评估业务影响度（高/中/低）
2. 对每个改进主题评估实施可行性（高/中/低）
3. 按"差距大小 + 业务影响度"矩阵输出 P0-P3 优先级
4. 结合用户指定的业务优先级进行校准

### Step 6：投入估算与时间线

1. 对所有 P0/P1 改进主题，估算投入金额和预计周期
2. 汇总总投入，与可用预算校准（若超预算，建议按优先级裁剪 P2/P3 项）
3. 生成建议时间线（分阶段的改进节奏，可对接 `asset-planning-roadmap`）

## 常见问法与处理策略

| 用户问法 | 处理策略 |
|---------|---------|
| "我们现在的 DCMM 能力和 3 级差多远？" | 调用 `dcmm-scoring` 获取当前评估 → 对比目标等级 → 输出八域差距表，标注严重/中等/轻度 |
| "哪个能力域差距最大？" | 输出按降序排列的差距表，高亮 Top 3，简要分析原因 |
| "先改哪个域比较划算？" | 结合差距大小和业务影响度，用优先级矩阵给出 P0/P1 建议，用投入产出比辅助说明 |
| "改进要花多少钱、多长时间？" | 对每个 P0/P1 改进主题分别估投入和周期，汇总输出，标注总投入和关键路径 |
| "DCMM 八大域全部分析太复杂，能聚焦吗？" | 先全面扫描确认差距分布，然后聚焦差距最大的 2-3 个域深度分析，其余域给出概要建议 |
| "目标定 3 级还是 4 级？" | 展示两套方案：目标 3 级的改进量 vs 目标 4 级的改进量，结合投入和周期对比，供用户选择 |
| "分析完了下一步做什么？" | 将改进主题和优先级输入 `asset-planning-roadmap`，生成分年度执行路线图 |
