# 💳 credit-scorecard-builder

数据信用评分卡构建工具。由数据信用评价机构（`dam-11`）使用，依据 T/MIITEC 025-2024 §5.1.11 要求，构建**数据质量 + 主体信用 + 交易表现**三维评分模型。输出标准化的评分卡（Scorecard），为 `credit-rating-committee` 的初评和 `credit-report-drafter` 的评级报告提供量化评分依据。

> 评分卡在使用前须通过 `credit-rating-committee` 的评审和备案。

## When to Use

- 首次搭建数据信用评分模型。
- 现有评分模型需要**参数调整**或**权重重新分配**。
- 评定某个数据交易主体或数据产品的信用评分。
- 信用政策要求对特定品类（如AI数据集）扩展评分维度。

## 工作原则

1. **三维均衡**：数据质量、主体信用、交易表现三大维度缺一不可。
2. **数据驱动**：权重分配和评分标准基于历史实际分布，避免纯经验设定。
3. **评分可解释**：每一分的增减都应有明确的指标和规则支撑。
4. **区分度优先**：评分模型应能将不同信用水平的对象有效区分（Gini > 0.4）。
5. **动态校准**：每半年基于新数据校准一次评分卡的参数。

## 一、评分维度

### 维度一：数据质量（权重 40%）

评价被评对象所提供/持有数据的质量水平。

| 二级指标 | 权重（维度内） | 评价内容 | 量化方法 |
|----------|----------------|----------|----------|
| 完整性 | 25% | 数据是否无缺失关键字段 | 缺失率 = 1 - (完整记录数 / 应有记录数) |
| 准确性 | 25% | 数据与真实情况一致 | 抽样验证准确率 |
| 时效性 | 20% | 数据是否及时更新 | 数据滞后天数 / 最大可接受滞后 |
| 合规性 | 20% | 数据来源与处理符合法律法规 | 合规性检查通过率（参见 `credit-red-lines` 结果） |
| 一致性 | 10% | 同一数据在不同来源是否一致 | 一致性检查通过率 |

**评分公式**：

```
数据质量得分 = Σ(二级指标得分 × 二级指标权重) × 100
数据质量得分归一化到 [0, 400] 区间
```

### 维度二：主体信用（权重 30%）

评价被评对象自身的企业信用。

| 二级指标 | 权重（维度内） | 评价内容 | 量化方法 |
|----------|----------------|----------|----------|
| 经营资质 | 25% | 注册时间、行业许可、资质等级 | 分段评分（如：>5年=100分） |
| 财务状况 | 30% | 资产负债率、营收规模、利润水平 | 行业排名百分位 |
| 诚信记录 | 25% | 是否有失信记录、行政处罚、涉诉 | 扣分制（每条 -X 分） |
| 行业地位 | 10% | 行业排名、市场占有率、品牌影响 | 行业排名映射 |
| 治理结构 | 10% | 股权结构清晰度、高管团队稳定性 | 专家评分 |

**评分公式**：

```
主体信用得分 = Σ(二级指标得分 × 二级指标权重) × 100
主体信用得分归一化到 [0, 300] 区间
```

### 维度三：交易表现（权重 30%）

评价被评对象在数据交易市场中的历史行为。

| 二级指标 | 权重（维度内） | 评价内容 | 量化方法 |
|----------|----------------|----------|----------|
| 成交率 | 25% | 挂牌产品成交率 | 成交数 / 挂牌数 |
| 履约率 | 30% | 按时、按质交付数据产品 | 履约次数 / 交易总次数 |
| 售后质量 | 20% | 售后响应速度、问题解决率 | 售后工单解决率 |
| 评价口碑 | 15% | 买方评价得分 | 加权平均分（>=4/5星） |
| 交易活跃度 | 10% | 交易频率和连续性 | 月均交易笔数 / 交易总月数 |

**评分公式**：

```
交易表现得分 = Σ(二级指标得分 × 二级指标权重) × 100
交易表现得分归一化到 [0, 300] 区间
```

### 总分计算

```
总分 = 数据质量得分 / 400 × 400 × 40%
     + 主体信用得分 / 300 × 300 × 30%
     + 交易表现得分 / 300 × 300 × 30%

简化：总分 = 数据质量得分 + 主体信用得分 + 交易表现得分
总分区间：[0, 1000]
```

## 二、权重分配方法

默认采用 **AHP（层次分析法）**，备选 **德尔菲法（专家打分法）**。

### AHP 步骤

1. 构建判断矩阵：三大维度两两比较重要性（1-9 标度）。
2. 计算权重向量：特征向量法求最大特征值对应的特征向量。
3. 一致性检验：CR = CI / RI，若 CR < 0.1 则一致性可接受。
4. 输出权重：各维度归一化权重（数据质量 40%、主体信用 30%、交易表现 30%）。
5. 二维权重迭代：对每个维度的二级指标重复同样过程。

### 特殊情况调整

- **纯产品评分**（无主体维度）：调整为主体信用 0%、商品质量 50%、交易表现 50%。
- **新入市场对象**（无交易历史）：调整为数据质量 50%、主体信用 50%，交易表现暂不计入。
- **数据服务商**：增加"服务能力"二级指标（服务 SLAs、工单效率等）。

## 三、评分标准

### 二级指标分段

以「履约率」为例：

| 履约率区间 | 得分 |
|------------|------|
| >= 98% | 100 |
| 95% - 98% | 90 |
| 90% - 95% | 75 |
| 80% - 90% | 60 |
| 70% - 80% | 40 |
| < 70% | 0 |

各二级指标的分段标准基于**历史数据的分布特征**设定，确保区分度。

### 分数异常处理

- **缺失值**：该指标取维度内其他指标加权平均分。
- **极端值（> 3σ）**：缩尾处理（winsorize），拉回到 3σ 边界。
- **样本不足（< 10 条记录）**：该指标标记为"样本不足，结果仅供参考"。

## 四、等级划分

| 等级 | 分数区间 | 等级描述 | 建议操作 |
|------|----------|----------|----------|
| AAA | 900 - 1000 | 信用极佳，违约风险极小 | 最高额度授信，优先合作 |
| AA | 800 - 899 | 信用优良，违约风险很小 | 常规额度，标准合作 |
| A | 700 - 799 | 信用良好，违约风险较小 | 常规额度，正常合作 |
| BBB | 600 - 699 | 信用一般，偶尔违约可能 | 缩减额度，加强监控 |
| BB | 500 - 599 | 信用较差，违约风险较大 | 限制额度，要求担保 |
| B | 400 - 499 | 信用差，违约风险大 | 最低额度，严格担保 |
| C | < 400 | 信用极差，高度违约可能 | 不建议授信 |

## Inputs

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `subject_id` | string | 是 | 评价对象唯一标识 | `"ENT-2024-00123"` |
| `subject_type` | enum | 是 | 对象类型 | `"数据交易主体"` / `"数据产品"` |
| `dq_result` | object | 是 | 数据质量评估结果 | 来自 `dq-6d-check` 的输出 |
| `entity_info` | object | 是 | 主体信息 | 含工商信息、财务数据、诚信记录 |
| `transaction_history` | object | 是 | 交易历史记录 | 含成交、履约、评价数据 |
| `weight_overrides` | object | 否 | 自定义权重（覆盖默认） | `{数据质量: 0.45, 主体信用: 0.25, 交易表现: 0.30}` |
| `segment_overrides` | object | 否 | 自定义分段标准 | `{履约率_95: 95, 履约率_90: 80}` |
| `red_line_result` | object | 是 | 红线审查结果 | 来自 `credit-red-lines` 的输出 |

## Steps

1. **红线前置检查**：确认 `red_line_result.passed == true`，如未通过则直接返回一票否决结果。
2. **数据预处理**：缺失值处理、极端值缩尾、数据类型转换和归一化。
3. **维度一评分**：计算数据质量的各二级指标得分，按权重加权得到维度分（0-400）。
4. **维度二评分**：计算主体信用的各二级指标得分，按权重加权得到维度分（0-300）。
5. **维度三评分**：计算交易表现的各二级指标得分，按权重加权得到维度分（0-300）。
6. **计算总分**：三维度得分求和得到总分（0-1000）。
7. **等级映射**：将总分映射到 AAA~C 等级。
8. **质量检查**：检查各维度得分是否存在异常（全0/全满分/单一指标驱动），做好标记。
9. **生成评分卡**：整合所有结果输出结构化评分卡。
10. **标注标准溯源**：各关键计算步骤标注引用的方法学来源。

## 常见问法与处理策略

| 问法 | 处理策略 | 注意事项 |
|------|----------|----------|
| "我们想调整权重，可以吗" | 可以自定义，在 `weight_overrides` 中传入，系统会记录实际使用的权重并在报告中说明调整原因 | 权重大幅偏离默认值时应附专家论证 |
| "新用户没有交易历史怎么办" | 使用"冷启动"模式：数据质量50%、主体信用50%，交易表现标注"N/A"；评分结果注明"基于不完整数据的参考评分" | 至少有3笔交易后才纳入交易表现维度 |
| "评分卡多久需要重新校准" | 默认每半年校准一次（基于新增数据）；市场环境剧变时提前校准 | 校准不应导致已评对象等级剧烈跳动 |
| "很多指标都是主观评分的，合理吗" | 对主观指标要求双人独立评分取均值；重要指标须有评分理由说明 | 主观评分的一致性是关键（Kappa > 0.6） |
| "和 credit-report-drafter 什么关系" | 评分卡是报告的输入之一；报告在评分卡基础上增加定性分析和风险展望 | 评分卡给出数字，报告给出结论和叙事 |
