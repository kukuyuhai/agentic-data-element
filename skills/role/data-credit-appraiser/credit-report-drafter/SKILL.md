# 📄 credit-report-drafter

信用评级报告起草工具。由数据信用评价机构（`dam-11`）使用，在完成 `credit-scorecard-builder` 评分并通过 `credit-rating-committee` 评审后，起草正式对外发布的**信用评级报告**。报告包含主体画像、数据资产质量、评分结果、主要风险分析和评级展望，是评级结论的最终呈现形式。

> 报告须经 `credit-rating-committee` 签发后方可对外发布。

## When to Use

- 完成评分卡计算和委员会评审后，需出具正式评级报告。
- 对外向交易合作方、监管机构或社会公众披露评级结论。
- 年度重评后更新评级报告。
- 被评方申请复评后出具新的评级报告。

## 工作原则

1. **结论先行**：评级报告以评级等级和展望为核心，其余段落均为支撑和解释。
2. **数据量化**：关键判断不空洞，尽可能引用数据和对比基准。
3. **风险不回避**：主要风险段必须具体、可操作，不可用套话代替。
4. **限制声明**：每份报告须包含明确的评级限制条件和使用声明。
5. **可被挑战**：每一结论性表述应有对应的工作底稿可追溯。

## 报告结构

### 一、评级概述

**必备要素**：

- **评级对象**：被评方全称、类型（数据交易主体/数据产品/数据服务）。
- **评级类型**：首次评级 / 年度重评 / 复评。
- **评级时点**：评级基准日及评级委员会上会时间。
- **评级等级**：信用等级（AAA~C）及其简要描述。
- **评级展望**：稳定 / 正面 / 负面。
- **有效期**：本次评级的有效期（通常为12个月）。

**示例**：

> 本次对 **[XX科技有限公司]** 进行 **[首次]** 信用评级，评级基准日为 **[2025-06-30]**，经信用评级委员会 **[2025-07-15]** 会议评审，评定信用等级为 **[AA]** 级，评级展望为 **[稳定]**。本评级有效期自 **[2025-07-15]** 起至 **[2026-07-14]** 止。

### 二、主体画像

对被评方的基本面貌进行概述，提供评级所需的背景信息。

| 子段 | 内容 | 数据来源 |
|------|------|----------|
| 基本信息 | 全称、注册资本、成立日期、法定代表人、注册地址 | 工商登记信息 |
| 经营范围 | 主营业务、行业分类、数据相关业务占比 | 工商登记 + 企业自述 |
| 经营状况 | 近2-3年主要财务指标（营收、利润、资产负债率） | 经审计的财务报表 |
| 治理结构 | 股权结构、实际控制人、组织架构、核心团队 | 工商登记 + 企业自述 |
| 行业地位 | 所处行业的市场份额、竞争优势、主要竞争对象 | 行业研究报告 |

**对比参考**：主体信息中的关键数据（如营收规模、注册年限等）可标注行业排名百分位。

### 三、数据资产质量

对被评对象的数据资产质量进行多维度评述。

| 子段 | 内容 |
|------|------|
| 数据资产概况 | 所持有/运营数据的类型、规模（记录数/存储量）、数据品类 |
| 质量评分 | 数据完整性/准确性/时效性/合规性/一致性的各维度得分（来自 `dq-6d-check` 和 `credit-scorecard-builder`） |
| 安全合规 | 数据安全保护措施（加密、访问控制、审计日志）、合规认证（等保/ISO 27001/DSMM） |
| 权属状况 | 数据持有权/加工使用权/产品经营权的确权情况，是否存在权属争议 |
| 不足之处 | 质量维度中得分偏低的部分（低于60%），需要改进的方面 |

### 四、评分结果

展示评分卡模型的详细计算结果。

| 子段 | 内容 |
|------|------|
| 总评分 | 总分（0-1000 分）、对应等级、在总体中的排名百分位 |
| 维度得分 | 数据质量（0-400）、主体信用（0-300）、交易表现（0-300）的三维度分值和权重 |
| 指标明细 | 各二级指标的具体得分和贡献度（高分项和低分项） |
| 评分变化 | 与上期评分的对比（如有），说明变化原因 |
| 评分方法说明 | 简要说明所采用的评分方法（AHP权重/AHP分段/冷启动等特殊情况处理） |

### 五、主要风险

逐类列出被评方面临的**具体风险**（非泛化描述）。

| 风险类别 | 关注点 | 风险等级 | 具体描述 |
|----------|--------|----------|----------|
| 技术风险 | 数据存储/处理技术的过时可能；关键技术人员流失 | 中/低 | [具体说明] |
| 市场风险 | 数据产品的市场需求变化；同类竞争品出现；客户集中度风险 | 中 | [具体说明] |
| 合规风险 | 监管政策变化带来的合规成本；数据跨境限制；PIPL/DSL 新规影响 | 高/中 | [具体说明] |
| 运营风险 | 数据采集能力受限、数据质量退化、服务水平下降 | 中/低 | [具体说明] |
| 财务风险 | 研发投入压力、现金流紧张、盈利模式不稳定 | 中 | [具体说明] |

**风险响应建议**：对每类高等级风险，给出简要的应对建议。

### 六、评级展望

| 展望 | 含义 | 判断依据 |
|------|------|----------|
| 正面 | 未来12月内评级可能上调 | 有明确的收入增长或质量改善趋势 |
| 稳定 | 短期内评级变动的可能性较低 | 经营环境稳定、趋势不明显 |
| 负面 | 可能出现导致评级下调的因素 | 已出现风险信号但尚未触发下调 |

### 七、限制条件

每份评级报告须包含的限制声明：

- 本评级基于评级基准日可获取的信息，此后发生的事件未纳入。
- 评级结论是对被评方信用水平的意见表达，不构成投资建议或交易推荐。
- 评级中使用的数据和信息来源于被评方或其公开披露，评级机构不对其真实性承担责任。
- 本报告未经评级机构书面同意，不得全文或部分复制、转载或用于其他目的。
- 评级报告的附属内容（评分卡、工作底稿）为内部工作文件，不对外提供。

## Inputs

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `subject_id` | string | 是 | 评级对象唯一标识 | `"ENT-2024-00123"` |
| `subject_type` | enum | 是 | 对象类型 | `"数据交易主体"` / `"数据产品"` |
| `rating_type` | enum | 是 | 评级类型 | `"首次评级"` / `"年度重评"` / `"复评"` |
| `scorecard_result` | object | 是 | 评分卡结果 | 来自 `credit-scorecard-builder` |
| `committee_resolution` | object | 是 | 委员会决议 | 来自 `credit-rating-committee` |
| `entity_profile` | object | 是 | 主体信息 | 工商信息、财务数据、行业排名 |
| `dq_result` | object | 是 | 数据质量评估 | 来自 `dq-6d-check` |
| `previous_report` | object | 否 | 上期评级报告 | 重评场景 |
| `custom_restrictions` | array | 否 | 额外限制条件 | 定制化限制声明 |

## Outputs

| 字段 | 类型 | 说明 |
|------|------|------|
| `rating_report` | string | 完整评级报告全文（Markdown/PDF-ready） |
| `executive_summary` | string | 管理层摘要（一页简报版） |
| `report_metadata` | object | 报告元数据（编号、日期、有效期、签发人） |
| `distribution_notice` | string | 发布/分发声明 |

## Steps

1. **确认输入完整性**：检查所有 required 字段是否齐全，previous_report 在重评场景下必填。
2. **撰写评级概述**：评级对象 → 类型 → 时点 → 等级 → 展望 → 有效期。
3. **撰写主体画像**：整合基本信息、经营状况、治理结构、行业地位。
4. **撰写数据资产质量**：描述数据概况、质量评分、安全合规、权属、不足。
5. **撰写评分结果**：总分/等级/排名、三维度得分、指标明细、变化说明、方法说明。
6. **撰写主要风险**：逐类列出具体风险，标注等级，给出响应建议。
7. **设定评级展望**：基于评分趋势和风险判断，设定正面/稳定/负面。
8. **添加限制条件**：插入标准限制声明，追加定制化限制（如有）。
9. **管理层摘要**：将核心结论浓缩为一页摘要。
10. **交叉检查一致性**：确保报告中的数字、结论、等级与 scorecard_result 和 committee_resolution 完全一致。
11. **标注标准溯源**：每个关键结论末尾标注评分卡结果编号或委员会决议编号。

## 常见问法与处理策略

| 问法 | 处理策略 | 注意事项 |
|------|----------|----------|
| "评级报告有多长" | 完整报告通常在 8-15 页（A4）；管理层摘要控制在 1 页以内 | 长度不重要，可读性和完整性重要 |
| "负面结论怎么写得体面" | 以事实和数据为基础，语气客观，避免主观贬义词汇；负面结论放入"主要风险"段而非概述段 | 既不失公允又不失尊重 |
| "报告中能不能提被评方的竞争对手" | 在"行业地位"段中可以客观提及，但不做主观优劣比较和排他性断言 | 仅引用可公开验证的信息 |
| "评分卡结果和报告结论冲突怎么办" | 最终以委员会决议为准，在报告中体现委员会裁决的理由（如定性调整） | 量化和定性不一致本身就是需要解释的点 |
| "报告发给谁、怎么发" | 报告主送被评方、抄送相关监管/交易所（按信用政策规定）；电子版加密发送，纸质版签字密封 | 未经授权不得向第三方透露评级结论 |
