# 📈 gmv-forecast-holt-winters

GMV 时序预测——Holt-Winters 三参数模型与 Prophet 自动化预测，含季节性与置信区间。

## When to Use

- 季度目标制定前，需要基于历史数据预测下季度 GMV 走势时
- 供需匹配预警，需要预测供给或需求是否会断档时
- 月度/季度运营会议，需要展示 GMV 预测趋势辅助决策时
- 新品类上线评估，需要预测新品类对整体 GMV 的贡献时

## 工作原则

1. **数据先行**：预测基于至少 2 个完整周期（如 2 年日数据或 8 个季度数据）的历史 GMV
2. **多模型对比**：同时运行 Holt-Winters 和 Prophet，对比 MAPE/RMSE 选优
3. **置信区间**：预测结果必须给出 80% 和 95% 置信区间，不可只给点估计值
4. **季节性优先**：数据交易存在明显季节性（月末预算集中、年底采购高峰），必须建模季节性
5. **人工修正**：模型预测值作为基线，结合业务判断（政策变化/竞争动态）做人工修正 `§5.2.4`

## 预测方法详解

### 一、Holt-Winters 三参数模型

**模型原理：** 将时间序列分解为三个分量——水平（level）、趋势（trend）、季节（seasonal），分别用 α、β、γ 三个平滑参数控制更新速度。

**数学表达：**

```
水平：   L_t = α(Y_t - S_{t-m}) + (1-α)(L_{t-1} + T_{t-1})
趋势：   T_t = β(L_t - L_{t-1}) + (1-β)T_{t-1}
季节：   S_t = γ(Y_t - L_t) + (1-γ)S_{t-m}
预测：   Ŷ_{t+h} = L_t + h·T_t + S_{t-m+h}
```

其中 m 为季节周期（日数据 m=7，月数据 m=12）。

**适用条件：**

| 条件 | 说明 |
|------|------|
| 数据量 | 至少 2 个完整季节周期 |
| 趋势特征 | 存在线性趋势（增长或衰减） |
| 季节特征 | 存在固定周期的季节性波动 |
| 异常值 | 少量异常值可接受，大量需预处理 |

**参数调优：**

| 参数 | 作用 | 推荐范围 | 调优方法 |
|------|------|---------|---------|
| α (alpha) | 控制水平更新速度 | 0.01-0.3 | 网格搜索最小化 MAPE |
| β (beta) | 控制趋势更新速度 | 0.01-0.1 | 网格搜索最小化 MAPE |
| γ (gamma) | 控制季节更新速度 | 0.01-0.2 | 网格搜索最小化 MAPE |

### 二、Prophet 模型

**模型原理：** Facebook 开源的加法模型，将时间序列分解为趋势、季节性、节假日效应三个组分。

**核心公式：**

```
y(t) = g(t) + s(t) + h(t) + ε_t

g(t)：趋势项（线性或逻辑增长）
s(t)：季节项（傅里叶级数）
h(t)：节假日效应
ε_t：噪声项
```

**优势对比：**

| 优势 | 说明 |
|------|------|
| 自动处理节假日 | 内置中国法定节假日，自动建模节假日效应 |
| 异常值鲁棒 | 对异常值不敏感，无需大量预处理 |
| 缺失值友好 | 可处理不规律时间序列和缺失值 |
| 可解释性强 | 各组分可分解展示，便于理解预测来源 |
| 自动调参 | 默认参数即可获得较好效果 |

### 三、评估指标

| 指标 | 公式 | 说明 | 合格标准 |
|------|------|------|---------|
| MAPE | mean(\|Y-Ŷ\|/Y) × 100% | 平均绝对百分比误差 | < 15% |
| RMSE | √(mean((Y-Ŷ)²)) | 均方根误差 | 越小越好 |
| MAE | mean(\|Y-Ŷ\|) | 平均绝对误差 | 越小越好 |
| R² | 1 - SS_res/SS_tot | 决定系数 | > 0.7 |

## 季节性检测

**检测方法：**

1. **可视化**：绘制历史 GMV 时序图，肉眼观察周期性波动
2. **ACF/PACF**：自相关函数和偏自相关函数，识别季节周期
3. **STL 分解**：将序列分解为趋势+季节+残差，量化季节强度
4. **周期图**：频域分析，识别主要周期频率

**数据交易常见季节性模式：**

| 周期 | 模式 | 原因 |
|------|------|------|
| 日内 | 工作日 10-11点、14-15点高峰 | 工作时间集中下单 |
| 周内 | 周一到周三高，周末低 | 工作日采购 |
| 月内 | 月末冲量，月初低谷 | 预算月末集中消耗 |
| 年内 | Q4 高峰，Q1 低谷 | 年底采购预算冲刺 |

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 历史 GMV 数据 | TimeSeries | 日/月粒度的历史 GMV（至少 2 年） |
| 节假日日历 | List | 中国法定节假日及调休安排 |
| 业务事件标注 | List | 历史上的重大业务事件（如促销、政策变化） |
| 品类明细 | TimeSeries | 各品类的 GMV 历史（用于品类分解预测） |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 预测报告 | Document | 点估计 + 置信区间 + 模型评估 |
| 预测数据 | TimeSeries | 预测值序列（含上下界） |
| 季节性分析 | Document | 季节性模式和强度分析 |
| 模型评估 | Document | MAPE/RMSE/R² 等评估指标 |

## Steps

1. 收集历史 GMV 数据，确保数据质量和连续性
2. 执行季节性检测，确定季节周期和模式
3. 数据预处理：处理异常值、填充缺失值、标注业务事件
4. 同时训练 Holt-Winters 和 Prophet 模型
5. 对比模型评估指标（MAPE/RMSE），选择最优模型
6. 生成预测值和置信区间
7. 结合业务判断进行人工修正
8. 输出预测报告 `§5.2.4`
9. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| GMV 预测用什么模型？ | 同时用 Holt-Winters 和 Prophet，对比 MAPE 选优；有节假日效应优先 Prophet |
| 需要多长历史数据？ | 至少 2 个完整季节周期（日数据至少 2 年，月数据至少 2 年） |
| 预测不准怎么办？ | 检查数据质量、季节性是否变化、是否有异常事件；增加特征或换模型 |
| 置信区间什么用？ | 给出预测范围而非单点值，80% 区间做保守目标，95% 区间做风险边界 |
| 怎么处理促销带来的异常值？ | Prophet 可自动处理；Holt-Winters 需要在促销期间做数据清洗或标注 |
| 品类层面能预测吗？ | 可以，按品类分别建模预测，再加总得到整体预测，但需注意品类间相关性 |
