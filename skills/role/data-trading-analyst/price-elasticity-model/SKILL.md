# 💹 price-elasticity-model

价格弹性模型——估计需求对价格的敏感度，基于 MR=MC 原则输出利润最大化建议价区间。

## When to Use

- 数据产品调价决策前，需要评估价格变动对销量的影响时
- 新产品定价，需要找到利润最大化的价格点时
- 竞品价格变化，需要评估是否跟随调整时
- 促销折扣设计，需要确定最优折扣力度时

## 工作原则

1. **数据驱动**：弹性系数基于历史价格-销量数据计算，而非主观判断
2. **分类分析**：不同品类、不同形态的产品弹性差异大，需分别建模
3. **动态更新**：弹性系数随市场变化而变化，建议每季度重新计算
4. **结合成本**：定价建议不仅看弹性，还需结合边际成本（MR=MC 原则）
5. **区间输出**：输出建议价区间而非单一价格点，留出谈判空间 `§5.2.4`

## 弹性模型详解

### 一、数据需求

| 数据项 | 粒度 | 时间范围 | 说明 |
|-------|------|---------|------|
| 历史价格 | 日/周 | 至少 6 个月 | 产品挂牌价及实际成交价 |
| 历史销量 | 日/周 | 同上 | 对应时期的成交量 |
| 成本数据 | 月 | 同上 | 边际成本（数据采集+加工+交付成本） |
| 竞品价格 | 周 | 同上 | 同类产品的市场报价 |
| 业务事件 | — | 同上 | 促销、调价、新品上线等标注 |

### 二、弹性系数计算

**基本公式：**

```
价格弹性系数 (Ed) = (ΔQ/Q) / (ΔP/P)
                   = (需求量变化率) / (价格变化率)
```

**分类标准：**

| 弹性系数 | 分类 | 含义 | 定价策略 |
|:-------:|:----:|------|---------|
| \|Ed\| > 1 | 富有弹性 | 价格敏感，降价可增收 | 适度降价，走量策略 |
| \|Ed\| = 1 | 单位弹性 | 价格变化与需求等比变化 | 维持现价 |
| \|Ed\| < 1 | 缺乏弹性 | 价格不敏感，可提价 | 适度提价，利润策略 |
| Ed = 0 | 完全无弹性 | 价格不影响需求 | 按价值定价 |

**计算方法：**

| 方法 | 适用场景 | 公式 |
|------|---------|------|
| 弧弹性 | 两次价格变动之间 | Ed = (ΔQ/平均Q) / (ΔP/平均P) |
| 点弹性 | 连续价格变动 | Ed = (dQ/dP) × (P/Q) |
| 回归法 | 多次价格变动 | ln(Q) = α + β·ln(P) + ε，Ed = β |

### 三、建议价区间计算

**利润最大化原则：** 边际收益 = 边际成本（MR = MC）

**推导过程：**

```
设需求函数：Q = a - bP（线性需求）
收益函数：  R = P × Q = P(a - bP) = aP - bP²
边际收益：  MR = dR/dP = a - 2bP
边际成本：  MC = c（单位成本）

令 MR = MC：
  a - 2bP = c
  P* = (a - c) / (2b)     ← 利润最大化价格

考虑弹性：
  当 |Ed| > 1 时，降价使 P* 向下偏移
  当 |Ed| < 1 时，提价使 P* 向上偏移
```

**建议价区间：**

| 价格点 | 计算方法 | 用途 |
|-------|---------|------|
| 下限价 | P* × (1 - 弹性调整系数) | 促销底价、走量策略 |
| 中位价 | P*（利润最大化点） | 标准挂牌价 |
| 上限价 | P* × (1 + 弹性调整系数) | 议价上限、独家产品 |

### 四、敏感性分析

**价格变动对关键指标的影响：**

| 价格变动 | 销量变化（弹性=1.5） | 收入变化 | 利润变化 |
|:-------:|:-------------------:|:-------:|:-------:|
| -20% | +30% | +4% | +12% |
| -10% | +15% | +3.5% | +8% |
| 0% | 0% | 0% | 0% |
| +10% | -15% | -6.5% | -10% |
| +20% | -30% | -16% | -22% |

> 上表假设弹性=1.5（富有弹性），此时降价有利、提价不利。

## Inputs

| 输入项 | 类型 | 说明 |
|-------|------|------|
| 历史价格-销量数据 | TimeSeries | 至少 6 个月的日/周粒度数据 |
| 边际成本数据 | Float | 单位产品的边际成本 |
| 竞品价格 | List | 同类产品的市场价格 |
| 历史调价记录 | List | 过去的调价事件及效果 |

## Outputs

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 弹性分析报告 | Document | 弹性系数 + 分类 + 建议价区间 |
| 敏感性分析表 | Document | 不同调价幅度的影响预测 |
| 定价建议 | Document | 推荐价格策略和调价方案 |

## Steps

1. 收集历史价格-销量数据，确保数据质量和连续性
2. 数据预处理：清洗异常值、标注促销事件、对齐时间粒度
3. 计算弹性系数（弧弹性 + 回归弹性）
4. 判断弹性分类（富有/单位/缺乏弹性）
5. 基于 MR=MC 原则计算利润最大化价格 P*
6. 结合弹性调整系数计算建议价区间
7. 执行敏感性分析，预测不同调价方案的影响
8. 输出弹性分析报告和定价建议 `§5.2.4`
9. 命中他岗职责时调用 `foundation/handoff-router` 转交

## 常见问法与处理策略

| 常见问法 | 处理策略 |
|---------|---------|
| 这个产品该不该降价？ | 先算弹性系数：\|Ed\|>1 则降价有利（走量增收），\|Ed\|<1 则提价有利（利润优先） |
| 建议价怎么算出来的？ | 基于 MR=MC 原则计算利润最大化价格 P*，再结合弹性调整输出区间 |
| 弹性系数需要多久更新一次？ | 建议每季度重新计算，市场环境变化时（如竞品调价、政策变化）即时更新 |
| 数据太少能算弹性吗？ | 至少需要 6 个月且期间有 2 次以上价格变动，否则结果不可靠 |
| 竞品降价要不要跟？ | 先算自己产品的弹性，如果缺乏弹性（\|Ed\|<1）可以不跟；如果富有弹性需评估跟随幅度 |
| 促销折扣设多少合适？ | 用敏感性分析表找利润最大化的折扣点，通常降价 10-20% 在弹性>1 时有利 |
