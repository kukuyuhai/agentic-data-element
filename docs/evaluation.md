# 评价体系（Evaluation）

数字员工作为标准岗位的"数字化实体"，同样需要按 T/MIITEC 025-2024 的等级要求
进行**能力评价**、**产出评价**与**风险评价**，以指导选型、优化与迭代。

## 1. 三层评价

| 层次 | 评价对象 | 频次 | 主责 |
|---|---|---|---|
| L1 能力评价 | 岗位能力覆盖度 | 版本发布时 | 项目维护者 |
| L2 产出评价 | 交付物质量 | 每次任务后 | 使用方 / 客户 |
| L3 风险评价 | 幻觉 / 越权 / 合规 | 持续 | 使用方 + 合规专员 |

## 2. L1 · 能力评价（Capability Coverage）

### 2.1 评分维度

| 维度 | 满分 | 说明 |
|---|---|---|
| 知识条目覆盖率 | 30 | agent 文件覆盖标准 §5.X.X a) 的条目数 / 标准条目总数 |
| 技能条目覆盖率 | 20 | 覆盖标准 §5.X.X b) 条目 |
| 工程实践条目覆盖率 | 30 | 覆盖标准 §5.X.X c) 条目 |
| 交付物模板可复制性 | 10 | 至少 3 个可直接复制的模板 |
| 协作对象完整性 | 10 | 上下游岗位声明齐全 |

### 2.2 采集方式

```bash
./scripts/validate.sh                # 结构性校验
./convert.sh index                   # 生成 dist/agents.index.json
# 结合 roster.json + agents.index.json 计算覆盖率
```

## 3. L2 · 产出评价（Output Quality）

针对每一次任务的交付物评分：

| 指标 | 权重 | 满分标准 |
|---|---|---|
| 结构完整性 | 20 | 严格按交付物模板输出 |
| 内容准确性 | 30 | 事实、法条、数字均可复核 |
| 标准符合度 | 20 | 明确引用 §5.X.X / GB/T / 法规 |
| 决策价值 | 20 | 结论明确 + 可行动 |
| 语言与可读性 | 10 | 结构清晰、专业术语准确 |

### 3.1 打分模板

```yaml
task_id: T-2026-0001
agent: DAM-07
task: 为 XX 数据集出具评估报告
scores:
  structure: 18
  accuracy:  27
  compliance: 20
  actionable: 18
  readability: 9
total: 92     # 90+ 为优秀
reviewer: 张三
review_at: 2026-01-15
```

## 4. L3 · 风险评价（Risk）

| 风险 | 描述 | 检测手段 |
|---|---|---|
| 幻觉 | 编造法条、编造数字 | 交付物中所有引用需可溯源 |
| 越权 | 越出岗位职责回答 | 检查是否引用了非本岗位标准章节 |
| 合规 | 涉密 / 涉个人信息处理 | 交付物需过 DAM-04 / DAT-02 合规审阅 |
| 幂等性 | 相同输入产生冲突输出 | 抽样重放测试 |

## 5. 数字员工评级映射

| 得分区间 | 评级 | 处置 |
|---|---|---|
| ≥ 90 | 卓越（S） | 优先推荐，作为最佳实践沉淀 |
| 80 ~ 89 | 优秀（A） | 常规使用 |
| 70 ~ 79 | 合格（B） | 复核后使用 |
| 60 ~ 69 | 及格（C） | 需人工修订 |
| < 60 | 不合格（D） | 停用并回炉 |

## 6. 与人才评价的映射

本项目虽以数字员工为主体，但天然可用作**真人能力评价的知识底座**：

- 每个 agent 文件即是一份"岗位胜任力字典"；
- L2 交付物评分体系可直接迁移用于真人；
- L3 风险评价可对接内部合规审计。
